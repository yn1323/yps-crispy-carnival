# ナビゲーション・コンテナ（モーダル/ドロワー/BottomSheet/トースト）

「どこに置く？」「どう開く？」の判断材料。

## ナビゲーション全体構造

### アプリレベル

| 部品 | 使い時 |
|---|---|
| **TopNav** | 横幅広い、グローバル機能少なめ、ロゴ+主要セクション |
| **SideNav** | セクション多い、深い階層、PC前提 |
| **BottomNav** | モバイル中心、3〜5個の主要タブ |
| **TabBar(上部)** | セクション内の切替（同コンテンツの別ビュー） |

このプロジェクトはMVP段階で**SideMenu使わない・BottomMenuのみ**。

### セクションレベル

| 部品 | 使い時 |
|---|---|
| **Tabs** | 同レベル・並列セクションの切替（プロフィール/設定/通知） |
| **Breadcrumb** | 階層が3+、戻る経路を示す |
| **AnchorNav** | 1ページ内の長いセクションをスキップ（追従） |
| **Stepper** | 多段プロセス（1→2→3） |
| **Pagination** | 多ページ一覧 |

### Tab vs Page

- **Tab**：軽量切替・同コンテキスト・URL不要 or アンカー
- **Page**：URLで共有したい・戻れる・別コンテキスト

## モーダル系の選択

```
編集・確認・選択・通知のどれ？
├── 編集 (フォーム)
│   ├── 1〜3項目 → Popover / インライン
│   ├── 4〜10項目 → Dialog (PC) / BottomSheet (Mobile)
│   ├── 11+ or 段階的 → 専用ページ / Drawer
│   └── 進行中追従できる → Drawer (右、非ブロッキング)
├── 確認
│   ├── 破壊的 → AlertDialog (赤ボタン、結果明示)
│   ├── 重要選択 → Dialog (オプション提示)
│   └── 軽微（戻れる） → 確認なし or Toast undo
├── 選択
│   ├── リッチピッカー → Popover / Dialog
│   └── 単純選択 → Select / Combobox
└── 通知
    ├── 一時的・成功 → Toast
    ├── 永続・重要 → Banner
    └── 緊急ブロッキング → Modal (極力避ける)
```

## Dialog（モーダル）

### 使うべき時
- ユーザーの注意を強制したい（操作完了が必要）
- フォーム入力が必要だが新ページ作るほどでない
- 確認が必要な操作

### 構造
- Title（強）
- 本文（簡潔）
- アクション（右下：Primary、左：Cancel）
- 閉じる × は右上
- 背景にoverlay（暗い半透明）
- ESCで閉じる、focusトラップ

### 鉄則
- **モーダル乱用禁止**：ナビゲーションを殺す
- **モーダルの中にモーダルを開かない**：迷子になる
- **長すぎる内容は避ける**：スクロールするモーダルは設計失敗
- **モバイルでDialog → BottomSheet**：自動切替推奨

## Drawer

### 使うべき時
- 詳細表示・編集を本体と並行で見たい
- 一覧 → 選択行 → 右からスライド詳細
- 設定パネル
- 段階的な詳細（Wizard的）

### 構造
- 右 or 左から（左ナビ用途以外は基本右）
- 幅：400〜600px
- 閉じる × 右上
- 背景は dimmed しない場合もある（操作続行可）

### Modal vs Drawer
- **Modal**：完了が必要、本体操作止まる
- **Drawer**：補助、本体並行可

## BottomSheet（モバイル）

### 使うべき時
- モバイルでDialog相当
- 選択肢のピッカー
- フォーム入力
- 詳細プレビュー

### バリエーション
- **コンパクト**：高さコンテンツ依存（半分以下）
- **拡張可能**：ドラッグで全画面まで広げられる
- **フルスクリーン**：実質ページ遷移

### 注意（このプロジェクト規約）
- **内部Selectは `usePortal={false}`**（Portalだとシート背後に出る）
- **ドロップダウンがクリップ → `overflowY="visible"`**
- 下端に固定ボタン置くなら、コンテンツに padding-bottom

## Popover

### 使うべき時
- コンテキストアクション・補助情報
- 小さなフォーム（〜3項目）
- 軽量な選択（フィルター、色選択）

### 構造
- 起点要素にanchor
- 矢印（任意）で起点を示す
- 外側クリックで閉じる
- ESCで閉じる
- z-indexはモーダル下、tooltip上

### Popover vs Tooltip
- **Tooltip**：純粋にラベル・説明、非インタラクティブ、hover only
- **Popover**：操作可、クリックでトリガー、リッチコンテンツ

## Menu（DropdownMenu）

### 使うべき時
- 派生アクション（複数選択肢）
- "..." overflow
- ユーザーアバターのドロップダウン

### 構造
- 起点：Button or IconButton（▼ or …）
- 項目：シンプル・短文・アイコン任意
- セクション分け（divider）
- 危険な項目は赤字 + 区切り

### 鉄則
- ネスト（サブメニュー）は1段まで
- スマホでは BottomSheet の方が指で選びやすい
- キーボード操作対応（↑↓Enter）

## Tooltip

- hover後 500〜700ms で表示（即出ると鬱陶しい）
- 触れたら即消える / focus でも表示（キーボード対応）
- インタラクティブにしない（リンクや button を入れない → Popoverへ）
- 自明なボタンには付けない（"保存" ボタンに「保存します」は不要）

## Toast / Snackbar

### 使うべき時
- アクション完了の通知（保存しました、送信しました）
- 軽微なエラー（接続復旧）
- 取り消し可能アクション（削除 → 元に戻す）

### 構造
- 画面端（多くは右上 or 右下、モバイルは下中央）
- 4〜6秒で自動消失
- アクション付き（[元に戻す]）の場合 7〜10秒
- 同時複数 → スタック（上限3件）

### 鉄則
- 重要情報を Toast に置かない（見逃す）
- エラーで操作がブロックされる場合 → Toast ではなく Banner / Dialog
- "成功"はあえて出さない選択肢もある（保存後ページ遷移なら自明）

## Banner / Alert

### 使うべき時
- 永続的な状態通知（メンテナンス、トライアル残日数、未確認の警告）
- ページ全体に関わる注意

### 構造
- ページ最上部 or 該当セクション直上
- アイコン + 簡潔な本文 + アクション（任意）+ ×（dismissible 時）
- 色は意味（warning=orange、error=red、info=blue、success=green）

## ページ遷移 vs オーバーレイ

迷ったときの基準：

| 観点 | ページ | オーバーレイ |
|---|---|---|
| URL共有したい | ✅ | ❌ |
| 戻るで戻れる | ✅ | △ |
| 元データを見ながら操作 | ❌ | ✅ |
| 軽量な確認 | ❌ | ✅ |
| 長い手順 | ✅ | ❌ |
| モバイル中心 | △ | ✅ (BottomSheet) |
| 完了強制 | △ | ✅ |

## モバイル特有のナビ

- **下部固定CTA**：プライマリーは親指届く位置
- **FAB (Floating Action Button)**：1個まで、画面右下、追加・作成のみ
- **戻るボタン**：左上、システム戻ると一致
- **ハンバーガーメニュー**：項目多い場合、BottomNav優先で

## キーボードショートカット

PC生産性ツールなら：
- `?` でショートカット一覧
- `/` で検索フォーカス
- `g` + `letter` でGoTo（Gmail風）
- `cmd+k` でコマンドパレット
- `esc` で閉じる

ショートカットを追加したらDocumentation必須。

## アクセシビリティ

- モーダル開いたら focus を内部に移す（focus trap）
- 閉じたら起点要素に focus を戻す
- ESCで閉じる
- 背景スクロールロック
- `aria-modal="true"`、`role="dialog"`、`aria-labelledby`
- フォーカスインジケータ（outline）を消さない
