# データ表示（一覧・テーブル・カード・ダッシュボード）

データの形・量・利用目的に応じた見せ方の判定。

## まず「読みたいか・操作したいか」を判定

| 目的 | 推奨 |
|---|---|
| 比較したい（スペック表） | Table |
| 一件ずつ把握したい | List / Card |
| 量・傾向を掴みたい | Chart / KPI |
| 操作したい（編集・移動） | List with actions / Kanban |
| ブラウズ（眺める） | Grid of cards |

## Table（テーブル）

### 使うべき時
- 多属性（5+）を横並び比較したい
- ソート・絞り込みが価値を生む
- 数値が多く、桁を揃えたい

### 構造
- 1行=1レコード
- 列ヘッダー：固定（sticky）、ソート可
- セル：左寄せ（テキスト）/ 右寄せ（数値）/ 中央（アイコン・状態）
- 行hover：薄い背景色
- 選択：先頭にCheckbox列、選択行は色変化
- 行アクション：右端の `…` Menu or hover時に出るボタン

### 設計の鉄則
- **列を増やしすぎない**：6〜8が上限。それ以上は隠す or 詳細に逃がす
- **重要な列は左**：左から順に重要度
- **数値は右寄せ + 等幅 or tabular-nums**：桁が揃う
- **長文は省略**：`...` + Tooltipで全文
- **グルーピング**：同じカテゴリは罫線 or 背景で区切る

### 件数別

- 〜50件：そのまま全表示 + ソート
- 50〜500：ページネーション
- 500+：仮想スクロール、検索必須
- 数千+：サーバーサイドページング、絞り込み必須

### モバイルでのTable

- 横スクロール（重要列を sticky）
- または **Tableをカードに変換**：1行 → 1カード（ラベル: 値）

## List（リスト）

### 使うべき時
- 1属性が主、補助属性が少数
- 行ごとにアクション
- スクロールで連続的に見る

### 構造
- 左：アイコン or アバター（任意）
- 中央：主タイトル + サブタイトル
- 右：時間・ステータス・アクション
- divider or 余白で行を分ける

### バリエーション
- **Single line**：タイトルのみ（メニュー項目）
- **Two line**：タイトル + サブ
- **Three line**：タイトル + サブ + プレビュー

### 行アクション
- swipeアクション（モバイル）：左スワイプで削除等
- hover時にアクションボタン表示（デスクトップ）
- タップで詳細遷移 or 展開

## Card（カード）

### 使うべき時
- 不均質な情報単位（タイトル・画像・タグ・メタ）
- 視覚的にブラウズ
- グリッドで配置

### 構造
- 画像 / アイコン（任意・上 or 左）
- タイトル（強）
- メタ情報（弱・gray）
- アクションボタン（任意・下 or 右）

### 鉄則
- **クリッカブル全体**：カード全体がクリックなら、内側にネストボタンを置かない（イベント衝突）
- **均質グリッドは情報階層を殺す**：重要なカードは大きく、その他を小さく
- **影は控えめ**：1段階のみ。複雑な影は乱雑感
- **角丸を統一**：全カード共通（4 or 8 or 12）

### Card vs Tile
- Card：詳細あり（タイトル+説明+アクション）
- Tile：シンプル・サムネ的（画像 + 短いラベル）

## Grid

### 使うべき時
- 視覚的（画像・サムネ）
- カテゴリ・ジャンル選択
- ポートフォリオ・ギャラリー

### グリッドの選び方
- **均等グリッド**：3列 / 4列。シンプル・整然
- **メイソンリー（Pinterest風）**：高さがバラバラ。視覚密度高
- **ハイブリッド**：メイン1つ + サブ複数（特集記事レイアウト）

### 件数とブレークポイント
- スマホ：1〜2列
- タブレット：2〜3列
- デスクトップ：3〜5列

## Timeline / Feed

### 使うべき時
- 時系列イベント（更新履歴、SNSフィード）
- ステータスの推移

### 構造
- 縦の中軸（線）+ 各イベント
- イベント：日時 / アイコン / タイトル / 詳細
- 新→旧 or 旧→新（用途で決める。SNSは新→旧、履歴は旧→新が多い）

## Tree

### 使うべき時
- 階層データ（ファイルシステム、組織図、カテゴリ）
- ドリルダウン的な操作

### 構造
- インデント + 開閉アイコン（▶ → ▼）
- アクティブパスをハイライト
- 大量階層 → 仮想化必須

## Kanban

### 使うべき時
- ワークフロー状態の遷移（Todo / Doing / Done）
- 状態ごとに件数を把握したい

### 構造
- 列 = ステータス
- 列内 = カードのリスト
- ドラッグ&ドロップで状態変更
- 各列の件数表示

### 注意
- 列が5+ → 横スクロール、状態を統合検討
- カード件数が爆発 → 折り畳み、検索

## Calendar

### 使うべき時
- 予定・スケジュール
- 日次・週次・月次の俯瞰

### バリエーション
- **月ビュー**：日ごとセル、複数イベントは省略
- **週ビュー**：時間軸（縦）× 曜日（横）
- **日ビュー**：1日の時間軸詳細
- **アジェンダビュー**：リスト風、モバイル向き

## Dashboard

### 構造（標準）

```
┌─────────────────────────────┐
│  Header (期間切替, アクション) │
├─────────────────────────────┤
│  KPI Card × 4〜6            │  ← 上部：今の状態を秒で
├─────────────────────────────┤
│  Trend Chart × 1〜3         │  ← 中部：推移、内訳
├─────────────────────────────┤
│  Detail Table / Top N       │  ← 下部：深堀り
└─────────────────────────────┘
```

### KPIカード

- 大きい数字（24〜36px）
- ラベル（gray, sm）
- 比較（前月比 ±%、矢印）
- 任意：スパークライン
- カード自体クリックでドリルダウン

### グラフ選択

| 目的 | グラフ |
|---|---|
| 時系列推移 | LineChart / AreaChart |
| 比較（カテゴリ） | BarChart |
| 構成比 | DonutChart / StackedBar（Pieは扇<5でないと読みづらい） |
| 相関 | ScatterPlot |
| 分布 | Histogram |
| 地理 | Map |

### グラフの鉄則
- **凡例は近くに**（離すと往復が必要）
- **色は意味**：series ごとに固定色（途中で入れ替えない）
- **0起点**：BarChartは必須（Lineは状況次第）
- **軸ラベル略さない**：「2024-01」より「24/1月」より「1月」（前後でわかれば）
- **3D禁止**：歪んで読めない
- **要素数を絞る**：7+ seriesは別グラフに

## ページネーション・ロード戦略

| 戦略 | 使い所 |
|---|---|
| **Pagination（番号）** | 戻りたい・参照する・件数把握重要 |
| **Load More（ボタン）** | 主にスクロール、戻ること少ない、明示制御欲しい |
| **Infinite Scroll** | フィード型・娯楽・離脱しても良い |
| **仮想スクロール** | 件数が大量、すべての行を一覧したい |

**Infinite ScrollはFooterが届かない**問題に注意。重要リンクはサイドに配置。

## フィルタ・ソート・検索

### フィルタの置き場所
- **インライン chips**：少数（3〜5）の頻出フィルタ
- **左サイドバー**：多数のファセット（EC、検索結果）
- **トップバー**：ソート + 数件のフィルタ
- **モバイル**：BottomSheet で「絞り込み」ボタン

### 適用済み表示
- 適用中のフィルタを Chip 表示（×で個別解除）
- "すべてクリア" を必ず用意

### 検索
- 入力即検索（debounce 250〜400ms）
- 検索結果ハイライト（マッチ部分）
- ゼロ件 → 条件提案（「条件を緩める」「全期間で検索」）

## バルクアクション

- 行Checkbox選択 → 上部に「選択中: 3件 [削除] [移動]」バー
- 全選択チェック（ヘッダー）
- 選択解除ボタン

## 空状態

データ表示で必須：
- 初回（まだない）：「最初の◯◯を作りましょう」 + Primary CTA
- ゼロ件（フィルタ後）：「条件に合うデータがありません」 + 「条件をクリア」
- 削除後：シンプルに「データがありません」
