# 招待機能

## 概要

マジックリンクを使用してスタッフやマネージャーを店舗に招待する機能。招待されたユーザーはリンクをクリックするだけでアクセス可能。

## 関連ファイル

- **Routes**: `src/routes/invite.tsx`
- **Pages**: `src/components/pages/Invite/`
- **Features**: `src/components/features/Shop/MemberAddModal/`
- **Convex**: `convex/invite/`

## 主な機能

- スタッフ招待（マジックリンク発行）
- マネージャー招待（Clerk認証必要）
- 招待の承諾
- 招待の再送信
- 招待の取り消し

## 招待フロー

### スタッフ招待（一般スタッフ）
```
管理者: MemberAddModalで招待
    ↓
staffsテーブルにpending状態で登録
    ↓
magicLinkToken発行
    ↓
スタッフ: /invite/:token にアクセス
    ↓
status: "active" に変更
```

### マネージャー招待
```
管理者: MemberAddModalで招待
    ↓
staffsテーブルにpending状態で登録
    ↓
inviteToken発行
    ↓
マネージャー: Clerk認証後、/invite/:token にアクセス
    ↓
userId紐付け、status: "active" に変更
```

## データモデル

招待関連フィールド（staffsテーブル内）:

```typescript
{
  // マジックリンク（一般スタッフ用）
  magicLinkToken: string?,
  magicLinkExpiresAt: number?,

  // 招待トークン（マネージャー用）
  inviteToken: string?,
  inviteExpiresAt: number?,

  // 招待メタ
  invitedBy: string?,  // 招待者authId
}
```

## 招待有効期限

- **有効期限**: 14日間
- **定数**: `INVITE_EXPIRY_DAYS = 14`

## 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| 招待承諾 | `/invite/:token` | 招待リンクのランディング |
| スタッフ招待モーダル | - | 店舗詳細画面から呼び出し |

## コンポーネント構成

```
Shop/
└── MemberAddModal/   # スタッフ招待モーダル

pages/
└── Invite/           # 招待承諾ページ
```

## API

### Queries
- `invite.queries.getByToken` - トークンで招待情報取得
- `invite.queries.validateToken` - トークン有効性検証

### Mutations
- `invite.mutations.create` - 招待作成
- `invite.mutations.accept` - 招待承諾
- `invite.mutations.resend` - 招待再送信
- `invite.mutations.revoke` - 招待取り消し
