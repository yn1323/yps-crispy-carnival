# ユーザー管理

## 概要

管理者ユーザー（店舗オーナー・マネージャー）の認証とプロフィール管理を行う機能。認証はClerkを使用。

## 関連ファイル

- **Routes**: `src/routes/_auth/mypage.tsx`, `src/routes/_auth/settings/`
- **Pages**: `src/components/pages/MyPage/`, `src/components/pages/Settings/`
- **Features**: `src/components/features/User/`, `src/components/features/Setting/`
- **Convex**: `convex/user/`

## 主な機能

- Clerk認証（OAuth、Email Magic Link）
- ユーザー登録
- プロフィール編集
- 設定管理

## データモデル

```typescript
users = {
  name: string,         // ユーザー名（2-20文字）
  email: string,        // メールアドレス
  authId: string?,      // Clerk認証ID
  status: string,       // "pending" | "active"
  createdAt: number,    // 作成日時
  isDeleted: boolean?,  // 削除フラグ
}
```

## 認証フロー

```
Clerk認証
    ↓
authId取得
    ↓
users.authIdで検索
    ↓
┌─ 存在する場合 → ログイン完了
└─ 存在しない場合 → ユーザー登録画面へ
```

## 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| マイページ | `/mypage` | ユーザーダッシュボード |
| 設定 | `/settings` | ユーザー・店舗設定 |
| ユーザー登録 | `/welcome` | 初回ユーザー登録 |

## コンポーネント構成

```
User/
└── UserRegister/     # ユーザー登録フォーム

Setting/
├── UserSetting/      # ユーザー設定
│   ├── UserProfile/  # プロフィール編集
│   └── ShopSetting/  # 店舗設定
```

## API

### Queries
- `user.queries.getByAuthId` - authIdでユーザー取得
- `user.queries.getCurrent` - 現在のユーザー取得

### Mutations
- `user.mutations.create` - ユーザー作成
- `user.mutations.update` - ユーザー更新

## 状態管理

```typescript
// src/stores/user/index.ts
userAtom        // 現在のユーザー情報
```
