# プロダクト方向性 v2

## 変更の背景

### 課題
スタッフユーザーの**ログイン・アカウント登録が手間**であることが判明。
- アプリインストールの心理的ハードル
- アカウント作成の煩雑さ
- パスワード管理の負担

### 方針転換
**スタッフはアカウント不要、マジックリンクでシフト申請**
→ 店長がメアドを登録するだけで即運用開始できる軽量なサービスへ

---

## ユーザー構成

### Before（v1）

| ロール | 認証 | 説明 |
|--------|------|------|
| Owner | Clerk | 店舗オーナー（全権限） |
| Manager | Clerk | マネージャー（シフト・メンバー管理） |
| Staff | Clerk | スタッフ（シフト申請、勤怠打刻） |

### After（v2）

| ロール | 認証 | 説明 |
|--------|------|------|
| **管理者** | Clerk | 店舗オーナー・店長（区分けなし） |
| **スタッフ** | マジックリンク | 店舗ごとに独立、メアドで管理 |

### スタッフの特徴
- **アカウント登録不要**
- **店舗ごとに独立**（同じメアドでも別店舗なら別人扱い）
- **マジックリンクはシフト申請サイクルごとに発行**
- シフト確定時にリンク無効化

## 機能構成

### 削除する機能
- タイムカード機能
- 勤怠記録・集計機能
- GPS打刻
- Owner/Manager の役割区分

### 維持する機能
- 店舗管理（CRUD）
- スタッフ管理（招待・編集・退職処理）
- シフト申請・確定フロー
- メール通知
- レスポンシブUI

### 追加・拡張する機能

#### スタッフ情報の拡張
| 項目 | 説明 |
|------|------|
| email | メールアドレス（DB管理） |
| skills | スキル管理（ポジション × 習熟度） |
| maxWeeklyHours | 週あたり労働時間上限（社保対策） |
| workStyleNote | AIフレンドリーなメモ（既存） |

#### スキル管理の例
```
[
  { position: "ホール", level: "未経験" },
  { position: "キッチン", level: "一人前" }
]
```

#### 店舗設定の拡張
- 時間帯別の必要人員設定（ポジションごと）

### 将来機能（未確定）
- **ヘルプ要請機能**：直前キャンセル時、同時間帯に希望を出していたスタッフへ連絡

## シフト申請フロー

```
管理者：シフト申請依頼メール送信
    ↓
システム：マジックリンク発行（スタッフごと）
    ↓
スタッフ：リンクからシフト希望を入力
    ↓
管理者：AIでシフト作成 → 調整
    ↓
管理者：シフト確定ボタン
    ↓
システム：リンク無効化 + 確定通知メール送信
```

### マジックリンク仕様
- **有効期間**：シフト申請サイクル単位（確定まで有効）
- **発行タイミング**：申請依頼メール送信時
- **無効化タイミング**：シフト確定時

## データモデル

### 変更概要

| テーブル | 変更内容 |
|----------|----------|
| **users** | 管理者のみに（スタッフは使わない） |
| **shops** | useTimeCard削除、その他維持 |
| **shopUserBelongings** | → **staffs** にリネーム・構造変更 |

### staffs テーブル（旧 shopUserBelongings）

#### 削除するフィールド
- `userId`（usersテーブルと紐付けない）
- `role`（owner/manager区分けなし）

#### 維持するフィールド
- `shopId`
- `displayName`
- `status`（pending/active/resigned）
- `memo`
- `workStyleNote`
- `hourlyWage`
- `invitedBy`
- `resignedAt` / `resignationReason`

#### 追加するフィールド
- `email`（メールアドレス）
- `skills`（スキル配列）
- `maxWeeklyHours`（週あたり労働時間上限）

#### 変更するフィールド
- `magicLinkToken` / `magicLinkExpiresAt`（サイクルごとに更新）
- `maxWorkingHoursPerMonth` → `maxWeeklyHours`（月→週に変更）

## AIシフト作成

### 入力情報
- スタッフのシフト希望（時間帯）
- スタッフのスキル（ポジション × 習熟度）
- スタッフの週あたり労働時間上限
- スタッフのworkStyleNote（AIメモ）
- 時間帯別必要人員設定

### 制約条件の例
- 「週20時間を超えない」
- 「キッチン一人前を必ず1人配置」
- 「土日は学生優先」
