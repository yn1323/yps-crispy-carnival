# ユーザー情報アクセス制御

## 背景・課題

店舗詳細画面において、一般ユーザー（general）が他のスタッフの詳細情報（勤務時間、所属店舗一覧など）を閲覧できてしまう状態になっている。

### 現状の問題点

1. **Convex側のクエリに権限チェックがない**
   - `getUsersInShop`: 誰でも店舗スタッフ一覧を取得可能
   - `getUserById`: 誰でも任意ユーザーの詳細情報を取得可能
   - `getUserShops`: 誰でも任意ユーザーの所属店舗一覧を取得可能

2. **フロントエンドのみのUI制御**
   - 編集ボタンの表示制御は実装済みだが、APIを直接叩けば迂回可能

---

## 要件

### スタッフ一覧（店舗詳細のスタッフタブ）

| 役割 | 表示範囲 |
|------|----------|
| owner | 全員表示（pending, active, resigned） |
| manager | 全員表示（pending, active, resigned） |
| general | **在籍ユーザーのみ**（status: "active"） |

### スタッフ詳細ページ

| 役割 | 対象 | 閲覧可能な情報 |
|------|------|----------------|
| owner | 全員 | 全詳細情報 |
| manager | 全員 | 全詳細情報 |
| general | **自分** | 全詳細情報 |
| general | **他人** | 名前・役割のみ |

### APIレスポンス

- 権限がない場合は**バックエンド側で情報をフィルタリング**して返す
- フロントエンドの表示制御に依存しない

---

## 実装方針

### 1. ステータス定数の追加

`convex/constants.ts` に `SHOP_USER_STATUS` を追加：

```typescript
export const SHOP_USER_STATUS = ["pending", "active", "resigned"] as const;
export type ShopUserStatusType = (typeof SHOP_USER_STATUS)[number];
```

### 2. Convex API修正

#### `getUsersInShop`（スタッフ一覧）

**現在の引数:**
```typescript
args: { shopId: v.string() }
```

**変更後の引数:**
```typescript
args: {
  shopId: v.string(),
  authId: v.string()  // 追加: 呼び出し元ユーザーの認証ID
}
```

**ロジック:**
1. `authId` から呼び出し元ユーザーの役割を取得
2. owner/manager → 全ユーザーを返す
3. general → `status: "active"` のユーザーのみ返す

#### `getUserById`（ユーザー詳細）

**現在の引数:**
```typescript
args: { userId: v.string() }
```

**変更後の引数:**
```typescript
args: {
  userId: v.string(),
  authId: v.string(),  // 追加: 呼び出し元ユーザーの認証ID
  shopId: v.string()   // 追加: 現在の店舗ID（権限判定用）
}
```

**ロジック:**
1. `authId` から呼び出し元ユーザーの役割を取得
2. owner/manager → 全詳細情報を返す
3. general（自分）→ 全詳細情報を返す
4. general（他人）→ `{ _id, name }` のみ返す

#### `getUserShops`（所属店舗一覧）

**現在の引数:**
```typescript
args: { userId: v.string() }
```

**変更後の引数:**
```typescript
args: {
  userId: v.string(),
  authId: v.string(),  // 追加: 呼び出し元ユーザーの認証ID
  shopId: v.string()   // 追加: 現在の店舗ID（権限判定用）
}
```

**ロジック:**
1. `authId` から呼び出し元ユーザーの役割を取得
2. owner/manager → 全情報を返す
3. general（自分）→ 全情報を返す
4. general（他人）→ 空配列 `[]` を返す

### 3. フロントエンド修正

各ページコンポーネントでクエリに `authId` を渡すように修正。

## APIレスポンス型

### `getUserById` レスポンス

**全詳細情報（owner/manager/自分）:**
```typescript
{
  _id: Id<"users">;
  name: string;
  authId?: string;
  status: string;
  createdAt: number;
  isDeleted?: boolean;
}
```

**制限情報（general/他人）:**
```typescript
{
  _id: Id<"users">;
  name: string;
  role: string;        // 店舗内での役割
  isLimitedView: true; // 制限ビューフラグ
}
```

### `getUserShops` レスポンス

**全情報（owner/manager/自分）:**
```typescript
Array<{
  _id: Id<"shops">;
  shopName: string;
  role: string;
  // ...その他の店舗情報
}>
```

**制限情報（general/他人）:**
```typescript
[] // 空配列
```
