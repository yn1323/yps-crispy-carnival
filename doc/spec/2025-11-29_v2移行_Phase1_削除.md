# v2移行 Phase 1: 削除・クリーンアップ

## 概要

タイムカード機能、勤怠機能、role関連コードを削除し、コードベースをクリーンアップする。

## 前提条件

- 本番データへの影響なし（機能削除のみ）
- テストが通ることを確認しながら進める

### 2. コンポーネント削除

### 3. スキーマ修正（shops）

`convex/schema.ts` から `useTimeCard` フィールドを削除：

```typescript
// Before
const shops = defineTable({
  shopName: v.string(),
  openTime: v.string(),
  closeTime: v.string(),
  timeUnit: v.number(),
  submitFrequency: v.string(),
  avatar: v.optional(v.string()),
  useTimeCard: v.boolean(),  // ← 削除
  description: v.optional(v.string()),
  createdBy: v.string(),
  createdAt: v.number(),
  isDeleted: v.boolean(),
});

// After
const shops = defineTable({
  shopName: v.string(),
  openTime: v.string(),
  closeTime: v.string(),
  timeUnit: v.number(),
  submitFrequency: v.string(),
  avatar: v.optional(v.string()),
  description: v.optional(v.string()),
  createdBy: v.string(),
  createdAt: v.number(),
  isDeleted: v.boolean(),
});
```

### 4. 定数削除（role関連）

`convex/constants.ts` から以下を削除：

```typescript
// 削除対象
export const SHOP_USER_ROLE = ["owner", "manager", "general"] as const;
export type ShopUserRoleType = (typeof SHOP_USER_ROLE)[number];
```

### 8. その他のクリーンアップ

以下のコマンドで `useTimeCard` の残存を確認：

