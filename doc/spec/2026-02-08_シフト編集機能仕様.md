# シフト編集機能 仕様書

**作成日**: 2026-02-08
**目的**: リファクタリング時の機能漏れ防止。シフト編集に関する全機能・操作・振る舞いを網羅的に記録する。
**方針**: コードを正とする。実装詳細（関数シグネチャ、コンポーネント構造）ではなく、ユーザー視点の機能・操作を記述する。

---

## 1. 概要

管理者がスタッフのシフトを視覚的に編集するための機能。
日付ごとに時間軸上でポジション（ホール・キッチン等）を配置・消去・リサイズできる。

### バリアント

| バリアント | 対象 | 編集方式 |
|-----------|------|---------|
| PC版 | デスクトップ | ドラッグ操作（ペイント/消去/リサイズ） |
| SP版 | モバイル | カードタップ → BottomSheet でSelect式編集 |

### ビューモード

| モード | 表示 | 編集可否 |
|--------|------|---------|
| 日毎ビュー | 1日分のシフトを時間軸で表示 | 可 |
| 俯瞰ビュー | 期間全体を一覧表示 | PC: 不可（閲覧のみ）/ SP: スタッフ追加のみ可 |

両ビューは常時マウント（`display:none`切替）され、UI状態が保持される。

## 2. 入口の型定義

### 2.1 共通Props（PC/SP同一）

```typescript
type ShiftViewSwitcherBaseProps = {
  shopId: string;                    // 店舗ID
  staffs: StaffType[];               // スタッフ一覧
  positions: PositionType[];         // ポジション種別一覧
  initialShifts: ShiftData[];        // 初期シフトデータ
  dates: string[];                   // 対象日付配列 ["2026-01-21", ...]
  timeRange: TimeRange;              // 時間軸設定
  holidays?: string[];               // 祝日配列
  isReadOnly?: boolean;              // 読み取り専用モード
  currentStaffId?: string;           // ハイライト対象スタッフID
};
```

### 2.2 PC版追加Props

```typescript
// PC版のみ追加
allShifts?: ShiftData[];             // 月合計用（対象期間外を含む全シフト）
requiredStaffing?: RequiredStaffingData[];  // 必要人員データ（充足度計算用）
```

### 2.3 ドメインモデル

```typescript
type ShiftData = {
  id: string;
  staffId: string;
  staffName: string;
  date: string;                      // "2026-01-28"
  requestedTime: {                   // スタッフの希望時間（null = 希望なし）
    start: string;                   // "09:00"
    end: string;                     // "17:00"
  } | null;
  positions: PositionSegment[];      // 確定済みポジションセグメント群
};

type PositionSegment = {
  id: string;
  positionId: string;
  positionName: string;              // "ホール", "キッチン" 等
  color: string;                     // 表示色（例: "#3b82f6"）
  start: string;                     // "09:00"
  end: string;                       // "17:00"
};

type PositionType = {
  id: string;
  name: string;                      // "ホール", "キッチン", "レジ", "休憩"
  color: string;
};

type StaffType = {
  id: string;
  name: string;
  isSubmitted: boolean;              // シフト希望提出済みか
};

type TimeRange = {
  start: number;                     // 開始時（例: 9）
  end: number;                       // 終了時（例: 22）
  unit: number;                      // スナップ単位（分）（例: 15, 30, 60）
};

type RequiredStaffingData = {
  dayOfWeek: number;                 // 0=日, 1=月, ..., 6=土
  slots: {
    hour: number;                    // 0-23
    position: string;
    requiredCount: number;
  }[];
};
```

### 2.4 モード型

```typescript
type ViewMode = "daily" | "overview";
type ToolMode = "select" | "assign" | "erase";        // PCのみ
type SortMode = "default" | "request" | "startTime";
```

### 2.5 ビジネス定数

| 定数 | 値 | 用途 |
|------|-----|------|
| Undo/Redo 履歴上限 | 50件 | 超過時は最古の履歴を破棄 |
| 1時間あたりの幅 | 120px | PC日毎ビューの時間軸固定幅 |
| 時間軸左右パディング | 30px | PC日毎ビューの時間軸余白 |
| リサイズ端判定閾値 | 10px | カーソルがバー端と判定される距離 |
| 自動スクロール発動距離 | 50px | テーブル端からの距離 |
| 自動スクロール速度 | 2〜10px/frame | 端に近いほど速い |
| スワイプ閾値 | 50px | SP版の日付切替スワイプ判定距離 |

### 2.6 充足度カラー（6段階）

| 充足率 | 色 |
|--------|-----|
| 0-20% | 赤 `hsl(0, 85%, 70%)` |
| 21-40% | オレンジ `hsl(30, 80%, 75%)` |
| 41-60% | 黄 `hsl(50, 80%, 75%)` |
| 61-80% | 黄緑 `hsl(80, 80%, 75%)` |
| 81-100% | 緑 `hsl(120, 80%, 75%)` |
| 101%以上 | 青 `hsl(210, 80%, 75%)` |

## 4. 操作・イベント一覧

### 4.1 PC 日毎ビュー

#### ツールバー（PositionToolbar）

| 操作対象 | 操作 | 結果 |
|---------|------|------|
| Undoボタン | クリック | 直前の操作を取り消す。操作不可時は disabled |
| Redoボタン | クリック | 取り消した操作をやり直す。操作不可時は disabled |
| 選択モードボタン | クリック | ツールモードを `select` に変更 |
| 配置モードボタン | クリック | ツールモードを `assign` に変更 |
| 消去モードボタン | クリック | ツールモードを `erase` に変更 |
| ポジションボタン群 | クリック | 選択中ポジションを変更。選択モード時は全ボタン disabled |

#### 日付タブ（DateTabs）

| 操作対象 | 操作 | 結果 |
|---------|------|------|
| 日付タブ | クリック | 選択日を変更。現在のソートモードで並び順を再計算 |

#### ソートメニュー（SortMenu）

| 操作対象 | 操作 | 結果 |
|---------|------|------|
| 「デフォルト」 | クリック | 元の配列順でソート |
| 「シフト希望順」 | クリック | 希望開始時刻 → 希望終了時刻 → 名前順でソート |
| 「出勤順」 | クリック | 最早ポジション開始時刻 → 終了時刻 → 名前順でソート |

#### シフト行（スタッフごとのドラッグ領域）

| 操作対象 | 操作 | 結果 |
|---------|------|------|
| シフト行 | 配置モードでドラッグ | ドラッグ範囲に選択中ポジションを配置。最小距離: timeRange.unit分。重複は上書き |

| 隣接バー境界 | 全モードでドラッグ | 連動リサイズ。前後バーの境界を同時移動。unit未満になったバーは自動削除 |
| シフト行 | 選択モードでドラッグ（バー端以外） | テーブルの横スクロール |
| スタッフ名 | クリック | スタッフ編集モーダルを表示 |

| 操作対象 | 操作 | 結果 |

## 5. ユースケース

### UC1: ポジション配置（PC）

1. ツールバーで「配置」モードを選択
2. 配置したいポジション（例: ホール）を選択
3. スタッフの行でドラッグ（開始位置 → 終了位置）
4. ドラッグ中、選択中ポジション色の半透明プレビューが表示される
5. ドラッグ距離が `timeRange.unit` 以上の場合、マウスアップでポジションが配置される
6. 既存ポジションと重複する範囲は上書きされる
7. 正規化が実行される（マージ + 休憩自動挿入）
8. Undo履歴に追加される

### UC2: ポジション配置（SP）

1. スタッフカードをタップ → 編集BottomSheetが開く
2. 「ポジション追加」セクションでポジションを選択
3. 開始時刻・終了時刻をSelectで設定
4. 「+」ボタンをタップ
5. ポジションが配置される（既存と重複する範囲は上書き）
6. 正規化が実行される
7. Undo履歴に追加される

### UC3: ポジション消去（PC）

1. ツールバーで「消去」モードを選択
2. ポジションが存在する位置でドラッグ開始
3. ドラッグ中、赤い点線ボーダーのプレビューが表示される
4. マウスアップでドラッグ範囲内のポジションが削除される
5. 部分的に重なるセグメントは分割して残る
6. 正規化 + Undo履歴追加

### UC4: ポジション削除（SP）

1. スタッフカードをタップ → 編集BottomSheet
2. 削除したいセグメントの「×」ボタンをタップ
3. セグメントが削除される → 正規化
4. または「全削除」ボタンで全ポジションを一括削除

### UC5: リサイズ（PC）

#### 単体リサイズ
1. ポジションバーの端（左端 or 右端）にカーソルを置く（カーソルが `ew-resize` に変化）
2. ドラッグでバーの開始/終了時刻を変更
3. `timeRange.unit` 単位にスナップ
4. 最小幅は `timeRange.unit`
5. 他のポジションと重なった場合、他方を上書き

#### 連動リサイズ
1. 隣接する2つのバーの境界にカーソルを置く
2. ドラッグで境界を移動（前バーの終了と次バーの開始を同時変更）
3. `timeRange.unit` 未満になったバーは自動削除

### UC6: Undo/Redo

1. PC: Ctrl+Z（Undo）/ Ctrl+Y or Cmd+Shift+Z（Redo）
2. SP: ヘッダーのUndo/Redoボタンをタップ
3. 編集操作ごとに履歴が追加される（最大50件）
4. 新しい操作を行うとRedo履歴はクリアされる
5. PC俯瞰ビュー時はキーボードショートカット無効

### UC7: ビュー切替（日毎 ↔ 俯瞰）

1. SegmentGroupで切替
2. 両ビューは常時マウントされており、切替時にUI状態（ツールモード、ソート、スクロール位置等）が保持される
3. シフトデータは両ビューで共有（日毎で編集 → 俯瞰に即反映）

### UC8: 俯瞰→日毎ナビゲーション

1. 俯瞰ビューで日付をクリック/タップ
2. 日毎ビューに自動切替
3. クリックした日付が選択される

### UC9: 未提出スタッフへの初回配置

1. ShiftDataが存在しないスタッフの行に配置操作を行う
2. 自動的に新しいShiftData（空のpositions）が作成される
3. 通常の配置フローが継続する

## 6. 表示仕様

### 6.1 スタッフ行の状態表示

| 状態 | PC日毎の表示 | SP日毎の表示 |
|------|-------------|-------------|
| 提出済み・希望あり | 希望時間バー（グレー破線） + ポジションバー | カード: 「希望 HH:MM-HH:MM」+ ミニバー |
| 提出済み・希望なし | 「希望なし」テキスト（gray.400） | カード: 「シフト希望なし」テキスト |
| 未提出 | 「未提出」テキスト（orange.400） | カード: 背景 `red.50` + ⚠アイコン + 「未提出」 |
| ポジション未割当 | （希望時間バーのみ表示） | カード: 「ポジション未割当」テキスト |
| ハイライト対象 | 行背景 `blue.50` + 左ボーダー `3px solid blue` | カード: 左ボーダー `3px solid blue.400` + 背景 `blue.50` |

### 6.2 カーソル変化（PCのみ）

| 条件 | カーソル |
|------|---------|
| ポジションバー端（全モード共通） | `ew-resize` |
| 選択モード・バー端以外 | `grab` |
| 選択モード・横スクロール中 | `grabbing` |
| 配置モード・ポジション選択済み | `crosshair` |
| 消去モード・ポジション存在する位置 | `crosshair` |
| その他 | `default` |

### 6.3 ドラッグプレビュー（PCのみ）

| モード | 表示 |
|--------|------|
| 配置（paint） | 選択中ポジション色の半透明ボックス（opacity: 0.3） |
| 消去（erase） | 赤の点線ボーダーボックス（opacity: 0.15） |
| リサイズ | プレビューなし（バー自体がリアルタイム更新） |

### 6.4 俯瞰ビューの日付色分け

| 条件 | 背景色 | 文字色 |
|------|--------|--------|
| 日曜・祝日 | `red.50` | `red.600` |

## 7. 自動処理ルール

### 7.1 正規化パイプライン

全てのシフト変更操作（配置・消去・リサイズ・時刻変更・削除）の後に自動実行される。

**処理順序**:
1. 既存の休憩セグメントを全て除去
2. 同じpositionIdの隣接/重複セグメントをマージ（結合）
3. ポジション間のギャップに休憩セグメントを自動挿入

**休憩自動挿入のルール**:
- `name === "休憩"` のポジション種別が存在する場合のみ
- バー間のギャップのみ。先頭・末尾のギャップには挿入しない

### 7.2 時間範囲制限

全てのドラッグ操作・時刻選択において:
- `timeRange.start` より前には設定できない
- `timeRange.end` より後には設定できない
- `timeRange.unit` 単位にスナップ

### 7.3 未提出スタッフの初回配置

- ShiftDataが存在しないスタッフに配置操作を行うと、空のShiftDataが自動作成される
- `requestedTime: null`、`positions: []` で初期化

### 7.4 ソートルール

| モード | ソートキー | 末尾グループ |
|--------|----------|-------------|
| `default` | 元の配列順 | なし |
| `request` | 希望開始時刻 → 希望終了時刻 → 名前（ja） | 未提出 → 希望なし |
| `startTime` | 最早ポジション開始時刻 → 終了時刻 → 名前（ja） | ポジション未割当 |

**共通ルール**:
- 未提出スタッフは常に末尾
- 日本語名前比較は `localeCompare("ja")` を使用
- ソートは実行時点のスナップショット（編集で並び順はリアルタイムに変わらない）
- ソート状態は日毎/俯瞰ビュー間で共有
