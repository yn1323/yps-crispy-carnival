# シフト管理機能 仕様書

**作成日**: 2026-01-03
**対象機能**: シフト募集・申請・確定機能
**関連コンポーネント**: Shift関連コンポーネント、Convex shift関連ファイル

---

## 概要

店舗管理者がシフト募集を作成し、スタッフがマジックリンク経由でシフト希望を申請、管理者が調整・確定するまでの一連のフロー。

### 目的
- スタッフのアカウント登録不要でシフト申請を実現（マジックリンク方式）
- 管理者の手間を最小限にしたシフト管理
- 時間帯別必要人員の可視化による適切なシフト配置

### 前提条件
- v2移行 Phase 1〜3 完了済み
- staffsテーブルが正常に動作している

## 業務フロー

### 全体フロー図

```
【管理者】          【スタッフ】          【システム】
    │                   │                   │
    ├─ シフト募集期間の設定                  │
    ├─ シフト募集開始 ──────────────────────→│
    │                   │                   ├─ シフト募集開始
    │                   │                   ├─ 申請用マジックリンク発行
    │                   │                   ├─ メール送信（Resend）
    │                   │←──────────────────┤
    │                   ├─ 募集期間のシフトを入力
    │                   ├─ 申請 ───────────→│
    │                   │                   │
    ├─ 募集終了（手動 or 自動）              │
    │                   │                   ├─ AIでシフト作成（将来）
    ├─ シフト案確認 ←───────────────────────┤
    ├─ シフト手動調整                        │
    ├─ シフト確定 ─────────────────────────→│
    │                   │                   ├─ 決定内容をメールで送信
    │                   │←──────────────────┤
    │                   ├─ 決定シフトの閲覧   │
    │                   │                   │
```

---

### フロー1: シフト募集作成（管理者）

```
1. 管理者がシフト管理画面を開く
2. 「新規募集作成」ボタン押下
3. 入力項目を入力
   - シフト対象期間（開始日〜終了日）
   - 申請締切日
4. 「募集開始」ボタン押下
5. システムが全スタッフにマジックリンク発行
6. Resendでメール送信
```

**入力項目:**
| 項目 | 必須 | 説明 |
|------|------|------|
| シフト対象期間（開始） | Yes | 実際に働く期間の開始日 |
| シフト対象期間（終了） | Yes | 実際に働く期間の終了日 |
| 申請締切日 | Yes | スタッフが希望を出せる期限 |

---

### フロー2: シフト希望申請（スタッフ）

```
1. メールのマジックリンクをクリック
2. シフト申請画面表示（ログイン不要）
   - 募集期間の日付一覧
   - テンプレート適用ボタン（設定済みの場合）
3. 日付ごとにシフト希望を入力
   - 一括登録: テンプレートを適用
   - 日ごと登録: 個別に時間を設定
4. 「申請する」ボタン押下
5. 申請完了
```

**入力項目:**
| 項目 | 必須 | 説明 |
|------|------|------|
| 日付 | Yes | 出勤希望日 |
| 開始時間 | Yes | 出勤希望開始時刻（自由入力） |
| 終了時間 | Yes | 出勤希望終了時刻（自由入力） |
| 備考 | No | 特記事項 |

---

### フロー3: 募集終了（管理者/システム）

```
【自動締め切り】
- 申請締切日を過ぎると自動でステータス変更

【手動締め切り】
1. 管理者が募集詳細画面を開く
2. 「募集締め切り」ボタン押下
3. 確認ダイアログで確定
```

---

### フロー4: シフト確定（管理者）

```
1. 管理者が募集詳細画面を開く
2. スタッフの申請状況を確認
   - 申請済み/未申請の一覧
   - 時間帯別の人員充足状況
3. シフト調整画面でシフトを編集
   - 希望をベースに手動調整
   - 必要人員との過不足を確認
4. 「シフト確定」ボタン押下
5. システムが全スタッフにメール送信
6. マジックリンク無効化
```

---

### フロー5: 確定シフト閲覧（スタッフ）

```
1. 確定通知メールを受信
2. メールのリンクをクリック
3. 確定シフト閲覧画面表示
   - 自分の確定シフト一覧
   - 日付、時間、ポジション
```

## シフトテンプレート機能

### 概要
スタッフが自分の「いつものシフト」を登録しておき、申請時にワンタップで適用できる機能。

### テンプレートの種類

| パターン | 例 | 説明 |
|---------|-----|------|
| **曜日+時間** | 月曜 9:00-17:00 | 特定曜日の固定シフト |
| **時間だけ** | 9:00-17:00 | 曜日に関係なく使う基本時間 |

### 登録・編集
- **誰が**: スタッフ自身がマジックリンク経由で登録・編集
- **タイミング**: シフト申請画面からアクセス可能

### 申請時の適用方法

| 方法 | 説明 |
|------|------|
| **一括登録** | テンプレートを全日程に一括適用 |
| **日ごと登録** | 1日ずつ個別に時間を設定 |

## 必要人員設定

### 概要
管理者が時間帯別の必要人員をあらかじめ設定しておく機能。シフト調整時に過不足を確認できる。

### 設定粒度
- **時間単位**: 1時間ごと
- **曜日**: 日〜土（0-6）
- **ポジション**: ホール、キッチンなど（店舗ごとにカスタム可能）

### 設定例
| 曜日 | 時間 | ホール | キッチン |
|------|------|--------|----------|
| 土曜 | 11:00 | 2人 | 1人 |
| 土曜 | 12:00 | 3人 | 2人 |
| 土曜 | 13:00 | 3人 | 2人 |
| 土曜 | 14:00 | 2人 | 1人 |

## データモデル

### 1. shiftRecruitments（シフト募集）

| フィールド名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| shopId | Id<"shops"> | Yes | 店舗ID |
| startDate | string | Yes | シフト対象期間（開始）"2025-12-01" |
| endDate | string | Yes | シフト対象期間（終了）"2025-12-07" |
| deadline | string | Yes | 申請締切日 "2025-11-25" |
| status | string | Yes | "open" \| "closed" \| "confirmed" |
| createdAt | number | Yes | 作成日時 |
| closedAt | number | No | 締め切り日時 |
| confirmedAt | number | No | 確定日時 |

**インデックス:**
- `by_shop`: shopIdでの検索用
- `by_shop_and_status`: shopId + statusでの検索用

**status:**
- `open`: 募集中（スタッフが申請可能）
- `closed`: 募集締め切り（管理者が調整中）
- `confirmed`: シフト確定済み

---

### 2. shiftRequests（シフト希望）

| フィールド名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| recruitmentId | Id<"shiftRecruitments"> | Yes | 募集ID |
| staffId | Id<"staffs"> | Yes | スタッフID |
| date | string | Yes | 希望日 "2025-12-01" |
| startTime | string | Yes | 開始時間 "09:00" |
| endTime | string | Yes | 終了時間 "17:00" |
| note | string | No | 備考 |
| createdAt | number | Yes | 作成日時 |

**インデックス:**
- `by_recruitment`: recruitmentIdでの検索用
- `by_staff`: staffIdでの検索用
- `by_recruitment_and_staff`: recruitmentId + staffIdでの検索用

---

### 3. confirmedShifts（確定シフト）

| フィールド名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| recruitmentId | Id<"shiftRecruitments"> | Yes | 募集ID |
| staffId | Id<"staffs"> | Yes | スタッフID |
| date | string | Yes | 確定日 "2025-12-01" |
| startTime | string | Yes | 開始時間 "09:00" |
| endTime | string | Yes | 終了時間 "17:00" |
| position | string | No | 担当ポジション |
| createdAt | number | Yes | 作成日時 |

**インデックス:**
- `by_recruitment`: recruitmentIdでの検索用

### 4. requiredStaffing（時間帯別必要人員）

## マジックリンク仕様

### URLフォーマット
```
{BASE_URL}/shift/{TOKEN}
```

### トークン仕様
- 形式: UUID v4（36文字）
- 一意性: staffsテーブルのmagicLinkTokenでユニーク
- 保存先: staffs.magicLinkToken

### 有効期限
- 期間: シフト募集ごとに設定（申請締切日まで）
- 判定: 募集のstatusがopenかつ締切日前
- 無効化タイミング: シフト確定時

### セキュリティ
- トークンは募集ごとに再生成可能
- 確定後は自動無効化
- 推測困難なUUID v4を使用

## 技術仕様

### メール送信
- **サービス**: Resend
- **実装**: Convex Actions
- **送信タイミング**:
  - 募集開始時: 全スタッフにマジックリンク付きメール
  - シフト確定時: 全スタッフに確定シフト通知

### 既存機能との連携
- **スタッフ管理**: staffsテーブルのskills（スキル情報）を参照
- **店舗設定**: shopsテーブルの営業時間を参照（将来）
