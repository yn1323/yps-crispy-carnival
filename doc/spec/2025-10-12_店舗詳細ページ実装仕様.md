# 店舗詳細ページ（ShopDetail）実装仕様

## 概要

店舗詳細ページの実装仕様。新設計方針（/routes → /pages → /features）に従った最初の本格実装として、各層の責務を明確化した構成となっている。

**実装日**: 2025-10-12

## 機能概要

店舗の詳細情報と所属スタッフ一覧を表示するページ。権限（owner/manager/staff）に応じて編集ボタンの表示を制御する。

## アーキテクチャ設計

### 3層構造の責務分離

```
/routes/_auth/shops/$id/index.tsx (ルーティング層)
    ↓ URLパラメータ取得
/pages/Shops/DetailPage/index.tsx (ページ層)
    ↓ データQuery + 条件分岐
/features/Shop/ShopDetail/index.tsx (Feature層)
    ↓ UI描画
```

#### 1. ルーティング層 (`/routes/_auth/shops/$id/index.tsx`)

**責務:**
- URLパラメータから`id`を取得
- `<Animation>`で`ShopsDetailPage`を呼び出し
- ルーティング以外のロジックは持たない

**実装状況:**
- ✅ 既に作成済み（修正不要）

```tsx
import { createFileRoute } from '@tanstack/react-router'
import { ShopsDetailPage } from '@/components/pages/Shops/DetailPage'
import { Animation } from '@/components/ui/Animation'

export const Route = createFileRoute('/_auth/shops/$id/')({
  component: () => {
    const { id } = Route.useParams()
    return (
      <Animation>
        <ShopsDetailPage id={id} />
      </Animation>
    )
  },
})
```

#### 2. ページ層 (`/pages/Shops/DetailPage/index.tsx`)

**責務:**
- URLパラメータから`shopId`を受け取る
- 初回描画で必要な全データをQuery:
  - `api.shop.getShopById` - 店舗基本情報
  - `api.shop.getUsersInShop` - 所属スタッフ一覧
  - `api.shop.getUserRoleInShop` - 現在のユーザーの権限
- ローディング・エラー・空状態の分岐処理
- Featuresコンポーネントの組み立て

**実装状況:**
- 🔧 修正予定（条件分岐ロジックを追加）

**状態分岐ロジック:**

```tsx
// ローディング状態
if (!shop || !users || userRole === undefined) {
  return <ShopDetailLoading />
}

// 店舗が見つからない
if (shop === null) {
  return <ShopDetailNotFound />
}

// エラー状態（必要に応じて）
// Convexのquery失敗時の扱いは要確認

// 通常表示
return <ShopDetail shop={shop} users={users} userRole={userRole} />
```

#### 3. Feature層 (`/features/Shop/ShopDetail/index.tsx`)

**責務:**
- 店舗詳細情報の表示（カード形式）
- 所属スタッフ一覧の表示
- 編集ボタン（owner/manager権限時のみ表示）
- 状態別コンポーネントの提供

**実装状況:**
- 🆕 新規実装予定

**Exportするコンポーネント:**

```tsx
// 通常表示
export const ShopDetail = ({ shop, users, userRole }) => { /* ... */ }

// ローディング状態
export const ShopDetailLoading = () => { /* ... */ }

// エラー状態
export const ShopDetailError = ({ error }) => { /* ... */ }

// 店舗が見つからない状態
export const ShopDetailNotFound = () => { /* ... */ }
```

## UI設計

### 基本情報カード

**表示項目:**

1. **店舗名（大見出し） + 編集ボタン**（権限あれば）
   - 見出しレベル: `<Heading>`
   - 編集ボタン: `<Button>` (owner/manager時のみ表示)

2. **営業時間**
   - 表示形式: `09:00 〜 18:00`
   - データ: `shop.openingTime` / `shop.closingTime`

3. **シフト提出頻度**
   - 表示形式: `週次` / `月次`
   - データ: `shop.shiftSubmissionFrequency`

4. **シフト時間単位**
   - 表示形式: `30分` / `60分`
   - データ: `shop.shiftTimeUnit`

5. **タイムカード利用**
   - 表示形式: `Badge`（利用中 / 未利用）
   - データ: `shop.useTimeCard`

6. **説明文**（あれば）
   - 表示形式: `<Text>`
   - データ: `shop.description`

### スタッフ一覧セクション

**構成:**

1. **セクション見出し** - 「所属スタッフ」
2. **スタッフカード** - 名前 + ロールBadge
3. **スタッフ追加ボタン**（権限あれば） ← 今回は表示のみ

**スタッフカード表示:**
- 名前: `user.name`
- ロール: `Badge` (owner/manager/staff)
- Badge colorPalette:
  - owner: `purple`
  - manager: `teal`
  - staff: `gray`

## 技術仕様

### 使用API

```tsx
// 店舗情報取得
const shop = useQuery(api.shop.getShopById, { shopId })

// スタッフ一覧取得
const users = useQuery(api.shop.getUsersInShop, { shopId })

// 現在のユーザー権限取得
const userRole = useQuery(api.shop.getUserRoleInShop, { shopId })
```

### 権限制御

**権限レベル:**
- `owner` - 店舗オーナー（全権限）
- `manager` - 店舗マネージャー（編集権限あり）
- `staff` - スタッフ（閲覧のみ）

**表示制御:**
```tsx
// 編集ボタンの表示条件
const canEdit = userRole === 'owner' || userRole === 'manager'

{canEdit && <Button>編集</Button>}
```

## データフロー

```
1. ユーザーが /shops/:id にアクセス
   ↓
2. /routes層がURLパラメータ取得
   ↓
3. /pages層が3つのQueryを実行:
   - getShopById (店舗情報)
   - getUsersInShop (スタッフ一覧)
   - getUserRoleInShop (権限)
   ↓
4. /pages層が条件分岐:
   - ローディング中 → ShopDetailLoading
   - 店舗なし → ShopDetailNotFound
   - エラー → ShopDetailError
   - 通常 → ShopDetail
   ↓
5. /features層が受け取ったデータを描画
```
