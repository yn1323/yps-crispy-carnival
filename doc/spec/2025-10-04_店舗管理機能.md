# 店舗管理機能 仕様書

**作成日**: 2025-10-04  
**対象機能**: 店舗CRUD機能（作成・取得・更新・削除）  
**関連コンポーネント**: CreateShop, ShopForm, Convex shop.ts

---

## 📋 概要

YPSにおける店舗管理機能の実装仕様。ユーザーが店舗を作成し、店舗に紐づくメンバーを管理できる機能を提供する。

### 目的
- 店舗の基本情報（営業時間、シフト設定など）を管理
- 店舗とユーザーの紐付け管理（owner/manager/staff）
- 権限に基づいた店舗情報の更新・削除

---

## 🗂️ データモデル

### 1. **shopsテーブル**

店舗の基本情報を管理するテーブル。

| フィールド名 | 型 | 必須 | デフォルト | 説明 |
|------------|-----|------|-----------|------|
| shopName | string | ✅ | - | 店舗名（2〜50文字） |
| openTime | string | ✅ | - | 開店時間（"09:00"形式） |
| closeTime | string | ✅ | - | 閉店時間（"22:00"形式） |
| timeUnit | number | ✅ | - | シフト時間単位（15, 30, 60など） |
| submitFrequency | string | ✅ | - | シフト提出頻度（"1w", "2w", "1m"） |
| avatar | string | ❌ | "" | 店舗アバター画像URL |
| useTimeCard | boolean | ✅ | true | タイムカード機能使用フラグ |
| description | string | ❌ | - | 店舗メモ（マネージャー向け） |
| createdBy | string | ✅ | - | 作成者のauthId（Clerk） |
| createdAt | number | ✅ | - | 作成日時（UNIX timestamp） |
| isDeleted | boolean | ✅ | false | 論理削除フラグ |

**インデックス:**
- `by_created_by`: createdByでの検索用

---

### 2. **shopUserBelongingsテーブル**

ユーザーと店舗の紐付け、および役割を管理するテーブル。

| フィールド名 | 型 | 必須 | デフォルト | 説明 |
|------------|-----|------|-----------|------|
| shopId | id("shops") | ✅ | - | 店舗ID（shopsテーブル参照） |
| userId | id("users") | ✅ | - | ユーザーID（usersテーブル参照） |
| role | string | ✅ | - | 役割（"owner" \| "manager" \| "staff"） |
| createdAt | number | ✅ | - | 紐付け作成日時（UNIX timestamp） |
| isDeleted | boolean | ✅ | false | 論理削除フラグ |

**インデックス:**
- `by_shop`: shopIdでの検索用
- `by_user`: userIdでの検索用
- `by_shop_and_user`: shopIdとuserIdの複合検索用

**役割（role）の種類:**
- **owner**: 店舗オーナー（店舗作成者、全権限）
- **manager**: マネージャー（店舗情報の更新可能、削除不可）
- **staff**: スタッフ（閲覧のみ）

## 🔧 API仕様（Convex Functions）

### Mutations（作成・更新・削除）

#### 1. **createShop**

店舗を作成し、作成者を自動的にownerとして紐付ける。

**引数:**
```typescript
{
  shopName: string;          // 店舗名（2〜50文字）
  openTime: string;          // 開店時間（"09:00"形式）
  closeTime: string;         // 閉店時間（"22:00"形式）
  timeUnit: number;          // シフト時間単位（15, 30, 60など）
  submitFrequency: string;   // シフト提出頻度（"1w", "2w", "1m"）
  useTimeCard: boolean;      // タイムカード使用フラグ
  description?: string;      // 店舗メモ（任意）
  authId: string;            // 作成者のauthId（Clerk認証ID）
}
```

**処理フロー:**
1. 入力値のバリデーション
   - shopName: 必須、2〜50文字
   - openTime/closeTime: 時間形式チェック（"HH:mm"）
   - timeUnit: SHOP_TIME_UNITに含まれるか
   - submitFrequency: SHOP_SUBMIT_FREQUENCYに含まれるか
   - authId: 必須、空文字チェック
2. usersテーブルからauthIdでuserIdを取得
3. shopsテーブルに店舗データを挿入
4. **shopUserBelongingsテーブルに作成者をownerとして自動追加**
5. 成功時、店舗IDと店舗名を返却

**戻り値:**
```typescript
{
  success: true;
  data: {
    shopId: Id<"shops">;
    shopName: string;
  }
}
```

**エラーケース:**
- `EMPTY_SHOP_NAME`: 店舗名が空
- `INVALID_TIME_FORMAT`: 時間形式が不正
- `INVALID_TIME_UNIT`: timeUnitが不正
- `INVALID_SUBMIT_FREQUENCY`: submitFrequencyが不正
- `USER_NOT_FOUND`: authIdに対応するユーザーが見つからない
- `CREATE_FAILED`: DB挿入失敗

---

#### 2. **updateShop**

店舗情報を更新する（owner/managerのみ実行可能）。

**引数:**
```typescript
{
  shopId: Id<"shops">;
  authId: string;            // 実行者のauthId
  shopName?: string;         // 更新する店舗名（任意）
  openTime?: string;         // 更新する開店時間（任意）
  closeTime?: string;        // 更新する閉店時間（任意）
  timeUnit?: number;         // 更新するシフト時間単位（任意）
  submitFrequency?: string;  // 更新するシフト提出頻度（任意）
  useTimeCard?: boolean;     // 更新するタイムカード使用フラグ（任意）
  description?: string;      // 更新する店舗メモ（任意）
}
```

**処理フロー:**
1. 店舗存在チェック（isDeleted = falseのみ）
2. usersテーブルからauthIdでuserIdを取得
3. **権限チェック: shopUserBelongingsでroleが"owner"または"manager"か確認**
4. 更新フィールドのバリデーション
5. shopsテーブルを更新

**戻り値:**
```typescript
Id<"shops">  // 更新された店舗ID
```

**エラーケース:**
- `SHOP_NOT_FOUND`: 店舗が見つからない
- `USER_NOT_FOUND`: ユーザーが見つからない
- `PERMISSION_DENIED`: 権限なし（staffまたは所属していない）
- `NO_FIELDS_TO_UPDATE`: 更新フィールドがない
- `UPDATE_FAILED`: DB更新失敗

---

#### 3. **deleteShop**

店舗を論理削除する（ownerのみ実行可能）。

**引数:**
```typescript
{
  shopId: Id<"shops">;
  authId: string;  // 実行者のauthId
}
```

**処理フロー:**
1. 店舗存在チェック
2. usersテーブルからauthIdでuserIdを取得
3. **権限チェック: shopUserBelongingsでroleが"owner"か確認**
4. shopsテーブルのisDeletedをtrueに設定

**戻り値:**
```typescript
{ success: true }
```

**エラーケース:**
- `SHOP_NOT_FOUND`: 店舗が見つからない
- `USER_NOT_FOUND`: ユーザーが見つからない
- `PERMISSION_DENIED`: ownerではない
- `DELETE_FAILED`: DB更新失敗

---

#### 4. **addUserToShop**

ユーザーを店舗に招待・追加する（owner/managerのみ実行可能）。

**引数:**
```typescript
{
  shopId: Id<"shops">;
  userId: Id<"users">;  // 追加するユーザーのID
  role: string;         // "manager" | "staff"（ownerは指定不可）
  authId: string;       // 実行者のauthId
}
```

**処理フロー:**
1. 店舗・ユーザー存在チェック
2. 実行者の権限チェック（owner/manager）
3. **roleが"owner"でないことを確認**（ownerは作成者のみ）
4. 既に所属していないかチェック
5. shopUserBelongingsに追加

**戻り値:**
```typescript
Id<"shopUserBelongings">  // 紐付けID
```

**エラーケース:**
- `INVALID_ROLE`: roleが"owner"または不正な値
- `ALREADY_BELONGS`: 既に所属している
- `PERMISSION_DENIED`: 権限なし

---

#### 5. **updateUserRole**

ユーザーの役割を変更する（owner/managerが実行可能）。

**引数:**
```typescript
{
  shopId: Id<"shops">;
  userId: Id<"users">;  // 役割変更対象のユーザーID
  newRole: string;      // 新しい役割（"manager" | "staff"）
  authId: string;       // 実行者のauthId
}
```

**処理フロー:**
1. 権限チェック（owner/manager）
2. **ownerからの降格は不可**（作成者は常にowner）
3. shopUserBelongingsのroleを更新

**戻り値:**
```typescript
Id<"shopUserBelongings">
```

**エラーケース:**
- `CANNOT_CHANGE_OWNER_ROLE`: ownerの役割は変更不可
- `PERMISSION_DENIED`: 権限なし

**新規作成ファイル:**

#### `schema.ts`
```typescript
import { z } from "zod";
import { SHOP_MIN_LENGTH, SHOP_MAX_LENGTH, SHOP_TIME_UNIT, SHOP_SUBMIT_FREQUENCY } from "@/src/constants/validations";
import { betweenLength, select, time } from "@/src/helpers/validation";

export const timeUnitOptions = SHOP_TIME_UNIT.map((v) => ({
  label: `${v}分単位`,
  value: String(v),
}));

export const submitFrequencyOptions = SHOP_SUBMIT_FREQUENCY.map((v) => ({
  label: v === "1w" ? "1週間ごと" : v === "2w" ? "2週間ごと" : "1ヶ月ごと",
  value: v,
}));

export const schema = z.object({
  shopName: z.string().min(1, "店舗名を入力してください").superRefine(betweenLength(SHOP_MIN_LENGTH, SHOP_MAX_LENGTH)),
  openTime: z.string().superRefine(time(15)),
  closeTime: z.string().superRefine(time(15)),
  timeUnit: z.string().superRefine(select({ options: timeUnitOptions, forceSelect: true })),
  submitFrequency: z.string().superRefine(select({ options: submitFrequencyOptions, forceSelect: true })),
  useTimeCard: z.boolean(),
  description: z.string().optional(),
});

export type SchemaType = z.infer<typeof schema>;
```

## 🔐 権限設計

| 操作 | owner | manager | staff |
|-----|-------|---------|-------|
| 店舗作成 | ✅ | ✅ | ✅ |
| 店舗情報閲覧 | ✅ | ✅ | ✅ |
| 店舗情報更新 | ✅ | ✅ | ❌ |
| 店舗削除 | ✅ | ❌ | ❌ |
| メンバー追加 | ✅ | ✅ | ❌ |
| メンバー役割変更 | ✅ | ✅ | ❌ |
| メンバー削除 | ✅ | ✅ | ❌ |

**注意事項:**
- ownerは店舗作成者のみ（後から変更不可）
- ownerの削除・役割変更は不可
- 店舗削除はownerのみ（論理削除）

## 📝 補足事項

### バリデーション
- 時間形式: "HH:mm"（例: "09:00", "22:30"）
- シフト時間単位: 1, 5, 10, 15, 30, 60のいずれか
- シフト提出頻度: "1w", "2w", "1m"のいずれか

### 論理削除
- shopsテーブル、shopUserBelongingsテーブルともに物理削除はしない
- isDeletedフラグで管理
- クエリ時は常に`isDeleted = false`でフィルタ

### エラーハンドリング
- ConvexErrorで統一
- エラーコードとメッセージを含める
- フロントエンドでトースト表示
