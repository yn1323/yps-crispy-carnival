# スタッフ管理機能 仕様書

## 概要
店舗オーナーがスタッフを管理するための機能群。スタッフの追加・編集・退職処理を行う。

## 用語定義

| 用語 | 定義 |
|------|------|
| マネージャー | `staffs.userId` が設定されているスタッフ（`users`テーブルに紐付いている） |
| スタッフ | `staffs.userId` が未設定のスタッフ（アプリにログインできない） |
| オーナー | 店舗を作成したユーザー（`shops.createdBy` と一致する `authId` を持つ） |

## 機能一覧

### 1. スタッフ追加（addStaff）

**実行権限**: オーナーのみ

**入力項目**:
| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| email | 必須 | string | メールアドレス（小文字変換、トリム処理） |
| displayName | 必須 | string | 表示名（トリム処理） |
| skills | 任意 | array | スキル配列（position, level） |
| maxWeeklyHours | 任意 | number | 週最大勤務時間 |

**バリデーション**:
- メールアドレス: 必須、同一店舗内で重複不可
- 表示名: 必須

**処理フロー**:
1. 店舗存在確認
2. オーナー権限確認
3. メールアドレス重複チェック
4. スタッフレコード作成（status: "active"）

**出力**: `{ success: true, staffId: Id<"staffs"> }`

---

### 2. スタッフ情報編集（updateStaffInfo）

**実行権限**: オーナーのみ

**編集可能項目**:
| 項目 | 型 | 説明 |
|------|-----|------|
| email | string | メールアドレス |
| displayName | string | 表示名 |
| skills | array | スキル配列 |
| maxWeeklyHours | number \| null | 週最大勤務時間（nullで削除） |
| memo | string | メモ |
| workStyleNote | string | 勤務スタイルノート |
| hourlyWage | number \| null | 時給（nullで削除） |

**バリデーション**:
- 更新フィールドが1つ以上必要
- スタッフが該当店舗に所属していること

**処理フロー**:
1. オーナー権限確認
2. スタッフ存在・所属確認
3. フィールド更新

**出力**: `{ success: true }`

---

### 3. スタッフ退職処理（resignStaff）

**実行権限**: オーナーのみ

**入力項目**:
| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| staffId | 必須 | Id<"staffs"> | 対象スタッフID |
| resignationReason | 任意 | string | 退職理由 |

**バリデーション**:
- スタッフが該当店舗に所属していること
- 既に退職済みでないこと
- **自分自身（オーナー）は退職処理不可**

**処理フロー**:
1. 店舗存在確認
2. オーナー権限確認
3. スタッフ存在・所属確認
4. 退職済みチェック
5. 自己退職チェック（オーナー自身の退職は不可）
6. ステータス更新（status: "resigned", resignedAt: 現在時刻）

**出力**: `{ success: true }`

**エラーケース**:
| コード | メッセージ |
|--------|-----------|
| STAFF_NOT_FOUND | スタッフが見つかりません |
| ALREADY_RESIGNED | このスタッフは既に退職済みです |
| CANNOT_RESIGN_SELF | 自分自身を退職処理することはできません |

## データモデル

### staffs テーブル

```typescript
{
  shopId: Id<"shops">,        // 所属店舗
  userId: Id<"users"> | null, // ユーザー紐付け（マネージャー判定用）
  email: string,              // メールアドレス
  displayName: string,        // 表示名
  status: "pending" | "active" | "resigned",
  skills: { position: string, level: string }[],
  maxWeeklyHours?: number,
  memo?: string,
  workStyleNote?: string,
  hourlyWage?: number,
  invitedBy: string,          // 招待者のauthId
  createdAt: number,
  resignedAt?: number,        // 退職日時
  resignationReason?: string, // 退職理由
  isDeleted: boolean,
}
```

### インデックス
- `by_shop`: shopId でスタッフ一覧取得
- `by_shop_email`: shopId + email でメール重複チェック
- `by_user`: userId で逆引き

## UI仕様

### スタッフ一覧表示
- マネージャー（`userId`あり）を先頭に表示
- マネージャーには紫色バッジを表示
- スタッフにはバッジなし

### スタッフ追加モーダル
- メールアドレス、表示名は必須
- スキル、週最大勤務時間は任意

### スタッフ編集画面
- 全項目編集可能
- メールアドレス編集時は重複チェック

### 退職処理
- 確認ダイアログ表示
- 退職理由の入力（任意）
- オーナー自身の退職ボタンは非活性
