# 店舗登録E2Eテスト設計

**作成日**: 2025-10-06
**対象機能**: 店舗登録機能のE2Eテスト
**関連ファイル**: e2e/scenarios/shop/register.test.ts

## 📋 概要

店舗登録機能（`/shops/new`）のE2Eテストの設計仕様。
バリデーションはフロントエンドで完結するため、E2Eではハッピーパスのみをテストする方針。

## 🎯 テスト対象

- **ルート**: `/shops/new`
- **コンポーネント**: `ShopRegister` (src/components/features/Shop/ShopRegister)
- **API**: `convex/shop.ts` の `createShop` mutation

## ✅ テストシナリオ

### ハッピーパス: 店舗を正常に登録できること

1. `/shops/new` にアクセス
2. 店舗名を入力（例: "E2Eテスト店舗"）
3. 店舗メモを入力（任意）（例: "E2Eテスト用の店舗です"）
4. その他の項目はデフォルト値を使用
   - 開店時間: 09:00
   - 閉店時間: 22:00
   - シフト提出頻度: 1週間ごと
   - シフト時間単位: 1分単位
   - タイムカード機能: ON
5. 「登録」ボタンをクリック
6. トースト「店舗登録が完了しました」が表示されることを確認
7. `/shops` に遷移したことを確認
8. 登録した店舗が一覧に表示されることを確認

## 🔧 実装方針

### 要素の取得方法
- **セマンティック要素優先**: `getByLabel()`, `getByRole()` を使用
- **data-testid 禁止**: プロジェクトポリシーに従い使用しない

### 認証
- 既存の `login.setup.ts` で作成される認証状態を利用
- テスト前に認証は完了している前提

### テスト範囲
- **ハッピーパスのみ**: バリデーションエラーのテストは不要
  - 理由: バリデーションはフロントエンドで完結しているため
- **エンドツーエンド**: 登録後に一覧画面での確認まで実施

## 📝 実装コード例

```typescript
import { expect, test } from "@playwright/test";

test.describe("店舗登録", () => {
  test.beforeEach(async ({ page }) => {
    await page.goto("/shops/new");
  });

  test("店舗を正常に登録できること", async ({ page }) => {
    const shopName = "E2Eテスト店舗";

    // 店舗名入力
    await page.getByLabel("店舗名").fill(shopName);

    // 店舗メモ入力（任意）
    await page
      .getByLabel("店舗メモ（マネージャー向け）")
      .fill("E2Eテスト用の店舗です");

    // 登録ボタンをクリック
    await page.getByRole("button", { name: "登録" }).click();

    // トースト確認
    await expect(page.getByText("店舗登録が完了しました")).toBeVisible();

    // /shops に遷移したことを確認
    await expect(page).toHaveURL("/shops");

    // 登録した店舗が一覧に表示されることを確認
    await expect(page.getByText(shopName)).toBeVisible();
  });
});
```

## 🚫 テスト対象外

### バリデーションエラーのテスト
以下のようなバリデーションエラーのテストは**実施しない**:
- 店舗名未入力
- 店舗名が2文字未満
- 店舗名が50文字超過
- 時間形式が不正

**理由**: フロントエンドのバリデーションで完結しており、E2Eで確認する必要がない
