**作成日**: 2026-01-28
**目的**: リファクタリング時の機能漏れ防止。現行コードの全機能・イベント・UI を網羅的に記録する。

## 1. 概要

管理者がスタッフのシフトを視覚的に編集するためのテーブルコンポーネント。
日付ごとに時間軸上でポジション（ホール・キッチン等）をドラッグ操作で配置・消去・リサイズできる。

## 2. 機能一覧

### 2.1 ツールモード（ToolMode）

3つのモードを切り替えて操作する。

| モード | 値 | 概要 | ドラッグ時の動作 | クリック時の動作 |
|--------|-----|------|------------------|------------------|
| 選択 | `select` | 閲覧・リサイズ専用 | ポジションバー端のリサイズ。端以外はテーブル横スクロール | シフトバークリックでポップオーバー表示 |
| 配置 | `assign` | ポジション配置 | 選択中ポジションをドラッグ範囲に配置（ペイント）。バー端はリサイズ優先 | バー上の既存ポジションクリックでポップオーバー表示 |
| 消去 | `erase` | ポジション削除 | ドラッグ範囲内のポジションを消去。バー端はリサイズ優先 | バー上クリックでポップオーバー表示 |

### 2.2 ドラッグ操作

#### 2.2.1 ペイント（配置モードのドラッグ）

- **条件**: `toolMode === "assign"` かつ `selectedPositionId` が選択済み
- **動作**: ドラッグ開始位置から現在位置までの範囲に、選択中ポジションのセグメントを追加
- **最小ドラッグ距離**: `timeRange.unit`（分）以上移動しないと適用されない
- **重複処理**: ドラッグ範囲内の既存ポジションは上書き（分割 or 削除）
- **未提出スタッフ**: シフトデータが存在しない場合、自動的に新しいShiftDataを作成
- **プレビュー**: 選択中ポジションの色で半透明ボックスを表示

#### 2.2.2 消去（消去モードのドラッグ）

- **条件**: `toolMode === "erase"` かつドラッグ開始位置にポジションが存在
- **動作**: ドラッグ範囲内のすべてのポジションセグメントを削除
- **部分重複**: 完全に含まれるセグメントは削除、一部重複のセグメントは分割して残す
- **プレビュー**: 赤い点線ボックスを表示

#### 2.2.3 単体リサイズ（全モード共通）

- **条件**: ポジションバーの端（左端 or 右端）にカーソルがある状態でドラッグ開始
- **判定閾値**: `RESIZE_EDGE_THRESHOLD`（10px）以内
- **動作**: 該当ポジションセグメントの開始時刻 or 終了時刻を変更
- **スナップ**: `timeRange.unit` 単位にスナップ
- **最小幅**: `timeRange.unit` 未満にはならない
- **上書き**: リサイズで他のポジションと重なった場合、他方を上書き

#### 2.2.4 連動リサイズ（全モード共通）

- **条件**: 隣接する2つのポジションバーの境界にカーソルがある状態でドラッグ開始
- **判定閾値**: `RESIZE_EDGE_THRESHOLD`（10px）以内
- **動作**: 前のセグメントの終了時刻と次のセグメントの開始時刻を同時に移動
- **自動削除**: リサイズにより `timeRange.unit` 未満になったセグメントは自動削除
- **カーソル**: `ew-resize`

### 2.3 Undo/Redo

- **履歴上限**: 50件（`MAX_HISTORY`）。超過時は最古の履歴を破棄
- **トリガー**: ドラッグ操作完了時（`setShiftsNormalized` 経由）
- **Undo**: 1つ前の状態に戻す。現在の状態を `future` スタックに退避
- **Redo**: `future` スタックから状態を復元。新しい操作を行うと `future` はクリア
- **キーボード**: Ctrl+Z（Undo）、Ctrl+Y / Cmd+Shift+Z（Redo）
- **UIボタン**: PositionToolbar 内にUndoボタン・Redoボタン。操作不可時は `disabled`

### 2.4 日付切替

- **UI**: DateTabs による横スクロール可能なタブ
- **表示形式**: `M/D(ddd)` 例: `1/28(火)`
- **動作**: タブ切替時に `selectedDate` を更新し、現在のソートモードで再ソートを実行
- **初期値**: `dates[0]`

### 2.5 ソート（SortMode）

| モード | 表示名 | ソートキー |
|--------|--------|-----------|
| `default` | デフォルト | 元の配列順 |
| `request` | シフト希望順 | 希望開始時刻 → 希望終了時刻 → 名前（日本語） |
| `startTime` | 出勤順 | 最早ポジション開始時刻 → 最早終了時刻 → 名前（日本語） |

**共通ルール**:
- 未提出スタッフは常に末尾
- 提出済みだが希望/ポジションなしのスタッフはその次
- 日本語名前比較は `localeCompare("ja")` を使用
- **スナップショット方式**: ソート実行時点の `sortedStaffIds` を保持。シフト編集でリアルタイムに並び順は変わらない

### 2.6 ポジションバー自動マージ + 休憩自動挿入

`normalizePositions()` がすべてのシフト変更操作後に自動実行される。

#### マージ（mergeAdjacentPositions）
- 同じ `positionId` を持つ隣接セグメント（`end === start`）を1つに結合
- 重複セグメント（時間が重なる）も結合

#### 休憩自動挿入（fillGapsWithBreak）
- ポジション間の隙間に「休憩」セグメントを自動挿入
- **条件**: `breakPosition`（`name === "休憩"` のポジション）が存在する場合のみ
- **対象**: バー間のギャップのみ。先頭・末尾のギャップには挿入しない
- **ID生成**: `break-{前セグメント終了時刻}-{次セグメント開始時刻}`

#### 正規化パイプライン（normalizePositions）
1. 既存の休憩セグメントをすべて除去
2. 同一ポジションの隣接セグメントをマージ
3. ギャップに休憩セグメントを挿入

### 2.7 人員充足サマリー（SummaryRow）

テーブルフッターに表示される充足状況。

#### 表示モード

| モード | 値 | 表示内容 |
|--------|-----|---------|
| 色表示 | `color` | 時間帯ごとの充足率を6色のグラデーションで表示 |
| 数値表示 | `number` | `配置済み/必要人数` の数値（例: `3/5`） |

#### 充足率カラーマップ（FILL_RATE_COLORS）

| 充足率 | 色 |
|--------|-----|
| 0-20% | 赤 (`red.400`) |
| 21-40% | オレンジ (`orange.300`) |
| 41-60% | 黄 (`yellow.300`) |
| 61-80% | ライム (`green.200`) |
| 81-100% | 緑 (`green.400`) |
| 101%以上 | 青 (`blue.300`) |

#### 展開/折りたたみ
- **折りたたみ時**: 合計行のみ表示
- **展開時**: 合計行 + ポジション別の行を表示
- **必要人数の算出**: `ceil(requiredCountPerHour / positionCount)`（ポジション均等分配）

### 2.8 ポップオーバー（ShiftPopover）

シフトバークリック時に表示される詳細/操作パネル。

#### 表示内容
- **ヘッダー**: スタッフの提出状態に応じて
  - 未提出: 「未提出」テキスト
  - 希望あり: 「希望: HH:MM - HH:MM」
  - 希望なし: 「希望: なし」
- **ポジション一覧**: スクロール可能（max-height: 200px）。各セグメントに削除ボタン（−アイコン）
- **全削除ボタン**: 赤ボタン。ポジションが0件の場合は `disabled`

#### 配置と動作
- `Portal` で描画。`position: fixed` でクリック位置基準に配置
- クリック位置の上下スペースを判定し、上 or 下に表示
- ウィンドウスクロール時に自動クローズ（`capture` フェーズで検知）
- ポップオーバー外クリックでクローズ

#### ポジション削除時の特殊処理
- **通常ポジション削除**: 単純なフィルター除去
- **休憩セグメント削除**: 前のポジションバーを延長してギャップを埋める（`deletePositionFromShift`）
- **全ポジション削除後**: ポップオーバーを自動クローズ
- **削除後の再正規化**: 残りのポジションに対して `normalizePositions` を再実行

### 2.9 キーボードショートカット

| キー | 動作 | 条件 |
|------|------|------|
| Ctrl+Z / Cmd+Z | Undo | INPUT/TEXTAREA/contentEditable にフォーカスしていないこと |
| Ctrl+Y | Redo | 同上 |
| Cmd+Shift+Z | Redo（Mac） | 同上 |

### 2.10 横スクロール（選択モードのみ）

- **条件**: `toolMode === "select"` かつ、リサイズ端でない位置でドラッグ開始
- **動作**: マウスの水平移動量（`dx`）に応じて `tableContainerRef.scrollLeft` を更新
- **カーソル**: スクロール中は `grabbing`、通常時は `grab`
- **終了**: `mouseUp` または `mouseLeave` でスクロール解除

### 2.11 スタッフ編集モーダル（StaffEditModal）

スタッフ名クリックでスタッフ情報を表示・編集できるモーダルを開く。

#### 表示内容
- **基本情報**: メールアドレス、表示名、週最大勤務時間、時給
- **スキル**: ポジション別スキルレベル（4段階：未経験/研修中/一人前/ベテラン）
- **メモ**: スタッフメモ、勤務スタイルメモ

#### 動作フロー
1. スタッフ名セルをクリック
2. モーダルが開き、ローディング状態を表示
3. `staffId` を使って API からスタッフ情報を取得
4. フォームに現在の値を表示
5. 編集 → 保存ボタンでデータ更新
6. 保存成功後、モーダルを自動クローズ

#### 特記事項
- **退職処理は含まない**: シフト作成時には不要のため、StaffEdit ページでのみ対応
- **シフト表との連携**: 保存後に `onSave` コールバックでシフト表の再描画が可能
- **共通フォーム**: StaffEditForm コンポーネントを StaffEdit ページと共用

### 3.4 シフト行（スタッフごとのドラッグ領域）

| 要素 | イベント | 動作 |
|------|---------|------|
| 行コンテナ (Box) | mouseDown | ドラッグ開始判定。モードに応じてペイント/消去/リサイズを開始。選択モードでドラッグ未開始なら横スクロール開始 |
| 行コンテナ (Box) | mouseMove | ドラッグ中: 位置更新（`handleMouseMove`）。横スクロール中: `scrollLeft` 更新。カーソルスタイル更新 |
| 行コンテナ (Box) | mouseUp | ドラッグ確定。ペイント/消去の場合はシフトデータ更新 + 正規化。ペイント/消去モードでクリック（移動なし）の場合はポップオーバー表示。横スクロール解除 |
| 行コンテナ (Box) | mouseLeave | ドラッグ中断（`handleMouseUp` + `stopScrollDrag`）。操作は破棄される |

#### mouseDown の詳細フロー

```
mouseDown
├── paintClickAnchorRef に現在座標を保存
├── handleMouseDown(e, staffId, rect) 呼び出し
│   ├── [全モード共通] リサイズ端の検出
│   │   ├── 連動リサイズ境界を検出 → mode: "position-resize-start" or "position-resize-end"
│   │   └── 単体リサイズ端を検出 → mode: "position-resize-start" or "position-resize-end"
│   ├── [assign モード] ポジション選択済みなら → mode: "paint"
│   │   └── 未提出スタッフの場合、新しい ShiftData を自動生成
│   ├── [erase モード] ポジションが存在するなら → mode: "erase"
│   └── [select モード] → return false（ドラッグ未開始）
└── [select モードで未開始の場合] 横スクロール開始
```

#### mouseUp の詳細フロー（ペイント/消去モード）

```
mouseUp
├── [paint モード] ドラッグ距離 < timeRange.unit の場合
│   └── ターゲットシフト上の既存ポジションをクリック → ポップオーバー表示
├── [erase モード] ドラッグ距離 < timeRange.unit の場合
│   └── ターゲットシフトをクリック → ポップオーバー表示
├── handleMouseUp() 呼び出し
│   ├── [paint モード] ドラッグ距離 >= unit → paintPosition() でセグメント追加
│   ├── [erase モード] ドラッグ距離 >= unit → erasePosition() でセグメント削除
│   ├── [resize モード] resizePosition() or resizeLinkedPositions() でリサイズ確定
│   └── dragState をリセット
└── stopScrollDrag()
```

### 4.3 シフト行の内部構造

各スタッフ行のセル内は `position: relative` のコンテナで、以下が重なる:

```
[最背面] GridLines       ... pointerEvents: none
[中間]   ShiftBar        ... 希望時間バー + ポジションバー群
[中間]   空状態テキスト   ... pointerEvents: none（「希望なし」「未提出」）
[最前面] DragPreview     ... ドラッグ中のみ表示
```

- **行の高さ**: 固定 `50px`
- **左右パディング**: `px: 5`（20px = `TIME_AXIS_PADDING_PX`）

### 4.4 ShiftBar の描画レイヤー

1つの ShiftBar 内に3レイヤーが重なる:

| レイヤー | 高さ | 位置 | 内容 |
|---------|------|------|------|
| 希望時間バー | 28px | `top: 50%`, `translateY(-50%)` | グレー破線ボーダー。`requestedTime` に基づく |
| ポジションバー群 | 20px | `top: 50%`, `translateY(-50%)` | 各 PositionSegment の色付きバー |
| 時刻ラベル | - | 各バー中央 | `xs` サイズ。「09:00」「17:00」等 |

#### ポジションバーの border-radius ルール
- 先頭セグメント: 左側に `md` radius
- 末尾セグメント: 右側に `md` radius
- 中間セグメント: radius なし（隣接バーと視覚的に接続）

#### リサイズ中のリアルタイム更新
- `linkedTarget` がある場合、前セグメント/次セグメントの幅をリアルタイムに更新して表示
- `currentMinutes` を使って、ドラッグ中の見た目を即座に反映

### 4.5 カーソル変化

| 条件 | カーソル |
|------|---------|
| ポジションバー端（全モード共通） | `ew-resize` |
| 選択モード・バー端以外 | `grab` |
| 選択モード・横スクロール中 | `grabbing` |
| 配置モード・ポジション選択済み | `crosshair` |
| 消去モード・ポジション存在する位置 | `crosshair` |
| その他 | `default` |

### 4.6 DragPreview（ドラッグプレビュー）

| モード | 表示 | 条件 |
|--------|------|------|
| paint | 選択中ポジション色の半透明ボックス（opacity: 0.3） | ドラッグ範囲 >= 0.5% |
| erase | 赤の点線ボーダーボックス（opacity: 0.15） | ドラッグ範囲 >= 0.5% |
| resize系 | 表示なし（ShiftBar 自体がリアルタイム更新） | - |

### 4.7 PositionToolbar のレイアウト

```
┌─────────────────────────────────────────────────────────────┐
│ [← →]  │ [選択][配置][消去] │ [ホール][キッチン][レジ][休憩] │ [色▾][数値▾]
│  履歴    │      ツール        │     ポジション（配置時のみ有効）│  サマリー表示
├─────────────────────────────────────────────────────────────┤
│ ヒントテキスト（左寄せ）                    凡例（右寄せ）    │
│                                          ■充足率カラー      │
│                                          ┊希望時間┊         │
└─────────────────────────────────────────────────────────────┘
```

- **ツールボタンの色分け**: 選択=青系、配置=緑系、消去=赤系
- **ポジションボタン**: 各ポジション固有の色。選択時はリング表示（`outline`）。選択モード時は `opacity: 0.4` + `pointerEvents: none`
- **凡例**: 充足率の6色スケール（0%〜100%超）+ 希望時間の破線ボックス

### 4.8 SummaryRow の表示

#### 色モード（`displayMode === "color"`）
- 時間帯ごとの充足率を計算し、対応する色で `linear-gradient` を生成
- 1時間ごとのブロックが連続的にグラデーション表示

#### 数値モード（`displayMode === "number"`）
- 各時間帯ごとに `配置済み人数/必要人数` を表示（例: `3/5`）
- 充足率に応じた色でテキスト色を変更

### 4.9 ShiftPopover の表示

```
┌───────────────────────┐
│ 希望: 09:00 - 17:00   │ ← ヘッダー（提出状態に応じて変化）
├───────────────────────┤
│ ■ ホール 09:00-12:00 [-] │ ← スクロール可能（max-height: 200px）
│ ■ 休憩   12:00-13:00 [-] │
│ ■ キッチン 13:00-17:00[-] │
├───────────────────────┤
│ [全てのポジションを削除]   │ ← 赤ボタン。0件時 disabled
└───────────────────────┘
```

- **幅**: 280px
- **配置**: クリック位置の下に表示。下にスペースがない場合は上に表示
- **zIndex**: 1000

### 4.10 空状態の表示

| 状態 | 表示テキスト | 色 |
|------|-------------|-----|
| 提出済み・希望なし（`no_request`） | 「希望なし」 | `gray.400` |
| 未提出（`not_submitted`） | 「未提出」 | `orange.400` |

- スタッフ名セルにも同じテキストが `xs` サイズで表示される
- シフト行内では中央配置。`pointerEvents: none`

### 4.11 maxHeight 計算

コンポーネント全体の `maxHeight` はビューポートからの余白を考慮:

| 画面サイズ | 計算式 | 備考 |
|-----------|--------|------|
| SP (base) | `calc(100dvh - 96px)` | py=8px*2 + mb=80px(BottomMenu) = 96px |
| PC (lg) | `calc(100dvh - 64px + 200px)` | py=32px*2 = 64px |

## 6. ユーティリティ関数

### 6.1 座標変換（shiftOperations.ts）

| 関数 | 入力 | 出力 | 用途 |
|------|------|------|------|
| `timeToMinutes(time)` | `"10:30"` | `630` | 時刻文字列 → 分 |
| `minutesToTime(minutes)` | `630` | `"10:30"` | 分 → 時刻文字列 |
| `pixelToMinutes({x, containerWidth, timeRange})` | ピクセル座標 | 分（unit単位にスナップ） | マウス位置 → 時刻 |
| `minutesToPercent({minutes, timeRange})` | 分 | パーセント | 時刻 → 表示位置 |
| `percentToCalcLeft(percent)` | パーセント | CSS `calc()` 式 | パディング考慮の左位置 |
| `percentToCalcWidth(startPercent, endPercent)` | パーセント2つ | CSS `calc()` 式 | パディング考慮の幅 |

### 6.2 検出関数（shiftOperations.ts）

| 関数 | 用途 |
|------|------|
| `findShiftAtPosition({shifts, staffId, date})` | 指定スタッフ・日付のシフトを取得 |
| `findPositionAtPosition({shift, minutes})` | 指定時刻にあるポジションセグメントを取得 |
| `detectPositionResizeEdge({shift, x, containerWidth, timeRange})` | カーソル位置がリサイズ端かを判定。結果: `{positionId, edge: "start" \| "end"}` or null |
| `detectLinkedResizeEdge({shift, x, containerWidth, timeRange})` | カーソル位置が連動リサイズ境界かを判定。結果: `LinkedResizeTarget` or null |

### 6.3 操作関数（shiftOperations.ts）

| 関数 | 用途 | 戻り値 |
|------|------|--------|
| `paintPosition({shift, start, end, position})` | 指定範囲にポジションセグメントを追加。既存と重複する部分は上書き（分割） | 更新後の ShiftData |
| `erasePosition({shift, startMinutes, endMinutes})` | 指定範囲内のポジションを削除。部分重複は分割して残す | 更新後の ShiftData |
| `resizePosition({shift, positionId, edge, newMinutes, minDuration})` | ポジションセグメントの開始/終了時刻をリサイズ | 更新後の ShiftData |
| `resizeLinkedPositions({shift, linkedTarget, newBoundaryMinutes, minDuration})` | 隣接2セグメントの境界を同時移動 | 更新後の ShiftData |

### 6.4 正規化関数（shiftOperations.ts）

| 関数 | 用途 |
|------|------|
| `mergeAdjacentPositions(positions)` | 同一positionIdの隣接/重複セグメントを結合 |
| `fillGapsWithBreak({positions, breakPosition})` | セグメント間のギャップに休憩を挿入 |
| `normalizePositions({positions, breakPosition})` | 上記2つを順に実行（休憩除去→マージ→休憩挿入） |
| `deletePositionFromShift({shift, positionSegmentId, breakPositionId})` | 単一セグメント削除。休憩削除時は前バーを延長 |

### 6.5 ソート関数（sortStaffs.ts）

| 関数 | 用途 |
|------|------|
| `sortStaffs({staffs, shifts, selectedDate, sortMode})` | 指定モードでスタッフ配列をソート |
| `compareByName(a, b)` | 日本語名前比較（`localeCompare("ja")`） |
| `compareByTailGroup(a, b, shifts, date)` | 未提出/希望なしを末尾にグループ化 |
| `getRequestedMinutes(staff, shifts, date)` | 希望時間の開始/終了を分で取得 |
| `getEarliestStartMinutes(staff, shifts, date)` | 最早ポジション開始時刻を取得 |
| `getEarliestEndMinutes(staff, shifts, date, startMinutes)` | 指定開始時刻における最早終了時刻を取得 |
