# E2Eテスト修正方針

## 問題の概要

21件のE2Eテストが失敗している。原因は以下の3つ:

1. **タブが存在しない（18件）**: テストは店舗詳細ページに「スタッフ」「店舗情報」タブを期待しているが、実際の実装では別ルートになっている
2. **店舗登録で要素が複数ヒット（1件）**: `getByText(shopName)` が3つの要素にマッチ
3. **店舗詳細でタブ確認（2件）**: 店舗情報タブの存在確認が失敗

## Playwrightで確認した実際のUI構造

### ルート構造（実際の動作）

| ルート | 内容 |
|--------|------|
| `/shops` | 店舗一覧 |
| `/shops/$shopId` | 店舗詳細（営業時間、シフト提出期限、時間単位、説明） |
| `/shops/$shopId/staffs` | スタッフ一覧 |
| `/shops/$shopId/staffs/$staffId` | スタッフ詳細 |

### 重要な発見

1. **店舗一覧のカードをクリックすると `/shops/{shopId}/staffs` に直接遷移する**（店舗詳細ではない！）
2. 店舗詳細ページにはタブがない
3. スタッフ一覧には検索・フィルター・「メンバーを追加」ボタンがある

## 修正方針

**E2Eテストを現状のルート構造に合わせて修正する（B案）**

## 修正対象ファイル

### 1. e2e/helpers/navigation.ts

- `goToFirstShop()` → 店舗カードが `/staffs` に遷移することを考慮（または店舗詳細用に別ヘルパー追加）
- `goToStaffTab()` → **削除**（タブは存在しない）
- `goToStaffList()` → 店舗カードクリックで直接スタッフ一覧に遷移するので簡略化
- `goToInviteTab()` → **削除**（タブは存在しない）

### 2. e2e/scenarios/shop/detail.test.ts

- タブ関連のテストを削除（店舗情報タブ、スタッフタブ切り替え）
- 店舗詳細ページの実際の内容（営業時間、シフト提出期限等）を確認するテストに書き換え
- 店舗詳細ページには直接URLで遷移する必要あり

### 3. e2e/scenarios/shop/list.test.ts

- 「店舗情報タブが表示される」→ 実際には `/staffs` に遷移するので「スタッフ一覧」確認に変更

### 4. e2e/scenarios/shop/register.test.ts

- `getByText(shopName)` → `getByRole("heading", { name: shopName, level: 3 })` に変更

### 5. e2e/scenarios/invite/create.test.ts

- `beforeEach` の `goToStaffTab()` を削除（店舗カードクリックで直接スタッフ一覧に遷移済み）

### 6. e2e/scenarios/staff/*.test.ts (list, add, edit, resign)

- `goToStaffList()` ヘルパーの修正により自動的に修正される

## 詳細な修正内容

### Step 1: navigation.ts ヘルパーの修正

```typescript
// 削除するヘルパー
// - goToStaffTab() - タブは存在しない
// - goToInviteTab() - タブは存在しない
// - waitForInviteList() - 招待タブは存在しない
// - goToInviteList() - 招待タブは存在しない

// 修正: goToFirstShop() - 店舗詳細ページ用（直接URL遷移）
export const goToFirstShopDetail = async (page: Page) => {
  await page.goto("/shops");
  const shopLink = page.getByRole("link").filter({ has: page.getByRole("group") }).first();
  const href = await shopLink.getAttribute("href");
  // href は /shops/{shopId}/staffs の形式
  const shopId = href?.match(/\/shops\/([^/]+)/)?.[1];
  await page.goto(`/shops/${shopId}`);
  await expect(page).toHaveURL(/\/shops\/[^/]+$/);
};

// 修正: goToStaffList() - 店舗カードクリックで直接スタッフ一覧に遷移
export const goToStaffList = async (page: Page) => {
  await page.goto("/shops");
  await page.getByRole("link").filter({ has: page.getByRole("group") }).first().click();
  // 店舗カードは /shops/{shopId}/staffs に遷移する
  await expect(page).toHaveURL(/\/shops\/[^/]+\/staffs$/);
  await waitForStaffList(page);
};
```


### Step 5: invite/create.test.ts の修正

```typescript
// Before

  await goToFirstShop(page);
```

1. `e2e/helpers/navigation.ts` を修正（ヘルパー削除・修正）
2. `e2e/scenarios/shop/detail.test.ts` を修正（タブテスト削除、店舗詳細確認に変更）
3. `e2e/scenarios/shop/list.test.ts` を修正（遷移先確認を変更）
4. `e2e/scenarios/shop/register.test.ts` を修正（セレクター具体化）
5. `e2e/scenarios/invite/create.test.ts` を修正（goToStaffTab削除）
6. `pnpm e2e` で全テスト実行・確認
7. 失敗があれば個別に修正

## 確認事項

- スタッフ関連テスト（list, add, edit, resign）はnavigation.tsの修正で自動的に動作する
- 店舗詳細テストは `goToFirstShopDetail()` で直接URLに遷移する必要がある
