# シフトボード ヘッダー再構成 — pencil.devプロンプト

## アプリの概要

YPS（シフト管理ツール）のシフトボード画面です。
小規模飲食店（6〜20名）の店長が、週次シフトを組むために使うメイン作業画面。
PC利用がメイン（閉店後のバックヤード作業）だが、SP（スマホ）でも閲覧・編集が可能。

## 今回の変更目的

シフトボードのヘッダー周りを再構成し、**ガントチャート（作業エリア）の表示領域を最大化**する。
現状はヘッダー周りに約200pxを使っており、ガントが窮屈。これを約100pxに圧縮する。

## 変更の方針

### 情報の優先度整理

ヘッダーに表示していた情報を3段階に仕分けた結果：

**A：作業中ずっと使う（常に見える必須）**
- ビュー切替タブ（日別 / 一覧）
- 保存ボタン
- 確定ボタン（確定後は「再確定する」に変化）

**B：開始時に1回確認（見えるが控えめでOK）**
- ← 戻る導線（ダッシュボードへ）
- 期間表示（例：1/20(月)〜1/26(日) のシフト）
- 確定済みステータス（確定後のみ表示。例：✓ 確定済み 3/28 23:15）

**C：ヘッダーから削除**
- 提出状況（8/10人完了）→ ガント上の「未提出」ラベルで代替可能
- ページタイトル「シフト表」→ 期間表示と重複

### 構成ルール

- グローバルヘッダー（緑帯）は全ページ共通で変更しない
- グローバルヘッダーの「シフト管理」テキストはダッシュボードへのリンクとして機能する（下線付き、ホバーで色変化）
- 日付ナビ（< 1/20(火) > ）はヘッダーではなくガント側（ShiftFormコンポーネント）の責務。ヘッダーには含めない
- ビュー切替タブ（日別 / 一覧）はヘッダー（ツールバー行）に配置する

## PC レイアウト（1280px幅基準）

### 構成（2行）

```
行1: グローバルヘッダー（56px、既存のまま）
  [← シフト管理                              ログアウト]

行2: ツールバー（44px、1行に全情報を配置）
  [← 戻る  1/20(月)〜1/26(日)のシフト  [日別|一覧]  [保存] [シフトを確定する]]
```

### ツールバー行の各要素（左→右）

| 要素 | 位置 | スタイル |
|------|------|----------|
| ← 戻る | 左端 | テキストリンク、グレー系、ホバーで下線 |
| 期間テキスト | ←戻るの右 | fontWeight: 600、メインの見出し扱い |
| 日別 / 一覧 タブ | 中央寄り | SegmentGroup（Chakra UI）、小サイズ |
| 保存ボタン | 右寄り | outline / ghost スタイル、セカンダリ |
| 確定ボタン | 右端 | solid / teal系、プライマリ |

### 確定後の変化

- 確定ボタンのラベル →「再確定する」（outlineスタイルに変更）
- 確定ボタンの下に小さく「✓ 確定済み（3/28 23:15）」テキスト（green.600）

## SP レイアウト（375px幅基準）

### 構成（3行）

```
行1: グローバルヘッダー（48px、既存のまま）
  [← シフト管理                    ログアウト]

行2: ナビ行（36px）
  [← 戻る              1/20〜1/26のシフト]

行3: ツールバー行（36px）
  [[日別|一覧]                [保存] [確定]]
```

### SP固有のルール

- 期間テキストは短縮表記（「1/20〜1/26」。曜日省略）
- 保存ボタンはラベルなしアイコンのみ（LuSave）でもOK
- 確定ボタンは「確定」に短縮
- 確定済みステータスは行2の下に小さく表示

## デザインの方向性

- Chakra UIライクなクリーンなスタイル
- ツールバー行は白背景 + 下ボーダー（gray.200）
- ボタン間のギャップは8px
- 全体を通して余白を詰めすぎず、各要素が視認しやすいこと
- グローバルヘッダーの色はteal.600

## 作成するパターン一覧

### パターン1：PC 日別ビュー（通常状態）
- ツールバーに「日別」タブがアクティブ
- 確定ボタンは「シフトを確定する」（プライマリ）
- ツールバーの下にガントチャートが続くイメージ（ガント自体の詳細は不要、上部だけ見えていればOK）

### パターン2：PC 一覧ビュー（確定済み状態）
- ツールバーに「一覧」タブがアクティブ
- 確定ボタンは「再確定する」（outlineスタイル）
- 確定ボタンの下に「✓ 確定済み（3/28 23:15）」テキスト
- ツールバーの下に一覧テーブルが続くイメージ

### パターン3：SP 日別ビュー（通常状態）
- 3行ヘッダー構成
- 確定ボタンは「確定」（短縮ラベル）
- ヘッダーの下にスタッフカードが続くイメージ

### パターン4：SP 一覧ビュー（確定済み状態）
- 3行ヘッダー構成
- 確定ボタンは「再確定」
- 確定済みステータス表示

## サンプルデータ

- 期間：1/20(月)〜1/26(日)
- 確定日時（パターン2, 4用）：3/28 23:15
