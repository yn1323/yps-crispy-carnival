# 追加スタッフへの希望シフト通知

## 概要

シフト募集中に管理画面からスタッフを追加した場合、そのスタッフが希望提出リンクを受け取れず希望シフトを提出できない問題を解消する。

- スタッフ追加時は、追加されたスタッフのメールアドレスへ希望提出リンクを送る。
- LINEログイン完了時、またはLINE公式アカウントを友だち追加したタイミングで、スタッフのLINEへ希望提出リンクを送る。
- 対象は `status === "open"` かつ締切前または締切当日のシフト募集。複数ある場合は募集ごとに1通ずつ送る。

## 実装方針

- 既存の募集作成時通知テンプレートと magic link 生成を再利用し、1スタッフ向けの internal action/query を追加する。
- `staff.mutations.addStaffs` では、新規insertされたスタッフIDだけを対象にメール通知 action を `scheduler.runAfter(0, ...)` で起動する。
- `line.mutations.finalizeLinking` では、LINEログイン完了時に `lineFollowing === true` の場合だけLINE通知 action を起動する。
- `line.mutations.dispatchWebhookEvents` では、followイベントで `lineFollowing` が `false` から `true` になった場合だけLINE通知 action を起動する。
- LINEログイン完了時に友だち追加されていない場合は、完了画面で友だち追加が必要な旨を表示し、後続のfollowイベントで通知する。
- 通知履歴テーブルは追加しない。メール通知とLINE通知は独立したイベントとして扱う。

## テスト方針

- Convexテスト
  - スタッフ追加時、締切前open募集のsubmit用magicLinkが追加スタッフ分だけ作成される。
  - confirmed募集、削除済み募集、締切超過募集は対象外。
  - LINEログイン完了時に `lineFollowing: true` なら通知がスケジュールされる。
  - follow Webhookで `false -> true` になった時だけ通知がスケジュールされる。
- E2E
  - 募集中の店舗でスタッフを追加し、追加スタッフ向けsubmit用magicLinkが発行され、リンクから希望提出できることを確認する。
  - LINE連携/follow相当のE2E補助mutationでLINE通知トリガーを作り、submit用magicLinkが発行されることを確認する。
  - 実メール配送/LINE Push成功は検証せず、通知URL発行と提出フローで検証する。
- 実行コマンド
  - `pnpm lint`
  - `pnpm type-check`
  - `pnpm test:convex -- convex/staff/mutations.test.ts convex/line/mutations.test.ts`
  - `pnpm e2e e2e/scenarios/<new-file>.test.ts`
