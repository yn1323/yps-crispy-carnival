## 1. MVP優先メール

シフト確定メール（募集開始メールはMVP後）

## 2. メール仕様

### 送信方法
- Resend（Clerk不使用）

### 差出人
- 名前: 店舗名（例: 居酒屋さくら）
- アドレス: noreply@yps.app

### 件名
```
【{店舗名}】{開始日}〜{終了日} シフト確定のお知らせ
```
例: 【居酒屋さくら】1/20(月)〜1/26(日) シフト確定のお知らせ

### 本文構成（オプションB: 個人シフト要約+リンク）

```
{スタッフ名}さん

{開始日}〜{終了日} のシフトが確定しました。

┌─ あなたのシフト ─────────────┐
│ 1/20 (月)  10:00 - 18:00    │
│ 1/21 (火)  休み              │
│ 1/22 (水)  12:00 - 21:00    │
│ 1/23 (木)  休み              │
│ 1/24 (金)  10:00 - 18:00    │
│ 1/25 (土)  休み              │
│ 1/26 (日)  休み              │
└─────────────────────────────┘

      [全員のシフトを確認する]

このリンクは24時間有効です。
リンクを開くと14日間閲覧できます。
期限切れの場合はこちらから再発行できます。

──────────────────────────────
シフトについてのご質問・変更希望は
お店に直接ご連絡ください。
※ このメールに返信しても届きません。
```

### 表示ルール
- 全日表示（休みも明示）
- 出勤日: 太字表示
- 休み: グレー文字
- 連絡先（電話番号等）はメールに含めない（スタッフは既に知っている前提）

## 3. マジックリンク認証設計

### 方式
- 自前トークン付きURL（Clerk不使用）
- URL形式: `https://shiftori.app/shifts/view?token={token}`

### トークン
- 有効期限: 24時間
- 用途: 初回アクセス時のセッション発行トリガー
- セキュリティ理由: 労働時間→収入推測リスクの軽減

### セッション
- 保存先: localStorage
- 有効期限: 14日間固定
- 初回トークン検証成功時に発行
- セッションIDでConvexクエリを認可

### リンク先ページ
- 全スタッフの確定シフトを表示（壁貼りシフト表のデジタル版）
- トークン期限切れの場合は同ページ内で案内表示

### セキュリティ上の許容トレードオフ
- リンクは転送可能（小規模チームでは意識的に受け入れ）
- セッションはブラウザに紐づく（リンク転送されても別端末では再認証必要）

## 4. 再発行フロー

### 概要
トークン期限切れ時に、スタッフが自分でリンクを再発行できる仕組み。

### 画面
- 状態1: メールアドレス入力フォーム
- 状態2: 「メールを送信しました」完了表示

### セキュリティ
- 入力されたメアドが登録済みかどうかに関わらず一律「メールを送信しました」と表示
- 未登録メアドの場合は実際には送信しない（メアド列挙攻撃の防止）

### MVPスコープ判断
- 24時間トークン＋管理者再送信だけでは管理者負担が増えるため、MVP内で実装
- 実装コスト: Convex mutation 1個 + シンプルなUI（入力 / 完了の2状態）
