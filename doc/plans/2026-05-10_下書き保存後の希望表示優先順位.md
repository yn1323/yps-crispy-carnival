# 下書き保存後の希望表示優先順位

## 背景

下書き保存後にスタッフが希望を提出・変更した場合でも、店長が保存した実シフトを勝手に上書きしない。一方で、保存時点で未提出かつ店長も未設定だったスタッフ×日付は、まだ店長判断がないため、保存後の希望を緑ラインとして表示する。

## 表示ルール

| 状態 | 緑ライン | 点線 |
| --- | --- | --- |
| 下書き保存前 + 希望あり | 最新希望 | 最新希望 |
| 下書き保存前 + 希望なし | なし | なし |
| 下書き保存後 + 保存済み実シフトあり | 保存済み実シフト | 最新希望があれば表示 |
| 下書き保存後 + 保存時未提出 + 店長未設定 + 保存後希望あり | 最新希望 | 最新希望 |
| 下書き保存後 + 保存時提出済み + 店長未設定 + 最新希望あり | なし | 最新希望 |

## 実装方針

- `recruitments.draftSavedAt` で下書き保存済みかを判定する。
- `shiftSubmissions.firstSubmittedAt` で、保存時点で提出済みだったかを判定する。
- `resolveDisplayShiftLine` に緑ライン判定を集約し、UI側では結果を `positions` に変換するだけにする。
- 既存の下書き保存注意モーダルは削除し、保存ボタンは即保存する。
