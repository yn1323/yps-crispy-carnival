## 0. 背景

URL を Slack / LINE / X / Discord 等に貼ったときに、**プレビューカードに「シフトリ」のロゴとキャッチコピーが出る状態**にする。
SPA（Vite + TanStack Router）なので、OGP はビルド時の `index.html` に埋め込めば十分（動的な OGP は今回不要）。

## 2. 必要なメタタグ

### 2-1. 基本

```html
<title>シフトリ | スタッフに「アプリ入れて」と言わないシフト管理</title>
<meta name="description" content="店長が登録すれば即日スタート。スタッフはメールのリンクをタップするだけ。シフトリは店舗スタッフ向けのシフト管理SaaSです。" />
<link rel="canonical" href="https://{{本番ドメイン}}/" />
```

### 2-2. OGP（Facebook / Slack / LINE 等）

```html
<meta property="og:type" content="website" />
<meta property="og:site_name" content="シフトリ" />
<meta property="og:title" content="シフトリ | スタッフに「アプリ入れて」と言わないシフト管理" />
<meta property="og:description" content="店長が登録すれば即日スタート。スタッフはメールのリンクをタップするだけ。" />
<meta property="og:url" content="https://{{本番ドメイン}}/" />
<meta property="og:image" content="https://{{本番ドメイン}}/ogp.png" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:locale" content="ja_JP" />
```

### 2-3. Twitter Card

```html
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="シフトリ | スタッフに「アプリ入れて」と言わないシフト管理" />
<meta name="twitter:description" content="店長が登録すれば即日スタート。スタッフはメールのリンクをタップするだけ。" />
<meta name="twitter:image" content="https://{{本番ドメイン}}/ogp.png" />
```

> ⚠️ Twitter アカウントはまだ無いので `twitter:site` / `twitter:creator` は省略。

### 2-4. noindex（身内リリース段階）

```html
<meta name="robots" content="noindex, nofollow" />
```

## 3. OGP 画像の作り方

### 3-1. デザイン方針

- 1200×630 px（OGP の標準）
- 背景：シフトリのブランドカラー（teal 系）
- 中央：ロゴ or アプリ名
- サブ：キャッチコピー「スタッフに『アプリ入れて』と言わないシフト管理」

### 3-2. 作成手段

| 手段 | 長所 | 短所 |
|------|-----|------|
| A. Figma / Pencil | 自由度高い | 時間かかる |
| B. Canva | テンプレ豊富 | ブランド統一が難しい |
| C. HTML/CSS → スクショ | 実装と同じ CSS でブランド統一 | 画像出力の手間 |
| D. AI 画像生成 | 速い | 細部のコントロールが効かない |

**選択：A（Pencil）または C（HTML/CSS）**。
既存のロゴ／ブランドカラーを流用しやすいため。1時間で終わらせるなら C が最短。

### 3-3. 画像の置き場所

- `public/ogp.png` に配置
- favicon も更新するなら `public/favicon.ico` を差し替え

## 4. 実装ステップ

### Step 1: 画像作成（30分）
1. Pencil or HTML/CSS で OGP 画像を作成
2. 1200×630 px で書き出し
3. `public/ogp.png` に配置
4. ファイルサイズを 300KB 以下に圧縮（tinypng 等）

### Step 2: メタタグ追加（15分）
1. `index.html` の `<head>` に上記メタタグを追加
2. `{{本番ドメイン}}` は staging のドメインで暫定設定
3. `description` と `og:description` を統一

### Step 3: 確認（15分）
1. staging にデプロイ
2. [OGP確認ツール](https://www.opengraph.xyz/) などで確認
3. Slack / LINE / X の DM に URL を貼って表示確認

## 5. 注意点

### 5-1. CDN キャッシュ
- Cloudflare Pages はキャッシュが強い
- OGP 画像を差し替えたら、URL にクエリパラメータを付けて判別可能にする
    ```html
    <meta property="og:image" content="https://{{ドメイン}}/ogp.png?v=20260409" />
    ```

### 5-2. 絶対URL
- `og:image` は絶対 URL で指定する（相対URL はサービスによって解釈が割れる）

### 5-3. SPA での制約
- TanStack Router は（今回の構成では）SSR していないので、ルート別の OGP は出せない
- 全ルート共通で LP 用の OGP が表示される
- ルート別 OGP が必要になった時点で Cloudflare Workers でヘッダーを書き換える等を検討（今回はやらない）

### 5-4. noindex 忘れ
- 「身内に見せるだけなのに Google に載ってしまう」を避ける
- `<meta name="robots" content="noindex, nofollow" />` は必ず入れる
- `public/robots.txt` も `Disallow: /` にしておく

### 5-5. 画像のトリミング
- X や LINE は画像の中央付近しか見えない場合がある
- 「文字を画像の端ギリギリに置かない」。中央60%以内に収める
