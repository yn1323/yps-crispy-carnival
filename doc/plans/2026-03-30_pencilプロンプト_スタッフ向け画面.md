# pencil.dev モック作成プロンプト: スタッフ向け画面

## 対象ファイル
- `design/shiftview.pen` → シフト閲覧ページ
- `design/shift-reissue.pen` → リンク再発行ページ

## 既存デザインシステム
既存の.penファイルのVariables（Chakra UI準拠のカラー・スペーシング・タイポグラフィ）を使用すること。
主要変数: `$N:fg/default`, `$N:fg/muted`, `$N:fg/subtle`, `$N:bg/default`, `$N:bg/muted`, `$N:bg/subtle`, `$N:border/default`, `$N:blue/solid`, `$N:blue/subtle`, `$N:fontSize/sm`(14), `$N:fontSize/md`(16), `$N:fontSize/lg`(18), `$N:fontSize/xl`(20), `$N:radii/lg`(8), `$N:spacing/4`(16), `$N:spacing/6`(24)

既存コンポーネント: Header/PC, Header/SP, ContentWrapper, FormCard, LoadingState, Empty, Dialog, Title 等が利用可能。

---

## 画面1: シフト閲覧ページ（shiftview.pen）

### コンテキスト
- スタッフがメールのマジックリンクを踏んで到着するページ
- 居酒屋の壁に貼ってあるシフト表のデジタル版
- 全スタッフの確定シフトが一覧で見える
- PC/スマホ両対応だがスマホ利用がメイン（スタッフは移動中に確認）
- ログイン不要（トークン認証）

### 状態パターン（2状態を同一ファイルに作成）

```
│  │      │ 月   │ 火   │ 水   │ 木   │ 金   │ 土日 │
│  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  │佐藤  │10-18 │ 休   │12-21 │ 休   │10-18 │ 休   │

│  ├──────┼──────┼──────┼──────┼──────┼──────┼──────┤

```

**仕様:**
- ヘッダー: 店舗名 + 対象期間
- テーブル: 縦軸=スタッフ名、横軸=日付（曜日付き）
- セル内容: 時間帯テキスト（例: 10:00-18:00）or 「休」
- 休みのセルはグレー文字 `$N:fg/subtle`、出勤セルは通常テキスト `$N:fg/default`
- テーブルは横スクロール可能（スマホ対応）、スタッフ名列は固定
- フッター: セッション残り日数の案内（控えめなinfo表示）
- スマホ幅（375px）を基準にデザイン。PC幅（1280px）でも破綻しないこと

- トークン期限切れ時にスタッフが自分で再発行するページ
- メアド入力 → 新しいマジックリンクをメール送信
- 極限までシンプルに。ITリテラシーが低い人でも迷わない

## デザイン共通ルール

- **背景**: `$N:bg/default`（白）
- **ヘッダー帯**: 店舗名を表示。既存のHeader/PCコンポーネントがあれば活用、なければシンプルなバーで
- **カード風レイアウト**: ContentWrapper or 独自フレームで中央配置、max-width: 640px
- **フォント**: 本文 `$N:fontSize/md`(16px)、補足 `$N:fontSize/sm`(14px)、見出し `$N:fontSize/xl`(20px)
- **角丸**: `$N:radii/lg`(8px)
- **ボタン**: 高さ40px、角丸 `$N:radii/lg`、`$N:blue/solid` 背景 + 白テキスト
- **余白**: セクション間 `$N:spacing/6`(24px)、要素間 `$N:spacing/4`(16px)
- **全画面をPC(1280px幅)とSP(375px幅)の2サイズで作成**
