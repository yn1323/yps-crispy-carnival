# ラベル改善とガント画面整理

## 概要

認知ウォークスルーで発見されたUX課題を修正する。ユーザー（田中さん = 飲食店店長、Excelレベル）の言葉に合わせたラベルへの変更と、ガント画面の操作モデル簡素化。

## 変更内容

### 1. 「募集」→「シフト」系ラベル変更

ユーザーの頭の中にある言葉は「来月のシフト希望を集める」。「募集」は仕組み側の言葉であり、求人と紛らわしい。

#### `src/components/features/Dashboard/RecruitmentSection/index.tsx`
- セクション見出し: `募集` → `シフト`
- ボタンテキスト: `新しい募集を作成` → `シフト希望を集める`
- Empty title: `募集がありません` → `シフトがありません`
- Empty description: `新しい募集を作成して、スタッフのシフトを集めましょう` → `新しいシフトを作成して、スタッフの希望を集めましょう`

#### `src/components/features/Dashboard/RecruitmentCard/index.tsx`
- ステータスバッジ: `募集中` → `収集・編集中`
- ステータスバッジ: `送付完了` → `確定済み`
- ボタンテキスト: `シフトボードを開く` → `シフトを編集する`

#### `src/components/features/Dashboard/DashboardContent/index.tsx`
- ダイアログtitle: `新しい募集を作成` → `シフト希望を集める`
- 成功トースト: `募集を作成しました` → `シフトを作成しました`

### 2. ガント画面: 保存ボタン削除 + 送信ラベル変更

#### 方針
- **保存ボタンを削除する**。Convexのリアルタイム特性を活かし、将来的に自動保存に移行する前提
- 現時点では `handleConfirm` 内で `saveShiftAssignments` が呼ばれるため、確定時に保存される動作は維持される
- 「送信」を「確定して通知」に変更し、このアクションがシフト確定であることを明示する

#### `src/components/features/ShiftBoard/ShiftBoardHeader/index.tsx`
- 「保存」ボタン（`<Button>` + `LuSave`）を削除
- `スタッフに送信する` → `確定して通知する`
- `再送信する` → `再通知する`
- `送信済み（{formatDateTime(confirmedAt)}）` → `確定済み（{formatDateTime(confirmedAt)}）`
- `LuSave` のimportを削除

#### `src/components/features/ShiftBoard/ShiftBoardSPHeader/index.tsx`
- 「保存」ボタンを削除
- `送信` → `確定して通知`
- `再送信` → `再通知`
- `{formatDateTime(confirmedAt)}` の前のテキストがあれば `確定済み` に統一

#### `src/components/features/ShiftBoard/ShiftBoardPage/index.tsx`
- `handleSave` 関数を削除
- `isSaving` state を削除
- `ShiftBoardHeader` / `ShiftBoardSPHeader` への `onSave`, `isSaving` props を削除
- 確定ダイアログのタイトル: `スタッフにシフトを送信しますか？` → `シフトを確定して通知しますか？`
- 再送信ダイアログのタイトル: `スタッフにシフトを再送信しますか？` → `シフトを再通知しますか？`
- ダイアログの送信ボタン（submitLabel）: `送信する` → `確定して通知する`
- 成功トースト: `送信しました` → `確定しました`
- 成功トースト: `保存しました` → 削除（使われなくなるため）

#### `src/components/features/ShiftBoard/ConfirmShiftContent/index.tsx`
- `全スタッフにメールでシフトが送信されます。` → `全スタッフにメールでシフトが通知されます。`

#### `src/components/features/ShiftBoard/types.ts`
- `ShiftBoardHeaderProps` から `onSave` と `isSaving` を削除

### 3. 曜日の日本語化

#### `src/components/features/Shift/ShiftForm/pc/DailyView/DateTabs.tsx`
- `formatDate` 関数を修正: `dayjs(dateStr).format("M/D(ddd)")` → `getWeekdayLabel` を使って `"M/D(曜)"` 形式にする
- `getWeekdayLabel` を `../../utils/dateUtils` からimport
- 例: `5/1(Fri)` → `5/1(金)`

```ts
// 変更前
const formatDate = (dateStr: string) => {
  return dayjs(dateStr).format("M/D(ddd)");
};

// 変更後
import { getWeekdayLabel } from "../../utils/dateUtils";

const formatDate = (dateStr: string) => {
  return `${dayjs(dateStr).format("M/D")}(${getWeekdayLabel(dateStr)})`;
};
```

#### `e2e/pages/DashboardPage.ts`
- `getByRole("button", { name: "新しい募集を作成" })` → `getByRole("button", { name: "シフト希望を集める" })`

- `getByText("募集を作成しました")` → `getByText("シフトを作成しました")`
- `getByRole("button", { name: "シフトボードを開く" })` → `getByRole("button", { name: "シフトを編集する" })`
- `expectRecruitmentCardVisible` 内も同様に更新

#### `e2e/pages/ShiftBoardPage.ts`
- `save()` メソッドを削除
- `confirm()` メソッド内:

  - `getByRole("button", { name: "送信する" })` → `getByRole("button", { name: "確定して通知する" })`

- `expectConfirmedStatus()`:
  - `getByText(/送信済み/)` → `getByText(/確定済み/)`
- `expectResendButton()`:

#### `e2e/scenarios/first-shift-delivery.test.ts`
