# シフト管理機能 UI改善計画

## 1. 背景・課題

### 現状の問題点
現在のシフト管理画面（申請状況確認・シフト確定）はカード形式＋セレクトボックスで実装されているが、以下の課題がある：

| 課題 | 詳細 |
|------|------|
| スタッフ間の比較がしづらい | 誰がいつ入れるか横並びで見たい |
| 時間帯ごとの人数が分かりにくい | 「10-11時は何人？」がパッと見たい |
| スクロールが多く全体把握が大変 | カードが縦に並ぶと一覧性が低い |
| 視覚的にわかりづらい | 色や形での直感的な把握ができない |

### 解決方針
**ガントチャート風の表形式UI**に変更し、視認性と操作性を向上させる。

## 4. 操作方法詳細

### 4.1 労働時間バーの操作

| 操作 | 方法 | 詳細 |
|------|------|------|
| 新規作成 | 空白エリアをドラッグ | ドラッグした範囲に新しいバーを作成 |
| 時間調整 | バー左右端をドラッグ | 開始/終了時間を延長・短縮 |
| 削除 | ホバー時Xボタン | バーにマウスを乗せるとXボタン表示 |
| 削除 | ポップオーバー内「シフトを削除」 | クリックでポップオーバー→削除ボタン |
| 削除 | 右クリック→削除 | コンテキストメニューから削除 |

### 4.2 ポジション色の操作

| 操作 | 方法 | 詳細 |
|------|------|------|
| 塗る | ポジションボタン選択→バー上をドラッグ | 1時間単位（店舗設定に依存）で塗る |
| 上書き | 別のポジションで塗る | 新しい色で上書き |
| 削除 | ポジション色部分のXボタン | その色だけ削除（労働時間バーは残る） |
| 削除 | ポップオーバー内の個別削除 | 各ポジション行の[✕]ボタン |

### 4.3 ポップオーバー（共通）

```
┌───────────────────────────┐
│ 10:00 - 14:00             │  ← 労働時間
├───────────────────────────┤
│ 🔵 ホール    10-12時  [✕] │  ← ポジション色（個別削除可）
│ 🟠 キッチン  12-14時  [✕] │
├───────────────────────────┤
│ [🗑 シフトを削除]          │  ← 全削除（労働時間バーごと）
└───────────────────────────┘
```

- 労働時間バー、ポジション色どちらをクリックしても同じポップオーバーが開く
- 個別削除[✕]: そのポジション色のみ削除
- シフトを削除: 労働時間バー＋すべてのポジション色を削除

### 4.4 右クリックメニュー

```
┌─────────────────┐
│ コピー     Ctrl+C│
│ 貼り付け   Ctrl+V│
├─────────────────┤
│ 削除            │
└─────────────────┘
```

### 4.5 キーボードショートカット

| ショートカット | 機能 |
|----------------|------|
| Ctrl+Z | Undo（元に戻す） |
| Ctrl+Y | Redo（やり直し） |
| Ctrl+C | コピー（ホバー中のバーをコピー） |
| Ctrl+V | ペースト（マウス位置のスタッフ行に貼り付け） |

### 4.6 コピー&ペースト操作フロー

```
1. コピーしたいバーにマウスを乗せる（ホバー状態 = 選択中）
2. Ctrl+C または 右クリック→コピー
3. ペースト先のスタッフ行にマウスを移動
4. Ctrl+V または 右クリック→貼り付け
5. 同じシフト（労働時間＋ポジション色）がコピーされる
```

## 5. エッジケース・挙動定義

| ケース | 挙動 |
|--------|------|
| ポジション色を労働時間外にドラッグ | 労働時間も自動で延長 |
| 労働時間を縮めてポジション色がはみ出し | ポジション色を自動でカット（削除） |
| 営業時間外にドラッグ | 許可（早朝準備・深夜作業対応） |
| 異なるポジション色の重なり | 新しい色で上書き |
| 他のスタッフの行にドラッグ | ブロック（移動不可） |
| 労働時間バー同士の重なり | 自動で結合（1本のバーになる） |
| 間のポジション色を削除 | 労働時間バーは維持（未設定部分ができる） |
| 休憩時間 | 休憩用ポジション色で表現 |

## 13. Phase 0 詳細実装計画

### 13.1 技術選択

| 項目 | 選択 | 理由 |
|------|------|------|
| ドラッグ操作 | カスタムマウスハンドラー | 3種類の操作（作成/リサイズ/塗り）に柔軟対応。@dnd-kitは並び替えに特化しており不向き |
| Undo/Redo | カスタム実装（履歴スタック） | ライブラリ不要、シンプル、最大50件保持 |
| 状態管理 | useState | モック実装なのでローカル状態で十分 |
| UI | Chakra UI v3 | 既存パターンに準拠 |

### 13.4 ドラッグ操作の実装方針

```typescript
// 操作判定ロジック (index.tsx)
const handleMouseDown = (e: React.MouseEvent, staffId: string) => {
  const x = e.clientX - rect.left;

  // 1. ポジション選択中 → 塗りモード
  if (selectedPosition) {
    setDragMode('paint');
    return;
  }
  // 2. バー端（10px範囲）→ リサイズモード
  const edgeShift = findShiftEdge(staffId, x);
  if (edgeShift) {
    setDragMode('resize');
    return;
  }
  // 3. 空白エリア → 作成モード
  if (!isShiftAtPosition(staffId, x)) {
    setDragMode('create');
  }
};
```
