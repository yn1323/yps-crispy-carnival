# シフト管理機能 UI改善計画

## 1. 背景・課題

### 現状の問題点
現在のシフト管理画面（申請状況確認・シフト確定）はカード形式＋セレクトボックスで実装されているが、以下の課題がある：

| 課題 | 詳細 |
|------|------|
| スタッフ間の比較がしづらい | 誰がいつ入れるか横並びで見たい |
| 時間帯ごとの人数が分かりにくい | 「10-11時は何人？」がパッと見たい |
| スクロールが多く全体把握が大変 | カードが縦に並ぶと一覧性が低い |
| 視覚的にわかりづらい | 色や形での直感的な把握ができない |

### 解決方針
**ガントチャート風の表形式UI**に変更し、視認性と操作性を向上させる。

### 3.2 バーの2層構造

```

   ┌────┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈────┐   ← 希望シフトバー（太い、点線グレー）

   └────┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈────┘

上から見た図:
│ Aさん │ ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │ 希望シフト（グレー点線、太い）
│       │ ████████████      ██████████████████  │ ポジション色（細い、希望の上に重なる）

```

**視覚的効果:**
- 希望シフトバー: グレー点線の太いバー（スタッフが提出した希望時間、**編集不可**）
- ポジションバー: 細いバー（マネージャーが設定、希望の上に重なる）
- 未設定部分: 希望シフトバーのみ見える状態（ポジション割り当て待ち）

**重要:** 希望シフトバーは**編集・削除不可**。スタッフが提出した希望をそのまま表示。

## 4. 操作方法詳細

### 4.1 希望シフトバーの表示

**希望シフトバーはスタッフが提出した希望時間を表示するもので、マネージャーは編集できない。**

| 状態 | 表示 |
|------|------|
| 希望提出済み | グレー点線バーで希望時間を表示 |
| 未提出 | 「(未提出)」ラベルを表示 |

※ 希望シフトバーの作成・時間調整・削除は**不可**

### 4.2 ポジション色の操作

| 操作 | 方法 | 詳細 |
|------|------|------|
| 塗る | ポジションボタン選択→ドラッグ | 1時間単位（店舗設定に依存）で塗る。希望時間内外どちらも可 |
| 上書き | 別のポジションで塗る | 新しい色で上書き |
| 削除 | 消しゴムボタン選択→クリック/ドラッグ | クリック/ドラッグした範囲のポジションを削除（希望シフトバーは残る） |
| 削除 | ポップオーバー内の個別削除 | 各ポジション行の[✕]ボタン |

**注意:** 希望時間外へのポジション割り当ては許可されるが、別の場所で警告を表示する（今回スコープ外）

**変更履歴(2026-01-23):** バー上のXボタンを廃止し、消しゴムモードに統一。操作のステップ削減と視認性向上のため。

### 4.3 ポップオーバー

```
┌───────────────────────────┐
│ 希望: 10:00 - 14:00       │  ← 希望シフト時間（参考表示）
├───────────────────────────┤
│ 🔵 ホール    10-12時  [✕] │  ← ポジション色（個別削除可）
│ 🟠 キッチン  12-14時  [✕] │
├───────────────────────────┤
│ [🗑 全ポジションを削除]    │  ← ポジションのみ全削除
└───────────────────────────┘
```

- 希望シフトバー、ポジション色どちらをクリックしても同じポップオーバーが開く
- 個別削除[✕]: そのポジション色のみ削除
- 全ポジションを削除: すべてのポジション色を削除（希望シフトバーは残る）

### 4.4 キーボードショートカット

| ショートカット | 機能 |
|----------------|------|
| Ctrl+Z | Undo（元に戻す） |
| Ctrl+Y | Redo（やり直し） |

## 5. エッジケース・挙動定義

| ケース | 挙動 |
|--------|------|
| ポジション色を希望時間外にドラッグ | 許可（警告は別の場所で表示、今回スコープ外） |
| 営業時間外にドラッグ | 許可（早朝準備・深夜作業対応） |
| 異なるポジション色の重なり | 新しい色で上書き |
| 他のスタッフの行にドラッグ | ブロック（移動不可） |
| 間のポジション色を削除 | 希望シフトバーは維持（未設定部分ができる） |
| 休憩時間 | 休憩用ポジション色で表現 |
| 希望シフトバーの編集 | **不可**（スタッフ提出の希望をそのまま表示） |

- **ボタン**: ツールバーに [← Undo] [Redo →] ボタン配置

- 募集期間の日付をタブで表示
- クリックで日付を切り替え
- 週表示は今回保留（将来対応）

### 6.3 ポジションボタン

```
[ホール🔵] [キッチン🟠] [レジ🟢] [休憩⬜]
    ↑選択中
```

- クリックで選択状態をトグル
- 選択中のポジションでバー上をドラッグすると色が塗られる
- 休憩も1つのポジションとして扱う

## 13. Phase 0 詳細実装計画

### 13.1 技術選択

| 項目 | 選択 | 理由 |
|------|------|------|
| ドラッグ操作 | カスタムマウスハンドラー | 3種類の操作（作成/リサイズ/塗り）に柔軟対応。@dnd-kitは並び替えに特化しており不向き |
| Undo/Redo | カスタム実装（履歴スタック） | ライブラリ不要、シンプル、最大50件保持 |
| 状態管理 | useState | モック実装なのでローカル状態で十分 |
| UI | Chakra UI v3 | 既存パターンに準拠 |

### 13.2 型定義 (types.ts)

```typescript
export type ShiftData = {
  id: string;
  staffId: string;
  staffName: string;
  date: string;
  // 希望シフト時間（スタッフ提出、読み取り専用）
  requestedTime: { start: string; end: string } | null; // null = 未提出
  positions: PositionSegment[];
};

export type PositionSegment = {
  id: string;
  positionId: string;
  positionName: string;
  color: string;
  start: string;
  end: string;
};

export type PositionType = { id: string; name: string; color: string };
export type StaffType = { id: string; name: string; isSubmitted: boolean };
export type TimeRange = { start: number; end: number; unit: number };
export type DragMode = 'paint' | null; // 希望シフトバーは編集不可のため、create/resizeは削除
```

### 13.4 ドラッグ操作の実装方針

```typescript
// 操作判定ロジック (index.tsx)
// 希望シフトバーは編集不可のため、ドラッグ操作はポジション塗りのみ
const handleMouseDown = (e: React.MouseEvent, staffId: string) => {
  // ポジション選択中のみ塗りモード開始
  if (selectedPosition) {
    setDragMode('paint');
    return;
  }
  // ポジション未選択時は何もしない（希望シフトバーは編集不可）
};
```

**削除された操作:**
- バー作成（空白エリアドラッグ）→ 希望シフトバーは編集不可
- バーリサイズ（左右端ドラッグ）→ 希望シフトバーは編集不可
