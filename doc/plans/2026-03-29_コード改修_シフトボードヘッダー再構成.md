# シフトボード ヘッダー再構成 — コード改修プロンプト

## 概要

シフトボードのヘッダー構成を変更し、ガントチャートの表示領域を最大化します。
主な変更点は以下の3つです：

1. **ビュー切替タブ（日別/一覧）をShiftFormからShiftBoardHeaderに移動**
2. **TitleTemplateの使用をやめ、ShiftBoardHeaderに戻る導線を統合**
3. **提出状況テキストをヘッダーから削除**

## 現状の問題

ヘッダー周りの縦方向の構成が冗長で、ガントチャートの表示領域を約200px圧迫しています：

```
現状（PC）:
  Header（56px固定）→ 「シフト管理」+ ログアウト
  TitleTemplate パンくず → 「ダッシュボード > シフト表」
  TitleTemplate タイトル → 「シフト表」
  ShiftBoardHeader → 期間 + 提出状況 + 保存/確定
  ShiftForm内 SegmentGroup → 日別/一覧タブ
  ─── ここからガント ───
```

## 変更後の構成

```
変更後（PC）:
  Header（56px固定）→ 「← シフト管理」（ダッシュボードへのリンク）+ ログアウト
  ShiftBoardHeader（44px）→ ← 戻る + 期間 + [日別|一覧] + [保存] + [確定]
  ─── ここからガント ───

変更後（SP）:
  Header（48px固定）→ 「← シフト管理」+ ログアウト
  ShiftBoardSPHeader 行1（36px）→ ← 戻る + 期間
  ShiftBoardSPHeader 行2（36px）→ [日別|一覧] + [保存] + [確定]
  ─── ここからガント ───
```

## 技術方針：Controlled Props パターン

現状、ビュー切替の状態（`viewModeAtom`）は `ShiftForm` 内部のJotai `<Provider>` スコープ内にあります。
ビュータブをShiftBoardHeaderに移動するため、viewModeの管理をページ（`ShiftBoardPage`）に引き上げます。

### ShiftFormの変更

`ShiftForm` に以下のcontrolled propsを追加してください：

```typescript
type ShiftFormProps = {
  // ... 既存props ...
  
  // 追加: 外部からビューモードを制御する場合に使用
  viewMode?: ViewMode;
  onViewModeChange?: (mode: ViewMode) => void;
};
```

**挙動:**
- `viewMode` と `onViewModeChange` が渡された場合 → 外部制御モード（内部atomの代わりにpropsを使用）
- 渡されない場合 → 従来通り内部atomで自己管理（後方互換性を維持）
- `hideViewSwitcher` は引き続き有効（ShiftBoardPageからは `hideViewSwitcher={true}` を渡す）

### 具体的なコード変更内容

#### 1. `ShiftForm/stores.ts`
変更不要。内部atomはそのまま残す（後方互換性のため）。

#### 2. `ShiftForm/index.tsx`
- `viewMode` props が渡された場合、内部の `viewModeAtom` の代わりにpropsの値を使用する
- `onViewModeChange` props が渡された場合、タブ切替時にそちらを呼ぶ
- `hideViewSwitcher={true}` の場合、SegmentGroupを描画しない（既存の挙動そのまま）

```typescript
// ShiftFormInner内のviewMode管理
const [internalViewMode, setInternalViewMode] = useAtom(viewModeAtom);
const viewMode = props.viewMode ?? internalViewMode;
const setViewMode = props.onViewModeChange ?? setInternalViewMode;
```

#### 3. `ShiftBoardPage/index.tsx`
- `TitleTemplate` ラッパーを削除
- `useState` で `viewMode` を管理
- `viewMode` を `ShiftBoardHeader` と `ShiftForm` の両方に渡す

```typescript
// ShiftBoardPage
const [viewMode, setViewMode] = useState<ViewMode>("daily");

return (
  <Box>
    {/* PC */}
    <Box display={{ base: "none", lg: "block" }}>
      <ShiftBoardHeader
        // ... 既存props ...
        viewMode={viewMode}
        onViewModeChange={setViewMode}
      />
    </Box>
    
    {/* SP */}
    <Box display={{ base: "block", lg: "none" }}>
      <ShiftBoardSPHeader
        // ... 既存props ...
        viewMode={viewMode}
        onViewModeChange={setViewMode}
      />
    </Box>

    <ShiftForm
      // ... 既存props ...
      viewMode={viewMode}
      onViewModeChange={setViewMode}
      hideViewSwitcher={true}  // ヘッダー側で表示するため非表示
    />
    
    {/* Modal等 */}
  </Box>
);
```

#### 4. `ShiftBoardHeader/index.tsx`（PC用）
- `viewMode` と `onViewModeChange` propsを追加
- 左端に `← 戻る` リンクを追加（`Link to="/dashboard"`）
- SegmentGroup（日別/一覧タブ）を追加
- `提出状況` テキストを**削除**
- レイアウトを1行構成に変更（Flex、align="center"、justify="space-between"）

props型の変更：
```typescript
type ShiftBoardHeaderProps = {
  periodLabel: string;
  // submittedCount: number;  ← 削除
  // totalStaffCount: number; ← 削除
  confirmedAt: Date | null;
  onSave: () => void;
  onConfirm: () => void;
  isSaving?: boolean;
  viewMode: ViewMode;              // 追加
  onViewModeChange: (mode: ViewMode) => void; // 追加
};
```

レイアウトイメージ（PC）:
```
[← 戻る] [1/20(月)〜1/26(日)のシフト] ... [日別|一覧] ... [保存] [確定する]
                                                            [✓ 確定済み 3/28 23:15]
```

#### 5. `ShiftBoardSPHeader/index.tsx`（SP用）
- 同様に `viewMode` と `onViewModeChange` propsを追加
- 2行構成に変更:
  - 行1: `← 戻る` + 期間テキスト
  - 行2: SegmentGroup + 保存ボタン + 確定ボタン
- `提出状況` テキストを**削除**

#### 6. `Header/index.tsx`（グローバルヘッダー）
- 「シフト管理」テキストを `Link to="/dashboard"` に変更
- スタイル: 白文字のまま、ホバーで少し明るくなる程度

```tsx
import { Link } from "@tanstack/react-router";

// 変更前
<Text color="white" fontWeight="semibold">シフト管理</Text>

// 変更後
<Link to="/dashboard">
  <Text color="white" fontWeight="semibold" _hover={{ opacity: 0.8 }} cursor="pointer">
    シフト管理
  </Text>
</Link>
```

#### 7. `ShiftBoard/types.ts`
- `ShiftBoardHeaderProps` から `submittedCount` と `totalStaffCount` を削除
- `viewMode` と `onViewModeChange` を追加
- `formatSubmissionStatus` ヘルパー関数は不要になるので削除

## 変更しないもの

- `ShiftForm` 内部のガントチャート、日付タブ（DateNavigator等）は一切変更しない
- `ShiftForm` の `<Provider>` スコープやatom構造は変更しない
- ダッシュボード画面は変更しない
- 確定ダイアログのUI/ロジックは変更しない
- `_auth.tsx` のレイアウト構成（Header + pt="56px"）は変更しない

## 注意事項

- `TitleTemplate` は他のページ（ダッシュボード等）で引き続き使用されるため、コンポーネント自体は削除しないこと
- `ShiftForm` の後方互換性を維持すること（`viewMode` propsが渡されない場合は従来の内部管理で動作する）
- `← 戻る` のリンク先は `/dashboard` 固定（`window.history.back()` ではなく明示的なリンク）
- ボタンのラベル: 「シフトを送る」→「シフトを確定する」、「再送する」→「再確定する」に統一（情報設計書に準拠）
