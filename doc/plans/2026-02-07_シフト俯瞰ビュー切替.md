# シフト俯瞰ビュー切替 実装計画

**作成日**: 2026-02-07
**目的**: 既存ShiftOverviewを拡張し、日毎ビュー（ShiftTableTest）と俯瞰ビュー（ShiftOverview）の切替を導入する

---

## 1. 背景

- 現状は日毎タブ切替（DateTabs）で1日分のシフトのみ表示
- `src/components/features/Shift/ShiftOverview/` に俯瞰ビューの基本実装が既に存在
- 2つのビューを切り替えるUIが未実装

## 2. 確定方針

| 項目 | 決定 |
|------|------|
| セル表示 | `09:00-17:00`（分まで含む）→ セル幅70→90px |
| 充足度 | フッターのサマリー行のみ（%+出勤数） |
| SP対応 | スコープ外 |
| ビュー切替 | SegmentGroup（日毎 / 俯瞰） |

## 3. 改修対象ファイル

| # | ファイル | 変更内容 | 新規/拡張 |
|---|----------|----------|-----------|
| 1 | `ShiftOverview/constants.ts` | セル幅 70→90px | 拡張 |
| 2 | `ShiftOverview/StaffRow.tsx` | セル表示4種区別（時間/休/未/-）+ 未提出者強調 | 拡張 |
| 3 | `ShiftOverview/OverviewHeader.tsx` | 未提出者数バッジ追加 | 拡張 |
| 4 | `ShiftOverview/SummaryFooterRow.tsx` | 充足度サマリー行（%+出勤数/必要人数） | 新規 |
| 5 | `ShiftOverview/types.ts` | 充足度・必要人数関連の型追加 | 拡張 |
| 6 | `ShiftOverview/index.tsx` | SummaryFooterRow統合、未提出者ソート | 拡張 |
| 7 | 親コンポーネント（ビュー切替） | SegmentGroupで日毎↔俯瞰切替 | 新規（場所要確認） |
| 8 | `ShiftOverview/index.stories.tsx` | 新機能のStorybook更新 | 拡張 |

## 4. 実装ステップ

### Step 1: 定数・型の更新
- `constants.ts`: DATE_CELL_WIDTH を 70→90 に変更
- `types.ts`: 充足度関連の型追加（SummaryFooterRowProps等）

### Step 2: StaffRow の改善
- セル表示4種区別:
  - 勤務あり: `09:00-17:00`（gray.800）
  - 休み（提出済み＆希望なし）: `休`（gray.400）
  - 未提出: `未`（orange.400）+ 行背景 red.50
  - シフトなし: `-`（gray.300）
- 未提出スタッフの行を視覚的に強調（薄い赤背景）
- スタッフ名横に ⚠ アイコン（未提出時）

### Step 3: OverviewHeader の改善
- 右上に「未提出: N名」バッジ表示（Badge colorPalette="red" variant="subtle"）
- 未提出者がいない場合は非表示

### Step 4: SummaryFooterRow の新規作成
- テーブルフッターに充足度サマリー行を追加
- 日付ごとに:
  - 充足率（%表示）+ FILL_RATE_COLORS の背景色
  - 出勤数/必要人数（例: `3/5`）
- sticky bottom で固定
- ShiftTableTest の SummaryRow と同様の充足度カラーを再利用

### Step 5: ShiftOverview index.tsx の統合
- SummaryFooterRow をテーブルフッターに追加
- 未提出スタッフを最上部にソート（既存ソートロジック拡張）
- 未提出者数のカウントをOverviewHeaderに渡す

### Step 6: ビュー切替親コンポーネント
- 設置場所: ShiftTableTestの呼び出し元を確認して決定
- SegmentGroup（Chakra UI v3）で「日毎ビュー」「俯瞰ビュー」を切替
- 俯瞰ビュー → 日付セルクリック → 日毎ビューの該当日タブへ遷移
- 日毎ビュー → 俯瞰タブクリック → ShiftOverviewへ遷移

### Step 7: Storybook 更新
- 未提出スタッフを含むストーリー追加
- 充足度サマリー表示のストーリー追加

### Step 8: コード品質チェック
- `pnpm format`
- `pnpm lint`
- `pnpm type-check`
- `pnpm test`（既存テスト通過確認）

## 5. データフロー

```
親コンポーネント
├── viewMode: "daily" | "overview" (SegmentGroup)
├── selectedDate: string
├── [日毎ビュー] ShiftTableTest
│   └── DateTabs → selectedDate更新
└── [俯瞰ビュー] ShiftOverview（改善版）
    ├── 未提出バッジ（OverviewHeader）
    ├── 4種セル表示（StaffRow）
    ├── 充足度サマリー（SummaryFooterRow）
    └── onDateClick → viewMode="daily" + selectedDate更新
```
