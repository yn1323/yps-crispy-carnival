**作成日**: 2026-02-08

## 1. 設計決定まとめ

| 項目 | 決定 |
|------|------|
| 日毎ビューSP | カード型、縦スクロール |
| 俯瞰ビューSP | 日付別カード |
| 編集方式 | フル編集。カードタップ → BottomSheet で操作 |
| 時間選択UI | セレクトボックス（timeRange.unit 刻みの選択肢） |
| 日付切替 | ◀ 1/28(火) ▶ ナビ + スワイプ対応 |
| ポジション表示 | ミニバー（色付き比例表示） |
| Undo/Redo | ヘッダーにボタン常設 |
| 俯瞰カードタップ | 日毎ビューの該当日に遷移 |
| 充足度表示 | バー + 出勤数/必要数 テキスト |

### Step 2: MiniShiftBar.tsx

ポジションセグメントを時間に比例した色付きバーで表示するコンポーネント。

**仕様**:
- timeRange全体に対する比例表示（例: 9-22時の13時間幅に対してバー位置を計算）
- 各セグメントはポジションの `color` で塗りつぶし
- バー下に開始・終了時刻のラベル（xs サイズ）
- ポジションなしの場合: グレーの点線バー

**実装ポイント**:
- `minutesToPercent` でセグメントの left/width を % で算出
- timeRange.start ～ end を 100% として比例計算
- バー高さ: 20px（PC版と同じ）
- 希望時間バー: グレー破線ボーダーで背面に表示

### Step 6: ShiftEditSheet.tsx（最重要・最大工数）

BottomSheet ベースのシフト編集UI。

**仕様**:

#### ヘッダー
- タイトル: `{スタッフ名}のシフト  {M/D(ddd)}`
- 希望時間表示: `希望: HH:MM - HH:MM` / `希望: なし` / `未提出`

#### 割当ポジション一覧
- 各セグメント: ■(ポジション色) + ポジション名 + [開始時刻 Select] - [終了時刻 Select] + [×削除]
- 時刻 Select の選択肢: timeRange.start ～ timeRange.end を unit 刻みで生成
  - 例: unit=30, start=9, end=22 → `["09:00", "09:30", "10:00", ..., "22:00"]`
- 時刻変更時: 即座に反映（normalizePositions 経由）
- 削除ボタン: 該当セグメントを削除（deletePositionFromShift 使用）

#### ポジション追加
- ポジション Select: positions 配列から選択
- 開始時刻 Select + 終了時刻 Select
- 「追加」ボタン: paintPosition でセグメント追加 → normalizePositions
- 初期値: 開始=timeRange.start, 終了=timeRange.start+unit

#### フッター
- 「全削除」ボタン（赤、ポジション0件時 disabled）
- 「完了」ボタン（Sheet を閉じる）

**実装ポイント**:
- 既存 `BottomSheet` UIコンポーネントを使用（maxH="60vh" は十分）
- 時刻 Select は既存 `Select` UIコンポーネントを使用
- 編集操作は全て `onShiftUpdate(updatedShift)` で親に通知
- 正規化パイプライン: 既存 `normalizePositions` を使用
- breakPosition の特定: `positions.find(p => p.name === "休憩")` で休憩ポジションを取得
- 未提出スタッフへの初回配置: 新しい ShiftData を自動作成（PC版の useDrag 内の処理と同じ）

**時刻 Select の選択肢生成ヘルパー**:
```typescript
const generateTimeOptions = (timeRange: TimeRange) => {
  const options = [];
  for (let m = timeRange.start * 60; m <= timeRange.end * 60; m += timeRange.unit) {
    const h = Math.floor(m / 60);
    const min = m % 60;
    options.push(`${String(h).padStart(2, "0")}:${String(min).padStart(2, "0")}`);
  }
  return options;
};
```

### Step 7: ShiftDailyCardSP/index.tsx

日毎ビューSPのメインコンポーネント。

**仕様**:
- レイアウト: `VStack` で縦積み
  1. DateNavigator（日付ナビ + ソート）
  2. FulfillmentBar（充足度サマリー）
  3. スタッフカードリスト（VStack、gap=3）
- 選択日のシフトデータでフィルタリング
- ソートは親（ShiftViewSwitcherSP）から sortedStaffs を受け取る
- スワイプによる日付切替: コンテナに onTouchStart/onTouchEnd を設置

**実装ポイント**:
- `shifts.filter(s => s.date === selectedDate)` で当日分を抽出
- 各 StaffCard に対応する shift を渡す
- BottomSheet の開閉状態: `useBottomSheet` + 選択中スタッフ state
- カードタップ → `setSelectedStaffId` + `open()` → ShiftEditSheet 表示

**スワイプ処理**:
```typescript
const touchStartX = useRef(0);
const SWIPE_THRESHOLD = 50; // px

const handleTouchStart = (e: React.TouchEvent) => {
  touchStartX.current = e.touches[0].clientX;
};

const handleTouchEnd = (e: React.TouchEvent) => {
  const deltaX = e.changedTouches[0].clientX - touchStartX.current;
  if (Math.abs(deltaX) > SWIPE_THRESHOLD) {
    const currentIndex = dates.indexOf(selectedDate);
    if (deltaX > 0 && currentIndex > 0) {
      onDateChange(dates[currentIndex - 1]); // 右スワイプ → 前日
    } else if (deltaX < 0 && currentIndex < dates.length - 1) {
      onDateChange(dates[currentIndex + 1]); // 左スワイプ → 翌日
    }
  }
};
```

```
│       ├── 全削除 → positions = [] にクリア
│       └── 全操作後 → normalizePositions() → onShiftsChange()
│
└── [display:block/none] ShiftOverviewCardSP
      受取: shifts, dates, onDateClick
      内部保持: sortMode（親から受取）
      カードタップ → setSelectedDate + setViewMode("daily")
```

## 7. 注意事項・エッジケース

1. **ShiftEditSheet の正規化**: 時刻変更・追加・削除の全操作で `normalizePositions` を実行する。休憩自動挿入/マージが正しく動作すること
2. **未提出スタッフの初回編集**: ShiftData が存在しない場合、BottomSheet 内で新規 ShiftData を作成する処理が必要
3. **スワイプと縦スクロールの競合**: deltaX の閾値を適切に設定し、縦スクロール意図のスワイプを誤検知しないこと
4. **時刻 Select の開始/終了バリデーション**: 開始 >= 終了 の場合は追加ボタンを disabled にする
5. **BottomSheet の高さ**: 多数のポジションセグメントがある場合、スクロール可能であること（既存 BottomSheet の maxH="60vh" + overflowY で対応）
6. **俯瞰カードのスタッフ表示件数**: スタッフが多い場合、初期表示は3-5名にして「他 X名」で折りたたむことを検討
7. **充足度のない場合**: requiredStaffing が未設定の場合、出勤人数のみ表示（充足率バーは非表示）
