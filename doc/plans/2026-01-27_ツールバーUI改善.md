# シフト管理UI ツールバー改善計画

## 1. 背景・決定事項

### 課題
現在のツールバーは以下の問題がある：
- **ツールと色が混在**: ポジションボタンが「塗るモードON＋色選択」を兼ねており、消しゴムと同列に並ぶ
- **選択状態の視認性が低い**: ボタンのvariant(solid/outline)の違いだけで判別しにくい
- **グループ化されていない**: Undo/Redo、ポジション、消しゴムがフラットに並んでいる
- **モード解除方法が不明確**: 同じボタン再押下のトグルは発見しにくい
- **操作に迷いやすい**: リサイズ、コピペ等の隠れた操作が多い
- **ポジション数増加への耐性が低い**: ユーザーが任意追加でき最大10個になりうる

### UI/UX議論の決定事項

| 項目 | 決定 |
|------|------|
| レイアウト方針 | A案（ペイントリボン型）: 3グループに分離 |
| ツール構成 | 👆選択（デフォルト）/ 🖌割当 / 🧹消す の3ツール |
| ツール名 | 「割当」（業務ツールとしての明確さ重視） |
| ポジション多数時 | 2行グリッド（5×2）、見切れ時は横スクロール |
| Tips欄 | CLS防止で固定1行確保、選択状態に応じた操作ヒント表示 |
| 完了メッセージ | 不要（「✓ 塗りました」等は表示しない） |
| 消しゴムのドラッグ | 対応する（範囲消去） |
| 選択ツールのドラッグ | 横スクロール |

## 2. ツールバー最終仕様

### 2.1 レイアウト

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│ ┌──────────┐  ┌───────────────────────────┐  ┌────────────────────────────────┐ │
│ │ [⟲] [⟳] │  │ [👆選択] [🖌割当] [🧹消す] │  │ [🔵ホール] [🟠キッチン] [🟢レジ]│ │
│ │          │  │           ↑選択中          │  │ [⬜休憩]   [🟣洗い場]          │ │
│ │  履歴    │  │         ツール             │  │         ポジション             │ │
│ └──────────┘  └───────────────────────────┘  └────────────────────────────────┘ │
│                                                                                  │
│  💡 バーをクリックして詳細を確認。バーの端をドラッグでリサイズできます             │
└──────────────────────────────────────────────────────────────────────────────────┘
```

**グループ構造:**
- 各グループはBorderまたは背景色で視覚的に区切る
- 各グループ下部にラベル（「履歴」「ツール」「ポジション」）を小さく表示
- セパレータはborderLeft（現在のコードと同様のパターン）

### 2.2 3ツールの挙動

| | 👆 選択（デフォルト） | 🖌 割当 | 🧹 消す |
|---|---|---|---|
| クリック | ポップオーバー表示 | — | 1セグメント消去 |
| ドラッグ | 横スクロール | ポジション色を塗る | 範囲消去 |
| バー端ホバー | ew-resizeカーソル→リサイズ | — | — |
| ポジション欄 | 通常表示 | 選択必須（未選択時はTipsで案内） | グレーアウト（opacity下げる） |
| カーソル | default（端: ew-resize） | crosshair | crosshair |

**重要な挙動:**
- 常にどれか1つのツールが選択状態（nullなし）
- 「割当」→「消す」→「割当」の切替時、前回選択していたポジションが復帰する
- リサイズは「選択」モード時のみ可能

### 2.3 Tips欄の表示内容

| 状態 | Tips表示 |
|---|---|
| 👆 選択（何もホバーなし） | バーをクリックして詳細を確認。バーの端をドラッグでリサイズできます |
| 👆 選択（バー端ホバー中） | ドラッグでリサイズ。隣接するポジションは連動します |
| 🖌 割当（ポジション未選択） | ポジションを選んでください |
| 🖌 割当（ポジション選択済） | 🔵 ホール — シフト表の上をドラッグで割り当て |
| 🧹 消す | クリックで1つ消去。ドラッグで範囲消去 |

**CLS防止:** Tips欄は常に1行分の高さ（`minHeight`）を確保し、内容のみ切り替える

### 2.4 ポジション表示

- 基本: flex-wrapで自動折り返し（2行グリッド相当）
- 5個以下: 1行に収まる
- 6-10個: 2行に折り返し
- 見切れ時: overflow-x: autoで横スクロール可能に
- ワンクリックで選択（セレクトボックスは使わない）
- 「消す」ツール選択時: ポジション欄全体をopacity: 0.4 + pointer-events: noneでグレーアウト

### 2.5 凡例

- 「┈┈┈ スタッフ希望時間」はTips欄の右端に配置（Tips行と統合して縦スペース節約）

## 3. 型定義の変更

### 現在
```typescript
export type ToolSelection = PositionType | "eraser" | null;
```

### 変更後
```typescript
// ツールモード（常にどれか1つ）
export type ToolMode = "select" | "assign" | "erase";

// 選択状態
// - toolMode: 現在のツール
// - selectedPositionId: 割当モードで使うポジション（erase/select時もnullにせず保持）
```

`ToolSelection`型を廃止し、`ToolMode`と`selectedPositionId`を分離する。
