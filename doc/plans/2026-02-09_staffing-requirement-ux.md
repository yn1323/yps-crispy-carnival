# StaffingRequirement UX改善プラン

## Context

StaffingRequirement（旧StaffingMatrix）は「必要人員設定」機能。店舗の曜日×時間帯×ポジションごとに「何人必要か」を設定する画面。シフト作成の前段階として、店舗の人員要件を定義する役割を持つ。

### リネーム: StaffingMatrix → StaffingRequirement
- ディレクトリ: `src/components/features/Shift/StaffingMatrix/` → `StaffingRequirement/`
- コンポーネント名: `StaffingMatrix` → `StaffingRequirement`
- ページ名: `StaffingSettingsPage` → そのまま（設定ページとして意味が通る）
- DBテーブル: `requiredStaffing` → そのまま（既存データとの互換性維持）

**問題**: 現在の実装は1セルずつ手入力で、14時間×4ポジション=56セル/日。週7日で392セル。操作効率・情報の見通し・初期設定フローに改善の余地がある。

---

## 現在のコード上の問題点（先に修正すべき）

| 問題 | 詳細 |
|------|------|
| StaffingTable二重実装 | `index.tsx`内のインライン版と`StaffingTable/`サブコンポーネント版が重複 |
| DAY_LABELS三重定義 | `index.tsx`, `DayTabs/`, `CopyModal/`の3箇所に同じ定義 |
| generateMockStaffing重複 | `AIGenerateForm/`と`RegenerateModal/`にコピペ |
| 型定義の散在 | `RequiredStaffingType`等が複数ファイルで再定義 |
| SetupWizard未接続 | 作ったが使われていない |

## UX改善提案

### Phase 1: コア効率化（Must）

#### 1-1. ステッパーボタン（Input → [-] N [+]）
- **解決**: 小さなInput欄への数値タイピングは特にSPで困難
- **操作**: タップで±1、長押しで連続増減
- **複雑度**: 低
- **対象**: `StaffingTable/`, `StaffingRequirement/index.tsx`内インライン版

#### 1-2. サマリーバー
- **解決**: 設定の全体像が見えない
- **表示**: 週合計人時 / ピーク時間帯 / 設定済み曜日数
- **複雑度**: 低
- **新規**: `SummaryBar/index.tsx`

#### 1-3. SetupWizard接続
- **解決**: 初回ユーザーが空テーブルで途方に暮れる
- **操作**: データ未設定時 → SetupWizard表示、完了後 → 通常編集画面
- **複雑度**: 低〜中
- **対象**: `StaffingSettingsPage/index.tsx`

### Phase 2: 情報アーキテクチャ（Should）

#### 2-1. 週間俯瞰ヒートマップ
- **解決**: 1曜日ずつしか見れず週全体が把握できない
- **操作**: 7列(曜日)×N行(時間帯)のグリッド、色の濃さで人数表現、クリックで日別編集へ
- **複雑度**: 中
- **新規**: `WeeklyHeatmap/index.tsx`

#### 2-2. ビューモード切替（Progressive Disclosure）
- **解決**: 全情報が一度に表示されて圧倒される
- **操作**: 「週間俯瞰」「日別編集」のタブ切替
- **複雑度**: 中
- **対象**: `StaffingRequirement/index.tsx`にモード状態追加

#### 2-3. SP向けアコーディオン
- **解決**: SPで14時間分のカードスクロールが長い
- **操作**: 朝/ランチ/午後/ディナー/夜のグループに折りたたみ、ヘッダーに小計表示
- **複雑度**: 中
- **新規**: `MobileAccordionView/index.tsx`（現StaffingCardListの代替）

#### 2-4. SP向けクイックナビ
- **解決**: ピーク時間帯へのジャンプ手段がない
- **操作**: 画面下部にフローティングチップ（朝|ランチ|午後|ディナー|夜）、タップでスクロール
- **複雑度**: 低
- **新規**: `QuickNavBar/index.tsx`

### Phase 3: スマート機能（Should）

#### 3-1. タイムプリセット
- **解決**: よくあるパターンをワンタップで
- **操作**: 「ランチ多め」「ディナー多め」「全時間均等」「リセット」チップ
- **複雑度**: 低
- **新規**: `PresetChips/index.tsx`

#### 3-2. 行/列ヘッダーのクイック設定
- **解決**: 「12:00台は全ポジション3人」を素早く
- **操作**: 時間帯ヘッダーやポジションヘッダーをクリック → ポップオーバーでステッパー
- **複雑度**: 中
- **対象**: `StaffingTable/index.tsx`

#### 3-3. パターン自動検出
- **解決**: 月〜金が同じ設定なら「平日パターン」として認識
- **操作**: 「平日パターン（月-金）」ラベル表示、編集時に「他の平日にも適用？」確認
- **複雑度**: 中
- **新規**: `PatternIndicator/index.tsx`, `utils/detectPatterns.ts`

### Phase 4: 発展（Could）

#### 4-1. ギャップ可視化（必要人員 vs 実際シフト）
- **解決**: 要件を設定しても実際のシフトとのズレが見えない
- **操作**: ShiftForm側のSummaryRowに実データの必要人数を接続
- **複雑度**: 高
- **対象**: `SummaryRow.tsx`, `SummaryFooterRow.tsx`、新Convexクエリ

#### 4-2. 曜日比較ビュー
- **解決**: 月曜と金曜の差を一目で確認したい
- **操作**: 2〜3日を選んでサイドバイサイド表示、差分ハイライト
- **複雑度**: 高
- **新規**: `ComparisonView/index.tsx`
