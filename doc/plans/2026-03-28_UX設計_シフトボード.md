# YPS v3 UX設計：シフトボード ワイヤーフレーム仕様

> **ステータス**: ワイヤーフレーム作成用
> **作成日**: 2026-03-28
> **前提ドキュメント**: 2026-03-26_情報設計書.md, 2026-03-28_セキュリティ・認証設計.md
> **対象デバイス**: PC（閉店後バックヤードでの利用想定）
> **画面サイズ**: 1280px幅を基準

---

## 1. 概要

シフトボードはYPSの**心臓部**。田中さんが日曜の閉店後にPCを開いて、来週のシフトを組む画面。
ダッシュボードの募集カードから「シフトボードを開く」ボタンで遷移する。

**2つのビューをタブで切り替える構成：**
- **日別ビュー（ガントチャート）** = シフトを組む作業場
- **一覧ビュー** = 全体の確認と確定

## 2. 作成する画面パターン一覧

| # | パターン | ビュー | 説明 |
|---|---------|--------|------|
| 1 | 日別ビュー：通常状態 | 日別 | 8/10人提出済み。希望バー＋実シフトバーが表示されている状態 |
| 2 | 日別ビュー：ドラッグ中 | 日別 | スタッフのシフトバーをドラッグで伸縮している瞬間 |
| 3 | 一覧ビュー：通常状態 | 一覧 | 全日程×全スタッフの時間が一覧表示。合計時間あり |
| 4 | 一覧ビュー：確定済み状態 | 一覧 | 確定ボタン押下後。「確定済み（3/28 23:15）」ステータスが出ている |

## 3. 共通ヘッダー（両ビュー共通）

```
┌─────────────────────────────────────────────────────────────┐
│  ← ダッシュボード    1/20(月)〜1/26(日) のシフト            │
│                      提出状況: 8/10人完了                    │
│                                                             │
│  [日別]  [一覧]                     [保存] [シフトを確定する]│
└─────────────────────────────────────────────────────────────┘
```

### 各要素の説明

| 要素 | 内容 | 備考 |
|------|------|------|
| ← ダッシュボード | パンくず/戻るリンク | ダッシュボードへ戻る |
| 期間タイトル | 「1/20(月)〜1/26(日) のシフト」 | この募集の対象期間 |
| 提出状況 | 「提出状況: 8/10人完了」 | テキスト。全員揃ったら「✓ 全員提出済み」に変わる |
| ビュータブ | [日別] [一覧] | アクティブなタブがハイライト |
| 保存ボタン | セカンダリボタン | 下書き保存。押すと「保存しました」トースト |
| 確定ボタン | プライマリボタン（目立つ色） | 確認ダイアログを挟んでから確定 |

### 確定後の変化
- 確定ボタンの下に「確定済み（3/28 23:15）」テキスト表示
- ボタンのラベルが「再確定する」に変わる
- 編集は引き続き可能

## 4. パターン1：日別ビュー（通常状態）

### レイアウト

```
┌─────────────────────────────────────────────────────────────┐
│  [共通ヘッダー]                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  日付タブ:  [1/20月] [1/21火] [1/22水●] [1/23木] ... [1/26日]│
│                                                             │
│  時間軸:     9:00  10:00  11:00  12:00  13:00 ... 21:00     │
│  ─────────────────────────────────────────────────────────  │
│  鈴木太郎    |░░░░░░░░░░░░░░|                               │
│              |■■■■■■■■■■■■■■|                               │
│  ─────────────────────────────────────────────────────────  │
│  佐藤花子           |░░░░░░░░░░░░░░░░░|                     │
│                     |■■■■■■■■■■■■■■■■■|                     │
│  ─────────────────────────────────────────────────────────  │
│  田中次郎    |░░░░░░░░░░░░░░░░░░░░░|                        │
│  （未提出）  |                      |  ← バーなし           │
│  ─────────────────────────────────────────────────────────  │
│  山田美咲                  |░░░░░░░░░░░░░|                  │
│                            |■■■■■■■■■■■■■|                  │
│  ─────────────────────────────────────────────────────────  │
│  高橋翔太    |░░░░░░░░░░░░░░|                               │
│              |■■■■■■■■■|     ← 希望より短く調整された       │
│  ...                                                        │
└─────────────────────────────────────────────────────────────┘
```

### 要素詳細

**日付タブ**
- 募集期間の全日付をタブで表示（例：月〜日の7日間）
- 選択中の日付がアクティブ表示
- 曜日＋日付で表示（「1/22 水」）
- 土日は色を変える（薄い背景色）

**縦軸：スタッフ列**
- 左端固定幅（約120px）
- スタッフ名を表示
- 未提出のスタッフは名前の下に「未提出」ラベル（赤系の小さなバッジ）
- スタッフの並び順：名前順（50音）

**横軸：時間軸**
- 9:00〜21:00（店舗の営業時間に基づく。MVPでは固定値）
- 30分単位のグリッド線（薄い縦線）
- 1時間ごとに時刻ラベル

**希望バー（░）**
- 薄い色（ストライプまたはドット柄）
- スタッフが提出した希望時間帯を表す
- **常に表示**（実シフトバーの背後に見える）
- ドラッグ不可（表示のみ）

**実シフトバー（■）**
- しっかりした色（塗りつぶし）
- 初期状態では希望バーと同じ位置・長さ（希望がそのまま反映）
- **ドラッグで左右に伸縮可能**（開始時間・終了時間の調整）
- **ドラッグで上下に移動不可**（スタッフの入れ替えはMVP外の可能性）
- バーの上に時間テキスト表示（「10:00-18:00」）

**未提出スタッフの行**
- 希望バーなし（空の行）
- 実シフトバーもなし（または管理者が手動で配置可能）
- 「未提出」ラベルが目立つ

## 5. パターン2：日別ビュー（ドラッグ中）

パターン1と同じレイアウト。以下が変化する：

- ドラッグ中のバーがハイライト（影またはアウトライン）
- バーの端をドラッグすると、スナップ表示（30分単位にスナップ）
- ドラッグ中のバー上にリアルタイムで時間テキスト更新（「10:00-17:30」→「10:00-18:00」）
- 希望バーとのずれが視覚的にわかる（希望より短い/長い/ずれている）

### レイアウト

```

│  [共通ヘッダー]                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│           │  月   │  火   │  水   │  木   │     │  日   │    │
│  ─────────┼───────┼───────┼───────┼───────┼─────┼───────┼────│
│  鈴木太郎 │10-18  │  休   │10-18  │10-14  │     │  休   │24h │
│  佐藤花子 │  休   │11-19  │11-19  │  休   │     │11-19  │24h │
│  田中次郎 │10-18  │10-18  │  休   │10-18  │     │  休   │24h │
│  ⚠        │       │       │       │       │     │       │    │
│  山田美咲 │14-21  │  休   │14-21  │14-21  │     │14-21  │28h │
│  高橋翔太 │10-15  │10-15  │  休   │10-15  │     │10-15  │20h │
│  ...      │       │       │       │       │     │       │    │
│  ─────────┼───────┼───────┼───────┼───────┼─────┼───────┼────│
│  日別人数 │  7人  │  6人  │  5人  │  7人  │     │  4人  │    │
└─────────────────────────────────────────────────────────────┘
```

### 要素詳細

**テーブル構造**
- 横軸：日付（募集期間の全日）＋右端に「合計」列
- 縦軸：スタッフ名
- 土日の列は薄い背景色で区別
- 最下行に「日別人数」（その日に出勤するスタッフ数）

**セルの表示**
- シフトあり → 「10:00-18:00」（時間テキスト）
- シフトなし → 「休」（グレーテキスト）
- 未提出スタッフ → セルが空 or 「—」

**合計列**
- そのスタッフの期間合計労働時間
- 「24h」のように表示

**未提出スタッフ**
- スタッフ名の横に ⚠ アイコン
- 行全体が薄くなるか、背景色で警告

**日別人数行**
- テーブル最下部に固定表示
- 各日のスタッフ出勤数を表示
- 人数が少ない日は赤字 or 警告色（※将来的に閾値設定できるようにする）

│                                     │
│   全スタッフにメールで               │

## 9. 操作フロー（田中さんのシナリオ）

```
ダッシュボード → 「シフトボードを開く」
  │
  ├→ 一覧ビューで提出状況を確認
  │   「8/10人提出済みか。田中次郎と…がまだだな」
  │
  ├→ 日別ビューに切り替え
  │   月曜から順番に見ていく
  │
  ├→ 月曜：希望バーが並んでいる。問題なさそう → そのまま
  ├→ 火曜：佐藤さんの終了時間を19:00→18:00に変更（バーの右端ドラッグ）
  ├→ 水曜：人が少ない。一覧で確認 → 一覧ビューに切り替え
  │
  ├→ 途中で疲れた → 保存ボタン → 「保存しました」トースト → 翌日に持ち越し
  │
  ├→（翌日）一覧ビューで全体確認
  │   合計時間のバランスを確認
  │
  ├→ 「シフトを確定する」ボタン
  ├→ 確認ダイアログ → 「確定する」
  ├→ 「✓ 確定済み（3/28 23:15）」表示
  │   全スタッフにメール送信
  │
  └→ ダッシュボードに戻る
```

## 10. デザインノート

| 項目 | 方針 |
|------|------|
| 色 | Chakra UIデフォルトテーマベース。希望バーは薄い青系、実シフトバーは濃い青系 |
| フォント | Chakra UIデフォルト |
| 保存ボタン | セカンダリ（outline or ghost）。確定ボタンと明確に差別化 |
| 確定ボタン | プライマリ（colorScheme="blue" or "green"）。最も目立つ |
| レスポンシブ | MVPではPC専用。SP対応はPhase 2以降 |
| アニメーション | ドラッグ時のスナップ感のみ。遷移アニメは不要 |
| 未提出の警告色 | orange or red系の小さなバッジ |

## 11. この仕様で確定していること / 未確定のこと

### 確定済み

- 2ビュー（日別/一覧）のタブ切り替え構成
- 希望バー（点線/薄色）＋実シフトバー（塗りつぶし）の2レイヤー
- 手動保存ボタン＋確定ボタンの2段階
- 確定後も編集可能＋再確定で再通知
- セキュリティ方針（セキュリティ・認証設計書に詳述）

### 未確定（モック作りながら詰める）

- バーの色の具体的な値
- 未提出スタッフの行の見せ方の詳細
- 日別人数の閾値（何人以下で警告色にするか）
- ドラッグ操作の詳細なインタラクション
