## 0. 背景と目的

現在の `src/routes/index.tsx` は「シフトリ」のロゴとログインボタンだけのダミー状態。

当初は「身内に触ってもらう前提の1枚ペラ」として設計していたが、**実際には身内テスターがいないため、たまたまLPに流入する一般ユーザー（田中さんペルソナ：40代飲食店店長、Excel+LINE利用、毎週2-3時間のシフト作成に苦しむ）向けに再設計**する。

### LPの役割（スコープを絞る）

1. **「毎週のExcel転記 2〜3時間」が消えると 5 秒で伝える** — 田中さんのP1痛みに即座に刺す
2. **ログイン／サインアップ導線（Clerk）**
3. **法務系リンク（規約・プライバシーポリシー）**

### スコープから意識的に外すもの

- **料金プラン訴求**：「無料」はFAQに1箇所書くだけ、ヒーローで煽らない。Phase 4で有料化予定との整合性のため
- **ユーザーの声・事例**：まだ実績が無い
- **大規模アニメーション／動画**：オーバースペック
- **「Hero → Features → Testimonials → CTA」的な王道LP**：個人開発にふさわしくない
- **フィードバック導線（目安箱）**：LP計画から切り出し、別タスクで扱う（優先度高）

> **UX駆動で1枚ペラに割り切る。** ただし一般ユーザー流入を想定するため、「痛みへの訴求」と「信頼構築」のバランスは取る。

### 前提として受け入れるリスク

- **身内テスターがいない** → 事前のコピー検証は**自己レビュー（毎日読み返し）のみ**
- **流入経路が未定**（SEO/SNS/広告/etc） → LPはまず「流入があったときに刺さる状態」で置いておく
- **フィードバックチャネルがLPから分離** → リリース後のフィードバック取得は別タスクで構築（優先度高）

## 2. デザイン仕様（pencil.dev 入稿用）

> このセクションは **pencil.dev にそのまま投入してデザイン案を生成できる粒度** で書いている。

### 2-1. デザインコンセプト

- **ターゲット**：40代飲食店店長「田中さん」。深夜、店を閉めた後に疲れてPCを開いている状態
- **トーン**：派手さより信頼感。個人開発らしい誠実さ
- **質感**：大人っぽく、落ち着いたモノトーン基調。絵文字は一切使わない
- **雰囲気ワード**：静か、誠実、親切、実直

### 2-2. カラーパレット

Chakra UI v3 デフォルトトークンを使う。独自カラーは定義しない。

| 役割 | トークン | 用途 |
|---|---|---|
| 基調（中性色） | `gray` スケール | テキスト、背景、境界線 |
| アクセント | `teal` もしくは `blue`（1色のみ選択） | CTAボタン、見出しアクセント、アイコン |
| 背景 | `white` | メイン背景 |
| セクション区切り | `gray.50` | 奇数セクションの背景（薄く） |
| 本文テキスト | `gray.800` | 通常テキスト |
| サブテキスト | `gray.600` | サポート文、補足 |

- **ダークモード**：未対応（ライトモード固定）
- **グラデーション**：原則使わない

### 2-3. タイポグラフィ

Chakra UI v3 デフォルトフォント（`system-ui` 系）を使用。

| 要素 | モバイル | デスクトップ | ウェイト |
|---|---|---|---|
| H1（ヒーロー大見出し） | `3xl` | `5xl` | `bold` |
| H2（セクション見出し） | `2xl` | `3xl` | `bold` |
| H3（ポイント・Step見出し） | `lg` | `xl` | `bold` |
| 本文 | `md` | `lg` | `normal` |
| サブコピー・補足 | `sm` 〜 `md` | `md` | `normal`、`color=gray.600` |

- **行間**：本文は `lineHeight="1.7"` 程度でゆったり
- **H1 のみ `lineHeight="1.3"` で詰める**（大見出しは引き締める）

### 2-4. レイアウト・スペーシング

- **カラム**：1カラム固定
- **コンテナ最大幅**：`maxW="container.md"`（約 768px）、中央寄せ
- **モバイル左右パディング**：`px={4}`（16px）
- **デスクトップ左右パディング**：`px={8}`（32px）
- **セクション間縦マージン**：
  - モバイル：`py={12}`（48px）
  - デスクトップ：`py={20}`（80px）
- **ヒーロー内要素の縦スペース**：`spacing={4}` 〜 `spacing={6}`
- **ポイント・Step のカード間隙**：`gap={6}`（モバイル）、`gap={8}`（デスクトップ）

### 2-5. アイコン

- **ライブラリ**：`lucide-react` v0.383.0（既存導入済み）
- **スタイル**：線画モノトーン、`strokeWidth={1.75}`
- **カラー**：アクセント色（teal or blue） or `gray.700`
- **サイズ**：
  - ポイントセクション：`32px`
  - Stepセクション：`24px`
- **装飾**：アイコン背景に `gray.100` の丸（`borderRadius="full"`、`padding={3}`）を敷く。これで **絵文字感を出さずに視認性を上げる**
- **使用するアイコン一覧**：
  - ポイント1：`UserCheck`
  - ポイント2：`Sunrise`
  - ポイント3：`Bell`
  - Step1：`Store`
  - Step2：`UserPlus`
  - Step3：`CalendarPlus`

### 2-6. CTAボタン

#### メインCTA「さっそく試す」

- サイズ：`size="lg"`
- 幅：
  - モバイル：`width="full"`
  - デスクトップ：`width="auto"`、`minW="240px"`
- カラー：`colorPalette={accentColor}`（teal または blue）
- 角丸：`rounded="md"`
- 高さ：約 `56px`（親指で押しやすいサイズ）
- フォントサイズ：`lg`、`fontWeight="bold"`

#### サブCTA「ログイン」

- サイズ：`size="lg"`
- スタイル：`variant="ghost"` もしくは `variant="outline"`
- 配置：
  - モバイル：メインCTAの直下に縦並び
  - デスクトップ：メインCTAの右横に並び

#### ヘッダー右上のログイン

- サイズ：`size="sm"` もしくは `size="md"`
- スタイル：`variant="ghost"`
- テキストのみ、派手さ不要

### 2-7. レスポンシブ基準

- **モバイルファースト**：iPhone SE（375×667）を最小基準
- **ブレイクポイント**：Chakra UI v3 デフォルト
  - `md`: 768px
  - `lg`: 992px
  - `xl`: 1280px
- **First View制約**：iPhone SE 375×667 で**ヒーローH1・サブコピー・メインCTAの下端**までが**スクロールなしで見える**こと
- **スクロール挙動**：通常スクロール、パララックスなど凝った演出は不要

### 2-8. アニメーション

- 原則なし
- 使うとしても Chakra の `<Fade>` / `<SlideFade>` によるページロード時の軽いフェードイン程度
- CTAボタンは Chakra デフォルトのホバー（色の微変化）のみ

### 2-9. セクション間の区切り

- 隣接するセクションで背景色を交互に：`white` ↔ `gray.50`
- 区切り線（`border`）は使わない。色の違いだけで区切る
- シャドウも最小限（使うなら `shadow="sm"` まで）

### 2-10. アクセシビリティ要件

- フォーカスリング：Chakra デフォルト（`outline: 2px solid var(--focus-color)`）を必ず出す
- リンクは色だけでなく**下線**も付ける
- コントラスト比：WCAG 2.1 AA（本文 4.5:1 以上）
- セマンティックHTML：`<header>` / `<main>` / `<nav>` / `<section>` / `<footer>` / `<h1>` / `<h2>` / `<h3>` を適切に使う
- 画像を使う場合は `alt` 必須（現状はアイコンのみなので `aria-label` or 装飾扱い）

## 3. 技術方針

### 3-1. ルーティング配置（レイヤールール遵守）

**Before（旧方針）**：`routes/index.tsx` 内に Navigate 判定を書く

**After（新方針）**：

```
routes/index.tsx                → <IndexPage /> を呼ぶだけ（ロジック禁止）
  ↓
pages/IndexPage/index.tsx       → useUser でログイン判定 → 振り分け
  ├─ ログイン済み → <Navigate to="/dashboard" />
  └─ 未ログイン  → <LandingPage />
  ↓
features/LandingPage/index.tsx  → LP本体（useQuery/useMutation 禁止）
```

### 3-2. ディレクトリ構成（コロケーション）

```
src/components/pages/IndexPage/
├── index.tsx                 # ログイン判定＋振り分け
└── index.stories.tsx         # ログイン済み / 未ログイン の Story

src/components/features/LandingPage/
├── index.tsx                 # <LandingPage />
├── index.stories.tsx         # LP全体の Story
├── constants.ts              # コピー・リンク URL などの定数
└── sections/
    ├── HeroSection/
    │   ├── index.tsx
    │   └── index.stories.tsx
    ├── PointSection/
    │   ├── index.tsx
    │   └── index.stories.tsx   # Variants Story（VRT節約）
    ├── StepSection/
    │   ├── index.tsx
    │   └── index.stories.tsx   # Variants Story（VRT節約）
    ├── FaqSection/
    │   ├── index.tsx

    └── FooterSection/
        ├── index.tsx
```

- `features/LandingPage/` は `useQuery` / `useMutation` を使わない（純粋な紹介ページ）
- Atom（`selectedShopAtom` 等）も触らない

### 5-2. 既存ルートを壊さない

- `src/routes/_unregistered/welcome.tsx` は**このタスクでは触らない**
- `_unregistered` レイアウトも今回は変更しない
- ログイン済み時の `/dashboard` への Navigate は必ず残す

### 5-3. 避けるべきコピー（再確認）

- **「業界No.1」「最先端AI搭載」等の根拠のない煽り** → NG
- **未検証の数字**（「30秒」「10倍速」等） → NG（「登録30秒」はこのルールに引っかかるので削除済み）

- **競合ディスり** → NG
- **導入店舗数などの実績数字** → まだデータがないので出さない

### 5-4. コピー検証プロセス

> 身内テスターがいないため、自己レビューを徹底する。

- **最低ライン**：コピー確定後、最低1晩寝かせて翌朝に読み直す
- **推奨**：毎日読み返して、違和感を潰す

- **5秒テスト**：LPを開いて5秒で閉じ、何のアプリか思い出せるか自己テスト
