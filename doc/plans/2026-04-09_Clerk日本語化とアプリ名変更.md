初回リリースまでに「シフトリ」として日本語で表示させる必要がある。

15分で終わるので、リリースタスクの最初にサクッと潰す。**他のタスクに依存しない**。

## 2. Clerk Dashboard での設定変更

> ⚠️ これは UI 操作のためコード変更ではない。**Claude Code に任せず、手動で対応する**箇所。

### 2-1. アプリケーション名
1. Clerk Dashboard → 対象アプリケーションを選択
2. Settings → General → Application name を「シフトリ」に変更

### 2-2. ブランディング
1. Settings → Customization → Branding
2. アプリ名 / ロゴ / favicon を「シフトリ」ブランドに差し替え
3. ロゴは `public/logo192.png` / `public/logo512.png` と同じものを使用

### 2-3. Allowed origins
1. Settings → Domains
2. staging 用 Cloudflare Pages のドメインを追加
3. 本番ドメイン（取得後）も追加予定

### 2-4. Sign-up/Sign-in 画面の項目
- 念のため、表示される項目（Email, Password）に不要なものが無いか確認
- ソーシャルログイン（Google 等）を使うか今回の範囲では未定 → 別タスクで検討

## 3. フロント側の日本語化

### 3-1. パッケージ導入

Clerk の日本語ロケールは `@clerk/localizations` に含まれている。

```bash
pnpm add @clerk/localizations
```

### 3-2. `src/main.tsx` の修正

```tsx
import { ClerkProvider } from "@clerk/clerk-react";
import { jaJP } from "@clerk/localizations";
// ...

<ClerkProvider
  publishableKey={CLERK_PUBLISHABLE_KEY}
  localization={jaJP}
>
  {/* ... */}
</ClerkProvider>
```

### 3-3. （任意）アピアランスのカスタマイズ

シフトリのブランドカラー（teal）に合わせるなら `appearance` プロップも指定できる。

```tsx
<ClerkProvider
  publishableKey={CLERK_PUBLISHABLE_KEY}
  localization={jaJP}
  appearance={{
    variables: {
      colorPrimary: "#0d9488", // Chakra の teal.600 相当
    },
  }}
>
```

> ⚠️ 15分で終わらせたいので、**アピアランス調整は今回やらない**。デフォルトで出しておく。
> 「ブランド統一が気になる」ならリリース後に別タスクで対応する。

## 5. 注意点

### 5-1. バージョン差異
- `@clerk/clerk-react` と `@clerk/localizations` のメジャーバージョンを揃える
- インストール前に `package.json` の `@clerk/clerk-react` のバージョンを確認

### 5-2. Dashboard の設定は環境ごとに別
- Clerk はプロジェクト（Instance）単位で設定が独立している
- **dev / staging / production でそれぞれ設定する必要がある**
- 今回は staging のみでOK。本番リリース時に再度対応

### 5-3. メール通知テンプレート
- Clerk のメール認証時（マジックリンク）のテンプレートも英語のまま
- ただし現状のマジックリンクは Clerk ではなくシフトリ独自実装（`convex/email/`）のはず
- 影響範囲を `src/components/features/auths/`、`convex/email/` で念のため確認してから着手

### 5-4. `appearance` を盛りすぎない
- ブランドカラー調整は魅力的だけど、沼にハマる
- 初回リリースは **デフォルト + 日本語化のみ** で割り切る
