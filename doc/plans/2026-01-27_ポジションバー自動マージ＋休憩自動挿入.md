# ポジションバーの自動マージ＋休憩自動挿入

## 課題

1. **同一ポジションバーの重複・隣接**: 同じpositionIdのバーが重なる/隣接すると別々のバーが残る → 1本にマージすべき
2. **バー間の空白**: ポジションバー間に隙間ができると空白のまま → 休憩で自動的に埋めるべき

## 仕様

### マージルール
- 同じ`positionId`のバーが隣接(`end === start`)または重複(`end > start`)→ 1本にマージ
- 異なる`positionId`のバーは影響を受けない

### 休憩自動挿入ルール
- ポジションバー間の空白を「休憩」バーで自動的に埋める
- **最初のバーの前**と**最後のバーの後**には休憩を挿入しない（端は埋めない）
- 塗り・リサイズ・削除すべての操作後に適用

### 消しゴムの挙動
- **休憩以外を削除** → 空白は休憩で自動的に埋まる
- **休憩を削除** → 休憩の前のバーを延長して隙間を埋める（前のバーがなければそのまま削除）

### 処理順序（正規化パイプライン）
1. マージ（同一positionIdの隣接/重複バーを統合）
2. ギャップ埋め（バー間の空白に休憩を挿入）

### Step 1: `shiftOperations.ts` にユーティリティ関数を追加

#### 1-1. `mergeAdjacentPositions()`
同一positionIdのバーが隣接・重複していたら1本にマージ。

#### 1-2. `fillGapsWithBreak()`
バー間の空白を休憩で埋める。端（最初のバーの前、最後のバーの後）は埋めない。
休憩バーのIDは`break-{startMinutes}-{endMinutes}`で安定させる。

#### 1-3. `normalizePositions()`
マージ → ギャップ埋めをパイプラインで実行。

```typescript
export const normalizePositions = (params: {
  positions: PositionSegment[];
  breakPosition: PositionType;
}): PositionSegment[] => {
  const merged = mergeAdjacentPositions(params.positions);
  return fillGapsWithBreak({ positions: merged, breakPosition: params.breakPosition });
};
```

#### 1-4. `deletePositionFromShift()`
休憩削除と通常削除を分岐。休憩削除時は前のバーを延長。

### Step 2: `index.tsx` を修正

#### 2-1. 正規化ラッパー
`setShifts` をラップして、すべてのシフト更新時に`normalizePositions`を適用。

```typescript
const breakPosition = positions.find(p => p.name === "休憩") ?? null;

const setShiftsNormalized = useCallback((newShifts: ShiftData[]) => {
  if (!breakPosition) {
    setShifts(newShifts);
    return;
  }
  setShifts(newShifts.map(shift => ({
    ...shift,
    positions: normalizePositions({ positions: shift.positions, breakPosition }),
  })));
}, [setShifts, breakPosition]);
```

## 検証方法

1. Storybookで確認:
   - 同じポジションで隣接するバーを塗る → 1本にマージされること
   - 同じポジションのバーをリサイズして重ねる → 1本にマージされること
   - 異なるポジション間に隙間がある → 休憩が自動挿入されること
   - 休憩以外を消しゴムで削除 → 休憩で埋まること
   - 休憩を消しゴムで削除 → 前のバーが延長されること
   - Undo/Redoが正常に動作すること
