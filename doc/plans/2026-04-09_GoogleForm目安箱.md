# Google Form 目安箱＋LPリンク 実装計画

> **作成日**: 2026-04-09
> **目安工数**: 1時間
> **前提ドキュメント**: 2026-04-09_リリースタスク_インデックス.md, 2026-04-09_LP実装計画.md, 2026-04-09_利用規約プライバシーポリシー.md
> **対象**: ユーザーフィードバック受付フォーム

---

## 0. 背景

身内リリース後、**「使ってみて気になった点」を受け取る経路**が絶対に必要。
シフトリ本体に問い合わせ機能を作るのはオーバーキルなので、Google Form で代用する。

目安箱の URL は以下から参照される：
- LP のフッター「目安箱」リンク
- プライバシーポリシー／利用規約の問い合わせ先
- 初回ユーザーへの連絡メールの末尾

**一箇所で受ける → 散らからない** のが最大の狙い。

---

## 1. Google Form の設計

### 1-1. フォームの目的
「シフトリについての感想・バグ報告・要望を気軽に送ってもらう」

### 1-2. 質問項目（最小限）

| # | 質問 | タイプ | 必須 |
|---|-----|-------|-----|
| 1 | 今日はどんなことで連絡くれましたか？ | ラジオ | ✓ |
| 2 | 詳しく教えてください | 長文テキスト | ✓ |
| 3 | どの画面で起きましたか？（任意） | 短文テキスト | — |
| 4 | お名前（匿名OK） | 短文テキスト | — |
| 5 | 返信してもよければメールアドレス | メール | — |

### 1-3. 質問1の選択肢

- 🐛 バグ報告
- 💡 こうなったらいいのに（要望）
- 🤔 使い方がわからない
- ❤️ 感想・応援
- 📝 その他

> ⚠️ 質問を増やしすぎない。**多いとそもそも書いてもらえない**。5問で十分。

### 1-4. Form の設定

- [ ] 回答を Google Sheets に自動連携
- [ ] 新規回答時にメール通知をONにする
- [ ] 1人で複数回答可
- [ ] ログイン不要（匿名提出可）
- [ ] 最後のメッセージ「ご意見ありがとうございます！確認次第対応します。」

## 2. URL 設計

### 2-1. URL の貼り付け先

| 場所 | 備考 |
|------|------|
| LP フッター「目安箱」 | `target="_blank"` で別タブ |
| プライバシーポリシー：問い合わせ先 | 同上 |
| 利用規約：問い合わせ先 | 同上 |
| スタッフ招待メール（末尾） | 今回は対象外、リリース後検討 |
| リリース連絡メールの末尾 | テンプレ文に含める |

### 2-2. 短縮 URL

Google Form のデフォルト URL は長いので、以下のどちらかで短縮する。

- [ ] `forms.gle/xxxxxxxx` 形式（Google Form 純正の短縮）
- [ ] Cloudflare Workers でリダイレクト（`{{ドメイン}}/feedback` → Google Form）

**選択：純正の `forms.gle` で十分**。凝らない。

## 3. 実装ステップ

### Step 1: Google Form 作成（20分）
1. [docs.google.com/forms](https://docs.google.com/forms) で新規作成
2. タイトル「シフトリ 目安箱」
3. 説明文：
    ```
    シフトリを使ってみて気になったこと、要望、バグ報告などを気軽に送ってください！
    ひとつでも大歓迎です。
    ```
4. 質問を上記のとおり設定
5. 「送信」→ リンクコピー（`forms.gle/...`）

### Step 2: Sheets 連携 & 通知設定（10分）
1. Form の「回答」タブ → 「スプレッドシートに記録」
2. 新しいスプレッドシートを作成（名前：「シフトリ 目安箱 回答」）
3. Form の「︙」→ 「通知設定」→ 「新しい回答が送信されたとき」にメール通知ON

### Step 3: URL を各所に反映（20分）
1. `src/components/features/LandingPage/constants.ts` の `LANDING_LINKS.feedback` に URL を設定
2. `src/routes/_unregistered/terms.privacy.tsx` の問い合わせ先に URL 反映
3. `src/routes/_unregistered/terms.terms.tsx` の問い合わせ先に URL 反映
4. staging でクリックして開くことを確認

### Step 4: ダミー投稿でテスト（10分）
1. 自分で実際にフォームに投稿してみる
2. 受信メールが届くことを確認
3. Sheets に反映されることを確認

---

## 4. 運用ルール

### 4-1. 対応ポリシー

- **毎朝確認**：Sheets を開いて新しい回答をチェック（リリース直後の1週間）
- **24時間以内に返信**：メアドが入っている場合、一次返信する（内容の確認や感謝）
- **バグ報告**：P0/P1 ならすぐ GitHub Issue 化して対応、P2 は Issue 化のみ
- **要望**：気になったものはメモとして `doc/plans/YYYY-MM-DD_ユーザー声.md` 等で蓄積

### 4-2. 返信しない／控えるケース

- 返信先メアドが入っていない → そのまま
- 誹謗中傷や本人特定不可 → スルー
- 同じ人から連投 → 最後の回答のみ対応

### 4-3. 振り返り

リリース1週間後を目安に、集まった回答を `design:research-synthesis` スキルで合成 → 次のイテレーションに反映。

---

## 5. 注意点

### 5-1. メアドの扱い
- フォームでメアドを取得する以上、プライバシーポリシーに「お問い合わせフォームで取得したメールアドレスは返信目的のみに利用する」と明記する
- 今回 #3 のプライバシーポリシーに記載する
- Google Form 側のデフォルトの「メールアドレスを自動収集する」設定は**OFF**にする（ログイン必須になってしまうため）

### 5-2. 画像添付
- バグ報告はスクショ欲しいが、画像アップロードは Google ログイン必須になる
- 初回は「テキストのみ」で割り切る。スクショは別送で対応

### 5-3. 送信完了後の導線
- 完了画面のメッセージをカスタマイズ
- 「別の回答を送信」リンクはONにしておく（連続投稿しやすく）

### 5-4. スパム対策
- 匿名OK にするとスパムリスクはあるが、身内リリースなら実害は薄い
- 増えてきたら reCAPTCHA 付きの別ツールに移行
