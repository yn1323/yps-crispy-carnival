## 実装タスク

### Step 1: 認証ラッパー作成

**ファイル**: `convex/_lib/functions.ts`

`convex-helpers` の `customMutation` / `customQuery` を使い、以下を作成:
- `managerMutation`: Clerk認証 → users テーブルでユーザー取得 → shops テーブルで所属店舗取得。ctx に `user` と `shop` を注入
- `managerQuery`: 同上（読み取り用）

認証フローの要件:
1. Clerk の `ctx.auth.getUserIdentity()` で認証情報を取得
2. `users` テーブルから `clerkId` で検索
3. ユーザーが見つからない場合は `null` を返す（query）or エラーをthrow（mutation）
4. `shops` テーブルから `ownerId` でユーザーの店舗を取得
5. ctx に `{ user, shop }` を注入

### Step 2: ダッシュボード用 query 作成

**ファイル**: `convex/dashboard/queries.ts`

#### `getDashboardData` query
管理者のダッシュボードに必要な全データを1つの query で返す。

返り値:
```typescript
{
  shop: { name: string } | null,  // 店舗未登録ならnull
  recruitments: Array<{
    _id: Id<"recruitments">,
    periodStart: string,
    periodEnd: string,
    deadline: string,
    status: "open" | "confirmed",
    responseCount: number,       // ← shiftRequests から集計
    totalStaffCount: number,     // ← staffs から集計
  }>,
  staffs: Array<{
    _id: Id<"staffs">,
    name: string,
    email: string,
  }>,
}
```

集計ロジック:
- `responseCount`: その recruitment に紐づく `shiftRequests` の `staffId` のユニーク数
- `totalStaffCount`: その shop の `staffs`（`isDeleted: false`）の総数
- recruitments は `isDeleted: false` でフィルタ、作成日降順
- staffs は `isDeleted: false` でフィルタ

### Step 3: mutation 作成

#### `convex/setup/mutations.ts` — `createShop`
- 店舗名、営業開始時間、営業終了時間を受け取る
- `shops` テーブルに挿入
- `users` テーブルにも管理者レコードを作成（Clerkの情報から）
- 既に店舗を持っている場合はエラー

#### `convex/staff/mutations.ts` — `addStaffs`
- `entries: Array<{ name: string, email: string }>` を受け取る
- 一括で `staffs` テーブルに挿入
- shopId は認証ラッパーから取得（引数で渡さない）
- 同じメールアドレスが既に存在する場合はスキップまたはエラー

#### `convex/recruitment/mutations.ts` — `createRecruitment`
- `periodStart`, `periodEnd`, `deadline` を受け取る
- `recruitments` テーブルに挿入（status: "open", isDeleted: false）
- shopId は認証ラッパーから取得

  shiftRequests: QueryResult["shiftRequests"],

2. 動作確認（店舗作成→スタッフ登録→募集作成が一通り動くか）
3. **プロンプト②（シフトボード）** を実行
4. 動作確認（シフト表示→ドラッグ編集→保存→確定が動くか）

### プロンプトに追記すべき情報
各プロンプトをAIに投げる際、以下も一緒に渡すと精度が上がる:
- `convex/CLAUDE.md` の全文
- 該当するコンポーネントのソースコード（特に types.ts と現在の index.tsx）
- `convex/schema.ts` の全文

### プロンプト投げ先
- **Claude Code**: ファイル操作できるので最適。`@` でファイル参照しながら使える
- **Claude.ai**: コード全文をコンテキストに入れれば使えるが、ファイル操作は手動
- **Cursor / Windsurf**: プロジェクトのコンテキストが自動で入るので効率的
