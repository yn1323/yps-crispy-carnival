# 店舗設定変更 実装計画

## 概要

店舗名・シフト時間帯を初回登録後にも変更できるようにする。
併せて、シフト時間帯の変更が既存の募集を壊さないようスキーマを修正し、`@convex-dev/migrations` によるマイグレーション基盤を整備する。

---

## 1. Pencil.dev デザイン計画

### 対象ファイル

`doc/designs/dashboard.pen`（既存）に追加フレームとして作成。

### 作成するフレーム

#### A. ダッシュボード — 店舗情報表示（通常状態）

- ダッシュボード最上部、「シフト」セクションの上に店舗情報行を追加
- レイアウト：`店舗名（HH:MM〜HH:MM）` ＋ 右端に歯車アイコン（⚙️）
- PC / SP 両方のフレームを作成
- 例：`居酒屋たなか（14:00〜翌1:00）⚙️`
- テキストスタイル：店舗名は太字、時間帯はグレー寄り、歯車はグレーのアイコン

#### B. 店舗設定編集モーダル / BottomSheet

- 歯車タップで開く
- PC：Dialog（モーダル）、SP：BottomSheet（既存パターンに合わせる）
- フォーム内容：
  - 店舗名（テキスト入力）
  - シフト開始時間（セレクト）
  - シフト終了時間（セレクト）
- 「営業時間ではなく、仕込み・片付けを含めたスタッフが勤務する可能性のある時間帯を入力してください」の注釈（SetupStep1と同じ）
- ボタン：「保存する」
- フォームの初期値は現在の店舗設定

#### C. 注意事項

- 既存パターンに合わせる（`AddStaffForm` / `EditStaffForm` のモーダル構造を踏襲）
- バリデーションは `setup/schemas.ts` の `createShopSchema` を再利用

## 2. マイグレーション基盤の計画

### 2-1. `@convex-dev/migrations` 導入

```bash
npm install @convex-dev/migrations
```

#### セットアップ手順

1. `convex.config.ts` にコンポーネント登録

```ts
// convex/convex.config.ts
import { defineApp } from "convex/server";
import migrations from "@convex-dev/migrations/convex.config";

const app = defineApp();
app.use(migrations);
export default app;
```

2. `convex/migrations.ts` にランナーとマイグレーション定義を配置

```ts
// convex/migrations.ts
import { Migrations } from "@convex-dev/migrations";
import { components } from "./_generated/api";

export const migrations = new Migrations(components.migrations);

// 汎用ランナー（CLIから実行用）
export const run = migrations.runner();
```

3. 個別マイグレーションは同ファイルに `migrations.define()` で定義

### 2-2. マイグレーション運用ルール

| 項目 | ルール |
|------|--------|
| 定義場所 | `convex/migrations.ts` |
| 命名規則 | `{テーブル名}_{操作内容}` 例: `recruitments_addShiftTimes` |
| 実行方法（dev） | `npx convex run migrations:run` |
| 実行方法（prod） | デプロイ後に手動で `npx convex run --prod migrations:run` |
| 冪等性 | `migrateOne` 内で既に処理済みかチェックする |
| 順序依存 | `next` パラメータでチェイン可能 |

### 2-3. `convex/CLAUDE.md` への追記内容

「スキーマ変更とマイグレーション」セクションの「バッチ書き換えが必要なとき」を更新：

```markdown
### マイグレーション基盤

`@convex-dev/migrations` コンポーネントを使用。

- **定義ファイル**: `convex/migrations.ts`
- **実行**: `npx convex run migrations:run`（dev/prod共通、`--prod`フラグで環境切替）
- **進捗確認**: `npx convex run --component migrations lib:getStatus --watch`
- **キャンセル**: `npx convex run --component migrations lib:cancel '{name: "migrations:xxx"}'`

#### マイグレーションの書き方

```ts
// convex/migrations.ts
export const myMigration = migrations.define({
  table: "targetTable",
  migrateOne: async (ctx, doc) => {
    // 冪等チェック
    if (doc.newField !== undefined) return;
    await ctx.db.patch(doc._id, { newField: "default" });
  },
});
```

#### デプロイ手順（マイグレーション込み）

1. スキーマに `v.optional()` で新フィールドを追加
2. コード側はフォールバック付きで読み取り
3. `npx convex deploy` でスキーマ＆コードをデプロイ
4. `npx convex run migrations:run` でデータ埋め
5. 全件完了を確認後、`v.optional()` を外してスキーマを Narrow
6. 再デプロイ
```

## 3. 店舗名・シフト時間変更の実装計画

### 依存順序

```
Step 1: マイグレーション基盤
  ↓
Step 2: スキーマ変更 + マイグレーション
  ↓
Step 3: バックエンド（mutation / query 修正）
  ↓
Step 4: フロントエンド（店舗設定UI）
```

### Step 1: マイグレーション基盤セットアップ

- `npm install @convex-dev/migrations`
- `convex/convex.config.ts` 作成（コンポーネント登録）
- `convex/migrations.ts` 作成（ランナー定義）
- `convex/CLAUDE.md` 更新

### Step 2: スキーマ変更 + マイグレーション

#### スキーマ変更（`convex/schema.ts`）

```ts
recruitments: defineTable({
  // ...既存フィールド
  shiftStartTime: v.optional(v.string()), // 新規追加（Widen）
  shiftEndTime: v.optional(v.string()),   // 新規追加（Widen）
})
```

#### マイグレーション定義（`convex/migrations.ts`）

```ts
export const recruitments_addShiftTimes = migrations.define({
  table: "recruitments",
  migrateOne: async (ctx, doc) => {
    if (doc.shiftStartTime !== undefined) return; // 冪等
    const shop = await ctx.db.get(doc.shopId);
    if (!shop) return;
    await ctx.db.patch(doc._id, {
      shiftStartTime: shop.shiftStartTime,
      shiftEndTime: shop.shiftEndTime,
    });
  },
});
```

#### 実行手順

1. `npx convex dev` でスキーマデプロイ
2. `npx convex run migrations:run` でバックフィル
3. 全件完了を確認
4. `v.optional` を外して `v.string()` に Narrow
5. 再デプロイ

### Step 3: バックエンド

#### 3-1. `convex/dashboard/queries.ts` — shop データ拡張

```ts
// 変更前
shop: { name: shop.name }

// 変更後
shop: {
  name: shop.name,
  shiftStartTime: shop.shiftStartTime,
  shiftEndTime: shop.shiftEndTime,
}
```

#### 3-2. `convex/shiftBoard/queries.ts` — timeRange の参照元を変更

```ts
// 変更前
const startHour = Math.floor(timeToMinutes(shop.shiftStartTime) / 60);
const endHour = Math.ceil(timeToMinutes(shop.shiftEndTime) / 60);

// 変更後（フォールバック付き）
const startTime = recruitment.shiftStartTime ?? shop.shiftStartTime;
const endTime = recruitment.shiftEndTime ?? shop.shiftEndTime;
const startHour = Math.floor(timeToMinutes(startTime) / 60);
const endHour = Math.ceil(timeToMinutes(endTime) / 60);
```

#### 3-3. `convex/recruitment/mutations.ts` — 募集作成時にシフト時間をコピー

```ts
// createRecruitment の handler 内
const recruitmentId = await ctx.db.insert("recruitments", {
  shopId: ctx.shop._id,
  periodStart: args.periodStart,
  periodEnd: args.periodEnd,
  deadline: args.deadline,
  status: "open",
  isDeleted: false,
  shiftStartTime: ctx.shop.shiftStartTime,  // 追加
  shiftEndTime: ctx.shop.shiftEndTime,       // 追加

```

#### 3-4. 新規 `convex/shop/mutations.ts` — 店舗設定更新

```ts
export const updateShopSettings = managerMutation({
  args: {
    name: v.string(),
    shiftStartTime: v.string(),
    shiftEndTime: v.string(),
  },
  handler: async (ctx, args) => {
    // バリデーション: 終了 > 開始
    if (timeToMinutes(args.shiftEndTime) <= timeToMinutes(args.shiftStartTime)) {
      throw new ConvexError("終了時間は開始時間より後にしてください");
    }
    await ctx.db.patch(ctx.shop._id, {
      name: args.name,
      shiftStartTime: args.shiftStartTime,
      shiftEndTime: args.shiftEndTime,
    });
  },
});
```
