# シフト表ソート機能 - 実装計画

**作成日**: 2026-01-27
**ステータス**: 実装完了

---

## 1. 概要

ShiftTableTest のスタッフ行にソート機能を追加する。
テーブルヘッダー「スタッフ」列にドロップダウンを配置し、3種類のソートを切り替え可能にする。

---

## 2. ソート仕様

### 2.1 ソート種類

| ソート | キー | 説明 |
|--------|------|------|
| デフォルト | `default` | propsの順番そのまま（ソートなし） |
| シフト希望順 | `request` | 希望あり → `requestedTime.start`昇順 → `.end`昇順。希望なし/未提出は末尾 |
| 出勤順 | `startTime` | 割当済みの最早開始時間順 → 同一なら終了時間早い順。未割当は末尾 |

### 2.2 シフト希望順の詳細

| 順序 | 条件 | 説明 |
|------|------|------|
| 1 | `requestedTime !== null` | 希望あり → 開始時間昇順 → 終了時間昇順 |
| 2 | `!isSubmitted` | 未提出（まだ希望を出していない） |
| 3 | `isSubmitted && !hasRequest` | 希望なし（出勤不可） |

同一グループ内は名前順でソート。

### 2.3 出勤順の詳細

- 割当済み: その日の最も早いポジションセグメントの `start` 時刻で昇順
- 同一開始時刻: 終了時間が早い方が上
- 未割当（ポジションなし）: 末尾に配置（未提出 → 希望なし → 名前順）

### 2.4 スナップショット方式

- ソート選択時点のデータでソート順を決定し、以後は固定
- 日付タブ切替時は自動で再ソート（日ごとにデータが異なるため）
- 既に選択中のソートを再クリックしても再ソートが走る

## 3. UI設計

### 3.1 配置: テーブルヘッダー「スタッフ」列

```
┌──────────────────┬──────────────────────────────────┐
│ スタッフ     ↕   │  9時    10時    11時    ...      │
├──────────────────┤                                   │
│ ┌────────────────┐│                                   │
│ │ ✓ デフォルト   ││                                   │
│ │   シフト希望順 ││                                   │
│ │   出勤順       ││                                   │
│ └────────────────┘│                                   │
└──────────────────┴──────────────────────────────────┘
```

### 3.2 インタラクション

- ヘッダーセル全体がクリッカブル（`cursor: pointer`, hover時に `bg: gray.100`）
- クリック → ドロップダウンメニュー表示
- メニュー項目をクリック → ソート実行 & メニュー閉じる
- 選択中の項目にチェックマーク（✓）表示
- 選択中の項目を再クリック → 再ソート実行

### 3.3 アイコン

- `LuArrowUpDown`（react-icons/lu）をソートインジケータとして使用
