**作成日**: 2026-01-27
**ステータス**: 実装完了

## 1. 概要

ShiftTableTest のスタッフ行にソート機能を追加する。
テーブルヘッダー「スタッフ」列にドロップダウンを配置し、3種類のソートを切り替え可能にする。

---

## 2. ソート仕様

### 2.1 ソート種類

| ソート | キー | 説明 |
|--------|------|------|
| 名前順（デフォルト） | `name` | あいうえお順（localeCompare） |
| アクション優先順 | `action` | 希望あり(未割当) → 希望あり(割当済) → 未提出 → 希望なし |
| 出勤順 | `startTime` | 割当済みの最早開始時間順。未割当は末尾 |

### 2.2 アクション優先順の詳細

| 優先度 | 条件 | 状態 |
|--------|------|------|
| 1 | `isSubmitted && hasRequest && positions.length === 0` | 希望あり・未割当 |
| 2 | `isSubmitted && hasRequest && positions.length > 0` | 希望あり・割当済 |
| 3 | `!isSubmitted` | 未提出 |
| 4 | `isSubmitted && !hasRequest` | 希望なし |

同一優先度内は名前順でソート。

### 2.3 出勤順の詳細

- 割当済み: その日の最も早いポジションセグメントの `start` 時刻で昇順
- 未割当（ポジションなし）: 末尾に配置、名前順
- 同一開始時刻: 名前順

### 2.4 スナップショット方式

- ソート選択時点のデータでソート順を決定し、以後は固定
- 日付タブ切替時は自動で再ソート（日ごとにデータが異なるため）
- 既に選択中のソートを再クリックしても再ソートが走る

## 3. UI設計

### 3.1 配置: テーブルヘッダー「スタッフ」列

```
┌──────────────────┬──────────────────────────────────┐
│ スタッフ     ↕   │  9時    10時    11時    ...      │
├──────────────────┤                                   │
│ ┌──────────────┐ │                                   │
│ │ ✓ 名前順    │ │                                   │
│ │   アクション │ │                                   │
│ │   出勤順    │ │                                   │
│ └──────────────┘ │                                   │
└──────────────────┴──────────────────────────────────┘
```

### 3.2 インタラクション

- ヘッダーセル全体がクリッカブル（`cursor: pointer`, hover時に `bg: gray.100`）
- クリック → ドロップダウンメニュー表示
- メニュー項目をクリック → ソート実行 & メニュー閉じる
- 選択中の項目にチェックマーク（✓）表示
- 選択中の項目を再クリック → 再ソート実行

### 3.3 アイコン

- `LuArrowUpDown`（react-icons/lu）をソートインジケータとして使用
