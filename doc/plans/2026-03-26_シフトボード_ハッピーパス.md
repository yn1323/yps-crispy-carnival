# シフトボード ハッピーパス（MVP）

> 田中さん（居酒屋店長・スタッフ10名）が来週のシフトを組む理想の流れ。
> 全員希望提出済み、人数も足りてる、何のトラブルもないケース。

---

## 前提条件

- スタッフ全員が希望を提出済み
- 希望がそのまま初期配置されている（希望 = 実シフトの初期値）
- 人手不足なし

---

## ステップ一覧

### HP-01：シフトボードを開く【開始】

| 項目 | 内容 |
|------|------|
| 操作 | ダッシュボードの募集一覧から「来週(4/6〜4/12)」をクリック |
| 応答 | シフトボードが開く。デフォルトは一覧ビュー |
| 見えるもの | 横軸に7日分の日付、縦軸にスタッフ10名、各セルに希望時間帯、全員「提出済み」✅、右端に合計時間 |
| 田中さんの判断 | 「全員出してるな、よし組み始めよう」 |

### HP-02：日別ビューに切り替え【開始】

| 項目 | 内容 |
|------|------|
| 操作 | 「日別ビュー」タブをクリック |
| 応答 | ガントチャートに切り替わる。最初の日（月曜 4/6）が表示される |
| 見えるもの | 横軸：時間帯(9:00〜23:00)、縦軸：スタッフ、色付きバー（=希望の初期配置）、背景に点線（=希望の元の位置）、上部に日付タブ |
| 田中さんの判断 | 「月曜は…ランチ帯に3人いるな、OK」 |

### HP-03：バーの端をドラッグして時間調整【編集】

| 項目 | 内容 |
|------|------|
| 操作 | スタッフAのバー（10:00-18:00）の右端をドラッグして 10:00-15:00 に縮める |
| 応答 | バーがリアルタイムで縮む、30分刻みでスナップ、合計時間が即時更新、点線は元の位置のまま残る |
| 見えるもの | バー: 10:00-15:00（実シフト）、点線: 10:00-18:00（元の希望）、差分が一目で分かる |
| 田中さんの判断 | 「Aさん今週はランチだけでいいな」 |

### HP-04：ポップオーバーでその日のシフトを削除【編集】

| 項目 | 内容 |
|------|------|
| 操作 | スタッフCのバーをクリック → ポップオーバーで「この日は休み」をクリック |
| 応答 | バーが消える（点線は残る）、「休」表示に、合計時間が減る |
| 見えるもの | Cさんの行にバーがなく、点線だけ残っている |
| 田中さんの判断 | 「Cさんは希望出してるけど、今週水曜は人足りてるから休みでいい」 |

### HP-05：次の日に移る【編集】

| 項目 | 内容 |
|------|------|
| 操作 | 上部の日付タブ「4/7 火」をクリック |
| 応答 | 火曜のガントチャートに切り替わる |
| 見えるもの | 月曜と同じレイアウトで火曜のデータ |
| 田中さんの判断 | 「火曜は…ディナー帯を見よう」 |

### HP-06：残りの日でも繰り返す【編集】

> HP-03〜05を水〜日曜でも繰り返す。
> ⚡ ここが作業時間の8割。この繰り返しが快適かどうかがUXの生命線。

### HP-07：一覧ビューで全体を確認【確認】

| 項目 | 内容 |
|------|------|
| 操作 | 「一覧ビュー」タブをクリック |
| 応答 | 週全体のテーブルビューに切り替わる |
| 見えるもの | 全スタッフ × 7日のマトリクス、各セルに確定した時間帯、右端に合計時間 |
| 田中さんの判断 | 「全体のバランスどうかな…Dさん週32時間は多すぎ？」 |

### HP-08：シフトを確定して送付【確定】

| 項目 | 内容 |
|------|------|
| 操作 | 「スタッフに送付する」ボタンをクリック |
| 応答 | 確認ダイアログ → 送付完了 → ステータスが「送付済み 4/5 23:15」に |
| 見えるもの | 送付済みラベルと日時、ボタンが「再送付する」に変わる |
| 田中さんの判断 | 「よし、これで完了！」 |

---

## MVPから除外したもの

| 機能 | 理由 | Phase |
|------|------|-------|
| バー丸ごと移動（中央ドラッグ） | 端のリサイズで代用可能。実装コスト高い割に被る | Phase 2 |
| 一覧ビューの調整セルマーク | 日別ビューの点線で差分は確認できる | Phase 2 |
