## 1. 今回のスコープ

### 対象（メインストーリー＝ハッピーパス）

田中さんが初ログインしてから、スタッフにシフト確定通知を送るまでの一本道。

> 初ログイン → スタッフ招待 → 募集作成（下書き） → スタッフに送信 → 希望提出 → シフトボード編集 → 確定通知

### 対象外（別ストーリーとして後回し）

| 内容 | 理由 |
|------|------|
| スタッフの編集・削除 | 例外系。メインストーリー確定後に設計 |
| 誤登録の対応 | 同上 |
| 募集のキャンセル | 同上 |
| 確定後の修正・再通知 | 同上 |
| 未提出者への催促フロー | 同上 |
| スタッフ側の登録エラー | 同上 |

## 2. 決定事項

### 2-1. 画面構成

| 項目 | 決定 | 備考 |
|------|------|------|
| ホーム画面 | ダッシュボードが起点 | 情報設計書どおり |
| メインデバイス | PC | 閉店後バックヤードでの利用想定 |
| ワイヤーフレーム | HTMLモック作成済み | PC確認は後日。構造と情報設計で議論 |

### 2-2. ダッシュボードの情報優先順位

1. **今アクティブな募集の状況**（提出率リング + 名前チップ + シフトボードへの導線）
2. **募集作成ボタン**（週1回の操作）
3. **過去の募集一覧**（参照用・振り返り）
4. **スタッフ管理**（低頻度操作）

### 2-3. 初回体験（オンボーディング）

**方針：B+Cハイブリッド**

- 入口（ダッシュボード初回）→ **B: ゴール直結型**（自分のデータで最速スタート）
- シフトボード初回 → **部分的にC**（操作チュートリアル要素）

**却下した案と理由：**

| 案 | 却下理由 |
|----|---------|
| A: ステップバイステップ誘導 | 「やらされ感」が出る。田中さんの目的は「シフト組みたい」であって「セットアップしたい」ではない |
| C単体: サンプルデータで体感 | ①田中さんは架空データに興味がない ②サンプル→本番の切り替えで認知負荷が高い ③自分のデータでの成功体験が遅れる |

### 2-4. スタッフ登録

**2ルート併用方式**

| ルート | 方法 | 想定シーン |
|--------|------|-----------|
| A（即時） | 田中さんがダッシュボードから名前+メアドを直接手入力 | 初回に2〜3人だけ登録してすぐ試したいとき |
| B（非同期） | 店舗単位の招待リンクをLINEグループに貼る。スタッフが自分で名前+メアドを入力して登録 | 残りのメンバーを一気に招待するとき |

**招待リンクの仕様：**
- マジックリンク方式（URLを知っていれば登録可能）
- **店舗単位で1つのURL**（LINEグループに1回貼るだけ）
- **有効期限あり**（一定期間後に無効化）
- セキュリティ：知らない人が登録された場合、田中さんがスタッフ一覧から削除で対応（MVP水準）

**非同期の罠への対策：**
初回にLINE招待だけだと、誰も登録するまで何もできず離脱するリスクがある。
→ ルートA（手入力）を併用することで、田中さんは2〜3人分だけ自分で入力して即座に試せる。

### 2-5. 募集作成の安全策

**下書き保存を挟む（即メール送信しない）**

- 募集作成 → 下書き状態で保存される
- 「スタッフに送信する」を明示的に押すまでメールは飛ばない
- 田中さんが「まだ試してるだけなのにメール飛んじゃった」という事故を防ぐ

**背景：**
田中さんはまだ本番運用の覚悟が決まっていない段階で触り始める。取り消せないアクション（メール送信）が不意に発生するのは恐怖。

### 2-6.「スタッフに何も要求しない」の再定義

**元の定義：**
> アカウント登録もアプリインストールも不要

**更新後の定義：**
> アカウント登録・アプリインストール不要は維持。初回の名前＋メアド入力1回は許容。

**理由：**
- 招待リンク方式により、田中さんのメアド手入力負荷が解消される
- スタッフ自身がメアドを入力することでタイポリスクも解消
- 「1回だけ名前とメアドを入れる」程度の負荷は、LINEで返信するのと同等

## 3. 初回フローの全体像

```
田中さん                                  スタッフ
  │                                         │
  ├─ 1. サインアップ（Clerk）                │
  ├─ 2. 店舗名を入力                        │
  ├─ 3. 空のダッシュボードが表示             │
  │     「スタッフを招待しましょう」          │
  │                                         │
  ├─ 4a. 2〜3人を手入力（ルートA）           │
  ├─ 4b. 招待リンクをLINEに貼る（ルートB）───┤
  │                                         ├─ 5. リンクを開く
  │                                         ├─ 6. 名前+メアド入力
  │                                         ├─ 7. 登録完了
  │   ← リアルタイムでスタッフが増えていく ←─┤
  │                                         │
  ├─ 8. 「募集作成」ボタン                   │
  ├─ 9. 期間・締切を設定 → 下書き保存        │
  ├─ 10.「スタッフに送信する」ボタン ────────┤
  │                                         ├─ 11. メール受信
  │                                         ├─ 12. リンクからシフト希望提出
  │   ← 提出状況がリアルタイムで更新 ←──────┤
  │                                         │
  ├─ 13. シフトボードで編集                  │
  ├─ 14.「確定通知を送る」ボタン ────────────┤
  │                                         ├─ 15. 確定シフト確認
  └─ 完了                                   └─ 完了
```
