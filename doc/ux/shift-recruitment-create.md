# シフト募集作成画面 UXデザイン

## 概要

シフト管理システムの管理者（店長/マネージャー）が、スタッフに対してシフト希望を募集するための画面。

## 対象ユーザー

- **プライマリ**: 店長、マネージャー
- **利用状況**: 通常1-2週間に1回程度の操作
- **利用デバイス**: スマートフォン（60%）、PC（40%）

---

## 1. ユーザーフロー

### メインフロー（ハッピーパス）

```
[シフト管理一覧]
       |
       v
[新規募集作成ボタン押下]
       |
       v
[募集作成画面表示]
       |
       +---> [募集期間入力] ---> [開始日選択] ---> [終了日選択]
       |
       +---> [締切日入力] ---> [締切日選択]
       |
       v
[入力内容確認]
       |
       +---> [バリデーションエラー] ---> [エラー表示・修正]
       |
       v
[作成ボタン押下]
       |
       v
[確認ダイアログ表示]
       |
       +---> [キャンセル] ---> [フォームに戻る]
       |
       v
[作成完了]
       |
       v
[シフト管理一覧に遷移（Toast表示）]
```

### エラーフロー

```
[入力エラー発生時]
       |
       +---> 終了日 < 開始日 ---> "終了日は開始日以降を指定してください"
       |
       +---> 締切日 >= 開始日 ---> "締切日は開始日より前を指定してください"
       |
       +---> 必須項目未入力 ---> "必須項目です"
       |
       v
[該当フィールドにフォーカス移動]
```

## 3. アクセシビリティ考慮事項

### WCAG 2.1 AA 準拠

| 項目 | 対応内容 |
|------|----------|
| **色のコントラスト** | テキスト 4.5:1以上、UI要素 3:1以上（gray.900/white = 12.6:1） |
| **フォーカス表示** | 全てのインタラクティブ要素にフォーカスリング表示 |
| **キーボード操作** | Tab移動、Enter/Space選択、Escape閉じる |
| **ラベル** | 全入力項目にlabel要素を関連付け |
| **エラー表示** | 色だけでなくテキストでもエラーを伝達 |
| **タッチターゲット** | 最小44x44px（モバイル） |

### キーボードナビゲーション

```
Tab順序:
1. 戻るボタン
2. 開始日入力
3. 終了日入力
4. 締切日入力
5. 作成ボタン

ダイアログ内:
1. キャンセルボタン
2. 作成するボタン
3. ESCキーでダイアログ閉じる
```

### ARIA属性

```tsx
// 必須項目
<Field.Root>
  <Field.Label id="startDate-label">
    開始日 <span aria-hidden="true">*</span>
    <VisuallyHidden>（必須）</VisuallyHidden>
  </Field.Label>
  <Input
    type="date"
    aria-labelledby="startDate-label"
    aria-required="true"
    aria-invalid={!!errors.startDate}
    aria-describedby={errors.startDate ? "startDate-error" : undefined}
  />
  <Field.ErrorText id="startDate-error" role="alert">
    {errors.startDate?.message}
  </Field.ErrorText>
</Field.Root>

// 確認ダイアログ
<Dialog.Root>
  <Dialog.Content
    role="alertdialog"
    aria-labelledby="dialog-title"
    aria-describedby="dialog-description"
  >
    <Dialog.Title id="dialog-title">シフト募集を作成しますか？</Dialog.Title>
    <Dialog.Description id="dialog-description">
      作成後、スタッフに通知が送信されます。
    </Dialog.Description>
  </Dialog.Content>
</Dialog.Root>
```

### エラーハンドリング

- エラー発生時は該当フィールドにフォーカス移動
- スクリーンリーダー向けにrole="alert"で即座に読み上げ
- エラーメッセージは入力フィールドの直下に表示

## 4. インタラクション仕様

### 日付入力

| 操作 | 動作 |
|------|------|
| フィールドタップ/クリック | ネイティブ日付ピッカー表示 |
| 日付選択 | ピッカー閉じる、値反映 |
| 不正な値 | バリデーションエラー表示 |

### バリデーション

| タイミング | 動作 |
|------------|------|
| blur時 | 該当フィールドのみバリデーション |
| submit時 | 全フィールドバリデーション |
| エラー時 | フィールド赤枠 + エラーメッセージ |

### バリデーションルール

```typescript
// 既存schema.tsより
const rules = {
  startDate: "必須、YYYY-MM-DD形式",
  endDate: "必須、開始日以降",
  deadline: "必須、開始日より前",
};
```

### 送信フロー

| ステップ | 動作 |
|----------|------|
| 1. 作成ボタン押下 | バリデーション実行 |
| 2. バリデーション成功 | 確認ダイアログ表示 |
| 3. ダイアログ「作成する」 | ローディング表示、API呼び出し |
| 4. 成功 | 一覧画面遷移、Toastで成功通知 |
| 5. 失敗 | Toastでエラー通知、フォーム維持 |

### ボタン状態

| 状態 | 表示 |
|------|------|
| デフォルト | teal背景、白文字 |
| ホバー | teal.600背景 |
| フォーカス | フォーカスリング表示 |
| 送信中 | ローディングスピナー、disabled |
| disabled | opacity: 0.4、クリック不可 |

### トランジション

- フォーカス移動: 150ms ease
- ダイアログ表示: 150ms ease-out（フェードイン）
- エラー表示: 即座に表示（アクセシビリティ考慮）

## 8. レスポンシブ対応

| ブレークポイント | レイアウト |
|-----------------|------------|
| base (0-767px) | 縦積み、日付フィールドは100%幅 |
| md (768px+) | 開始日・終了日は横並び2カラム |

### モバイル最適化

- 日付入力はネイティブ`type="date"`を使用
- タップ領域は最小44x44px確保
- フォーム全体がスクロール可能
- キーボード表示時も送信ボタンが見える配置

---

## 9. エッジケース

| ケース | 対応 |
|--------|------|
| 既に同期間の募集が存在 | API側でエラー、Toastで通知 |
| 過去日を選択 | 許容（過去期間の募集も作成可能と想定） |
| 長期間（1ヶ月超）の募集 | 許容（バリデーションなし） |
| ネットワークエラー | Toastでエラー通知、再試行可能 |

---

## 10. 実装優先度

1. **P0（必須）**: 基本フォーム、バリデーション、送信機能
2. **P1（重要）**: 確認ダイアログ、アクセシビリティ対応
3. **P2（あると良い）**: アニメーション、ヘルパーテキスト充実
