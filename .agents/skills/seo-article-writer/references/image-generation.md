# 記事画像生成ガイド

ArticleSiteの記事画像を作るときに使います。
先に `public/sample-touch.png` を目で確認します。

## 目指すタッチ

`sample-touch.png` の方向性に合わせます。

- 白背景。
- やわらかい薄いミント色の背景形状。
- 太めの黒い手描き線。
- ティール色の差し色。
- 店長、スタッフ、スマホ、カレンダー、チェック、シフト表などの小さなお店らしい要素。
- 写真風にしない。
- 3Dにしない。
- 文字を入れすぎない。
- 画像内の日本語文字は原則使わない。崩れやすいため、記号、線、チェック、丸、表の抽象行で表す。

プロンプトの土台：

```text
白背景の日本語SaaSヘルプ記事向け手描きイラスト。黒い丸みのある太線、ティールの差し色、薄いミント色のやわらかい背景形状。小さなお店の店長とスタッフ、スマホ、カレンダー、シフト表、チェックアイコン。親しみやすく実務的。文字なし、写真風なし、3Dなし、暗い背景なし。
```

## 何を画像にするか

画像は、本文の理解を助ける概念だけに使います。

向いている題材：

- 希望回収からシフト確定までの流れ。
- LINE、紙、Excel、アプリの違い。
- 未提出チェック。
- 提出期限の逆算。
- シフト表の整理。
- 店長とスタッフのやり取り。
- 通知や共有の流れ。

向かない題材：

- なんとなく明るい雰囲気。
- 本文と関係のない人物カット。
- `効率化` や `成功` のような抽象概念だけの画像。
- 本文の説明を薄く繰り返すだけの画像。

## 枚数と配置

目安：

- 短い記事：0〜1枚。
- 標準記事：1〜2枚。
- 長い比較・手順記事：2〜4枚。

配置：

- 画像で説明したい段落の近くに置く。
- 画像の前後に、何を見るべきかが分かる文を置く。
- 画像だけを連続させない。
- 比較画像は、比較表や判断基準の近くに置く。

## サイズと形式

推奨生成サイズ：

- ヒーローまたは横長図解：1200x800 または 1024x768。
- 正方形の挿絵：768x768 または 1024x1024。
- 本文で300px程度に表示する画像：元画像は幅600px以上。

ArticleSiteでの表示幅：

- `heroImageWidth`：280〜340。
- 本文の左右寄せ画像：280〜360。
- 中央図解：480〜720。

ファイル：

- 記事ディレクトリに置く。
- 編集用にPNGを残してよい。
- MarkdownではWebPを優先する。
- ファイル名は小文字のハイフン区切りにする。
- WebPは可能なら200KB未満を目指す。

変換例：

```bash
sips -Z 1200 input.png --out resized.png
cwebp -q 88 resized.png -o output.webp
```

確認：

```bash
sips -g pixelWidth -g pixelHeight output.webp
ls -lh output.webp
```

## altとキャプション

altは、記事内での画像の役割を書きます。
見た目の列挙だけにしません。

良い例：

```md
LINEで集めた希望を、未提出チェックとシフト表に分けて整理する流れ
```

弱い例：

```md
スマホとカレンダーのイラスト
```

キャプション：

- 読者に見てほしいポイントを1文で書く。
- altと同じ文を繰り返さない。
- キーワードを詰め込まない。

## 画像QA

完了前に確認します。

- sample-touchの白背景、黒線、ティール差し色に合っている。
- 画像内に崩れた文字がない。
- 本文の近くで説明している内容と一致している。
- altとキャプションが本文とずれていない。
- スマホで大きすぎない。
- WebPの容量が重すぎない。
