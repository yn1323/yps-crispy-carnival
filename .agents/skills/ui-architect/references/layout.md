# レイアウト・視覚階層

ページ骨格・グリッド・スペーシング・タイポ・カラー・elevation・モバイル対応の判断材料。

## ページ骨格パターン

### 中央1カラム
- **用途**：単一フォーム、記事、ウィザード、ログイン
- **幅**：フォーム=480〜640px、本文記事=640〜720px
- **理由**：1行40〜80字（日本語20〜40字）が読みやすい上限
- **垂直リズム**：セクション間 48〜96px

### ヘッダー + 本体
- **用途**：詳細表示、ダッシュボード、リスト
- **ヘッダー**：パンくず、タイトル、アクション（右）
- **本体**：12カラムグリッド or 1カラム
- **粘着ヘッダー**：スクロール時に主要操作が必要なら sticky、それ以外は流す

### サイドナビ + 本体
- **用途**：管理ツール、設定、長いセクション分割が必要
- **サイドナビ幅**：240〜280px（折り畳めるとより良い）
- **モバイル**：BottomMenu or ハンバーガーに格納
- ※ このプロジェクトのMVP段階ではSideMenuは含めない

### 3ペイン（左ナビ・中本体・右プロパティ）
- **用途**：エディタ、メールクライアント、選択 + 詳細
- **左**：240px / **右**：280〜320px
- **狭幅で右ペインを畳む**：< 1280px で右ペインをDrawerに

### マスタ・ディテール（一覧 + 選択 → 詳細）
- **用途**：受信箱、CRM、リスト型データ
- **モバイル**：一覧 → タップで詳細画面、戻るで一覧

### ダッシュボード
- **層構造**：上=KPI（4〜6カード）、中=推移グラフ（1〜3）、下=詳細テーブル
- **理由**：F字スキャン、左上に最重要、深掘りは下へ
- **カード幅**：揃えすぎない（重要なものを大きく）

### モバイル特有
- **下部CTA固定**：主操作が常に親指届く位置
- **BottomSheet**：モーダル代替として使う（フルスクリーンより手前で完結）
- **タブ下部**：3〜5項目、それ以上はメニュー化
- **親指ゾーン**：画面上半分は届きにくい → 重要な操作は下半分

## グリッド・スペーシング

### 8ptグリッド（厳守）
許容値：`0, 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96`

これ以外（13px, 17px, 22px等）は使わない。Chakraトークン（`p={4}` = 16px）を使えば自然と従う。

### 用途別の標準値

| 用途 | 値 |
|---|---|
| アイコンとテキストの間 | 4〜8 |
| ボタン内padding（縦） | 8〜12 |
| ボタン内padding（横） | 12〜24 |
| フォームフィールド間 | 16〜24 |
| カード内padding | 16〜24 |
| カード間 | 16〜24 |
| セクション間 | 48〜80 |
| ページ外周padding（mobile） | 16 |
| ページ外周padding（desktop） | 24〜48 |

### 12カラムグリッド
- ガター：16〜24px
- 推奨配分：6+6（並列）、8+4（メイン+サブ）、9+3（コンテンツ+メタ）、4+4+4（カード3つ）

## タイポグラフィ

### 階層は3段が上限

例（Chakra v3 token）：
- `2xl` (24px) = ページタイトル
- `lg` (18px) = セクション見出し
- `md` (16px) = 本文
- `sm` (14px) = キャプション・メタ

**サイズ・太さ・色のいずれかで差をつける**。全部一気に変えると煩い。

### 行高

- 見出し：`1.2`〜`1.3`
- 本文：`1.5`〜`1.7`
- 密集UI（テーブル）：`1.3`〜`1.4`

### 太さ

- 通常：400 / 500
- 強調：600 / 700
- ※ 100 / 900 は基本使わない（読みづらい）

### 配色

- 主要文字：`gray.900` 相当
- 副次文字：`gray.700`
- メタ・補助：`gray.500`
- 無効化：`gray.400`

3色で十分。Disabled / Inactive を青系・赤系にしない。

### 文字数（読みやすさ）

- 日本語1行：20〜40字
- 英語1行：50〜75字
- これより長いコンテンツ幅を作らない

## カラー階層

### 4カテゴリ（増やさない）

1. **ブランド**：プライマリーアクション、選択中、キーアクセント（このプロジェクトは`teal`）
2. **状態**：success/warning/error/info（規約色を使う）
3. **中立**：background, surface, border, text の階段
4. **アクセント**：稀。本当に強調したい1要素のみ

### 60-30-10ルール

ある画面の彩度配分：
- 60% 中立（背景・カード）
- 30% 補助（薄ブランド・border）
- 10% アクセント（プライマリーCTA）

**プライマリーCTAは画面に1つ**。複数あると視線が散る。

### このプロジェクトの色規約

- `teal` = ブランド・主アクション
- `orange` = 要対応・警告（注意ラベル、リマインド）
- `green` = 達成・完了（OK、提出済み）
- `gray` = 中立・完了済み・非アクティブ
- `red` = 削除・致命エラー
- `yellow` = ほぼ未使用

### コントラスト（WCAG）

- 通常テキスト：4.5:1 以上
- 大きい文字（18px+ Bold or 24px+）：3:1 以上
- グレー on 白背景の最薄値：`gray.500`（`gray.400`は本文には不可）
- 装飾枠線・borderは 1.5:1 でもOK

### Dark mode（このプロジェクトは現状非対応）

実装するなら：
- 反転ではなく専用パレット
- 純黒（#000）ではなく `#1a1a1a`〜`#222`
- elevationを「明るくして」表現する（影は薄くなる）

## elevation（影）

### 階層は2段で十分

- `base`：カード、入力欄など平面要素（影なし or `shadow.sm`）
- `raised`：浮く要素（モーダル、ドロワー、Toast = `shadow.lg`）

3段以上重ねると不自然。Modalの上にPopoverが乗る場合のみ`xl`。

### Border or Shadow？

- 静的UI、密集 → border
- 動く要素・浮く要素 → shadow
- 両方を併用するとうるさい → どちらか

## 整列・近接（Gestalt）

- **近接**：関係するものは近く（`gap-2`）、関係ないものは離す（`gap-8`）
- **整列**：左揃え or 中央揃えの統一。Mixはダメ
- **類似**：同じ役割は同じ見た目に
- **連続**：表は罫線で連続感、カードは余白で分離
- **閉合**：枠で囲んでグルーピング（borderが強すぎないように）

## レイアウトの「バランス」チェック

実装後、以下を目視確認：

- [ ] 1画面1プライマリー：最強の視覚的重みは1つだけか
- [ ] 同じ役割は同じ見た目か（カードの中身がバラバラじゃないか）
- [ ] 余白がリズムを作っているか（密と疎の区別がついているか）
- [ ] 色は意味を持っているか（装飾になっていないか）
- [ ] 階層は3段以下か（H1, H2, Bodyで十分）
- [ ] モバイル幅で破綻しないか（375px / 414px / 768px / 1024px）
- [ ] Storybookでloading/empty/error/successが見れるか

## 反復ユニット（カード・行）の設計

カードや行は何件並ぶかわからない。**1ユニットを設計したら、5件・100件・1件・0件で見え方を確認**。

- 5件：ちょうどよく見える設計が基本
- 100件：仮想スクロール検討、視覚密度
- 1件：寂しくない（場合によっては別レイアウト）
- 0件：空状態（必須）

## レスポンシブブレークポイント

Chakra v3 標準：
- `sm` 480
- `md` 768
- `lg` 992
- `xl` 1280
- `2xl` 1536

実用的には：
- ≤ 480px：スマホ縦
- 481〜768：スマホ横・タブレット縦
- 769〜1024：タブレット横・小ノート
- 1025+：デスクトップ

**モバイルファースト**：基本スタイルはスマホ向け、`md:` 以上で拡張。
