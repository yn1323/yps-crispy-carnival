# 日本語スタイル・トーン・用語ガイド

日本語ならではの「書式・トーン・語彙」で迷ったときに参照。

## ボイスとトーン（声の人格）

プロダクトが人間だったらどんな人か？を決めておく。本プロジェクトの基準は：

- **寄り添う**：困っていそうなところに先回りする
- **でしゃばらない**：ドヤらない、売り込まない、自画自賛しない
- **自然体**：堅すぎない・カジュアルすぎない中間。同僚っぽい距離感
- **正直**：機能の限界を隠さない、嘘を言わない

こう聞かれたら NG：
- 「業界最高水準の〜」→ 自画自賛
- 「革新的な〜」→ ドヤ感
- 「〜させていただきます」連発 → 距離ができる、媚び
- 「〜しちゃおう！」→ カジュアル過ぎ、お子さま扱い
- 「本機能をご利用いただくことで」→ 堅すぎ

## 敬語の使い分け

### 基本方針

- **ボタン・見出し・ラベル**：無敬語（体言止めOK）
- **説明文・本文**：です・ます調で統一。過剰な敬語は削る
- **新規登録、アンケート、問い合わせ、ヘルプ**：一段だけ丁寧に

### ボタン・見出しは「無敬語」

| ❌ 敬語過多 | ✅ 無敬語 |
|---|---|
| シフトを登録してください | シフトを登録 |
| お支払い情報のご入力をお願いします | 支払い情報を入力 |
| お名前をご記入ください | 名前（必須） |

### 本文は「です・ます」

| ❌ 重い | ✅ 軽い |
|---|---|
| 〜することができます | 〜できます |
| ご入力いただけますと幸いです | 入力してください |
| ご確認のほどよろしくお願いいたします | 確認してください |
| お支払いいただきますようお願いいたします | 支払いをお願いします |
| 〜させていただいております | 〜しています |

### 二重敬語・過剰敬語の削り方

| ❌ | ✅ |
|---|---|
| おっしゃられた | おっしゃった |
| ご利用させていただいております | 利用しています |
| ご覧になられる | ご覧になる |
| お伝えさせていただきます | お伝えします |

### "お／ご" の整理

全部に「お・ご」を付けない。本当に必要な場面だけ。

- ❌「お名前・お住所・お電話番号・お支払い」全部に「お」
- ✅「名前／住所／電話番号／支払い」（無印で十分）

お客様に直接行為を求めるときだけ「ご〜」（ご入力／ご確認／ご連絡）。

## 文の長さ・書式

### 1文の長さ

- **目安：80字以内**（日本語）。50字で読みやすく、60字で読みにくくなり始める
- 1文に**1メッセージ**（1つのことを伝える）
- 並列情報は**箇条書き**に逃がす

### タイトル・見出し vs 本文

**タイトル／見出し／サブタイトル**
- 句読点なし
- 改行や半角スペースでリズム（ただし本文中では半角スペースを入れない）
- 体言止めOK
- 例：「もう LINE でシフト調整しなくていい」

**本文**
- 句読点あり（普通に「。」「、」を使う）
- 述語で終わる（「〜できます。」「〜です。」）
- 体言止めの連打は避ける（1段落に1回まで）
- 擬音・擬態語（「ぱっと」「サクサク」）は使わない。具体で書く

### 句読点ルール

- **1文だけのラベル・見出しは句点なし**（「保存しました」）
- **2文以上の本文は各文に句点**（「変更を保存しました。次の画面に進みます。」）
- 中黒（・）は並列の強調にのみ。URL・メールアドレスには使わない
- 括弧：強調「」、引用『』、補足（ ）、注釈 ※ の使い分けを統一

### 数字・日付

- 横書きは**半角算用数字**を使う
  - ✅「2026年4月20日」「30人」「3分」「980円」
  - ❌「二〇二六年四月二十日」「三十人」
- 日付は**「2026年4月20日(月) 10:00」**（半角括弧・HH:MM）
- 価格は「月980円」「年払い8,800円」で統一。カンマは3桁ごと

### カタカナ

- **長音符は付ける**：「ユーザー」「スキャナー」「コンピューター」（「ユーザ」はNG）
- 読みにくい外来語は日本語に置換：「ダッシュボード→画面」「アラート→お知らせ」

## 漢字・ひらがなのバランス

- 目安：**漢字：ひらがな ≒ 3:7**
- 漢字が多いと「難解・堅い」、ひらがなが多すぎると「幼稚・頼りない」に見える

### 漢字をひらがなに開く典型

UIライティングでは以下は**ひらがなで書く**ことが多い。

| 漢字 | 開き |
|---|---|
| 事 | こと |
| 物 | もの |
| 時 | とき |
| 様々な | さまざまな |
| 出来る | できる |
| 頂く | いただく |
| 下さい | ください |
| 致します | いたします |
| 宜しく | よろしく |
| 予め | あらかじめ |
| 尚 | なお |
| 及び | および |
| 或いは | あるいは |

### 漢字で書きたいもの

意味が濃く、読みやすさを損なわない漢字は使う：「変更」「保存」「登録」「確認」「削除」など。

## やめるべき表現パターン

### "〜することができます" → "〜できます"

ほぼ全部短縮できる。

- ❌「シフトを確認することができます」
- ✅「シフトを確認できます」

### "〜いたします／〜させていただきます" の連打

一段だけでいい。連発するほど敬意は伝わらない（むしろ距離ができる）。

- ❌「ご登録いただきますと、翌日にメールをお送りさせていただきます」
- ✅「登録後、翌日にメールをお送りします」

### 冗長な接続

- ❌「〜における」→ ✅「〜の」「〜での」
- ❌「〜に関しまして」→ ✅「〜について」
- ❌「〜のために」→ しばしば削れる

### 不要な「こと／もの」

- ❌「機能を使うことができます」→ ✅「機能を使えます」
- ❌「確認が必要となります」→ ✅「確認が必要です」

### "一切／確実に／完璧に／必ず"

断定しすぎると嘘っぽくなる・責任を問われる。

- ❌「情報が一切漏れることはありません」
- ✅「暗号化して保管しています」

## 禁止・要注意表現

### 上から目線／責める語

- 「正しく入力してください」「不正です」「失敗しました」
- 「当然」「簡単にできるはずです」

### ドヤ感・自画自賛

- 「業界をリードする」「自信を持ってお届け」「最高水準」「最先端」
- 「革新的な」「画期的な」「スマートな」

### AI-slop な修飾語

- 「シームレスな」「効率化」「最適化」「ソリューション」「ワークフロー」「イノベーション」「パワフル」
- 中身が空洞に見える。具体に置き換える

### カタカナ用語の置換表

小規模店舗向けプロダクトでは特に重要。

| カタカナ | 日本語に |
|---|---|
| ダッシュボード | 画面・一覧 |
| アラート | お知らせ |
| プッシュ通知 | お知らせ |
| インテグレーション | 連携 |
| ワークフロー | 手順・段取り |
| ソリューション | 〜できる／解決 |
| マイグレーション | 移行・引っ越し |
| エクスポート | 書き出し |
| インポート | 取り込み |
| コンバージョン | 成約・申し込み |
| ローンチ | 公開・開始 |
| リテンション | 続けてもらう |
| オンボーディング | 使い始めの案内 |
| サジェスト | 候補 |
| バリデーション | 入力チェック |

ただし**業界で定着している言葉は無理に置換しない**（SaaS、LP、URL、メール、LINE など）。読み手が日常で使っている言葉を優先する。

## 小規模店舗・個人事業主向けの語彙の感覚

### 効く言葉

- **具体の業務動作**：「シフト表を作る」「スタッフに一斉送信」「レジと連携」
- **時間・人数の具体**：「30人分」「3分で」「毎週の作業」「月末の締め」
- **業界の日常語**：「まかない」「締め」「閉め作業」「発注」「仕込み」「棚卸し」
- **業態の呼び方**：「カフェ」「居酒屋」「美容室」「クリニック」など具体名

### 避けたい言葉

- 「経営者」「事業者」→ 硬い。「店長さん」「オーナーさん」のほうが自然
- 「小さなお店」「零細」→ 見下しの含意。✅「少人数のお店」「個人店」
- 「業務効率化」「DX」→ 実感がない。✅「毎週の作業が減る」「手書きをやめる」
- 「人材」→ ✅「スタッフ」「メンバー」

### 避けたい比喩・修飾

- 「AIが自動で」「かしこい」→ 過剰な期待を作る。実態と揃える
- 「革命」「イノベーション」→ 大げさ。✅「今までより早くできる」
- 「神」「激」などスラング → 小規模店舗オーナー層には刺さらない

## 用語の一貫性ルール

同じ機能を違う名前で呼ばない。プロジェクトの用語集を作って共有する。

例（シフト管理プロダクトの場合）：

- **シフト提出** vs シフト申請 vs シフト希望
- **確定** vs 登録 vs 保存
- **スタッフ** vs メンバー vs 従業員
- **店舗** vs お店 vs 会社

1つに決めたら、LP・UI・ヘルプ・メール全部で揃える。

## アクセシビリティ

- 視覚位置で指示しない：❌「下のボタン」✅「『次へ』ボタン」
- 色だけで情報を伝えない：赤文字だけでエラーを示さず、アイコン・テキストも併用
- 画像の代替テキスト（alt）は**画像の意味**を書く。装飾画像は空文字にする

## 半角スペースのルール

- **本文の文字間に半角スペースを入れない**（日本語の本文ではスペースを打つ必要がない）
- **見出し・タイトルだけ**区切りとしての半角スペースを使ってよい（句読点の代わり）
  - ✅（見出し）「もうLINEで シフト調整しなくていい」
  - ❌（本文）「もう LINE で シフト調整 しなくて いい です。」
- 数字と単位の間は詰める：「30分」「980円」「700店舗」

## 迷ったときの判断順

1. 読み手が**一発で**同じ意味に取れるか？
2. 具体（数字・動作・固有名）があるか？
3. 敬語が重すぎないか？1段だけに抑えたか？
4. カタカナを日本語に置き換えられないか？
5. 同じ語が別画面で違う呼ばれ方をしていないか？
6. 嘘・誇張が混ざっていないか？
