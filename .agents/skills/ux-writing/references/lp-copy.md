# LP（ランディングページ）の文章

LP・紹介ページ・料金・FAQ・訴求文を書くときに参照。

## 大前提：ユーザーは3秒で去る

訪問から最初の3秒で「自分に関係あるか」を判断される。ヒーロー（ファーストビュー）だけで勝負が決まると思って書く。ここで伝わらないと下を読んでもらえない。

ヒーローで絶対に言うこと：

1. **これは誰のためのものか**（ペルソナ・用途）
2. **何ができるようになるか**（ベネフィット）
3. **次に何をすればいいか**（CTA）

この3つが5秒で読める・わかる・押せる、が合格ライン。

## ヒーロー（H1・サブヘッド・CTA）の公式

### H1（キャッチコピー）

**15〜25字**で、**1つの**ベネフィットを言い切る。機能名や社名は入れない。

公式パターン（使いやすい順）：

| パターン | テンプレ | 例 |
|---|---|---|
| 結果訴求 | 〜が／〜できる | 30人分のシフトが3分で組める |
| 時間短縮 | 〜分で／〜を〜時間減らす | 毎週のシフト作成を5時間減らす |
| 置換 | もう〜しなくていい | もうLINEでシフト調整しなくていい |
| 解放 | 〜から解放される | Excelのシフト表から卒業 |
| 対比・新しい〜 | 新しい〜／〜のための〜 | 少人数の店のためのシフト管理 |
| 約束 | 〜を、〜に | めんどうなシフト作成を、3分に |

NG パターン：

- **機能名だけ**（「AIシフト自動生成機能」）
- **自画自賛**（「業界最高水準のシフト管理」）
- **抽象的な動詞**（「シフト管理を革新する」）
- **二重の修飾**（「誰でもかんたんにスマートにシフトが組める」）

### サブヘッド（サブキャッチ）

H1の直後、1〜2行でH1の**「どうやって」「どのくらい」**を補足する。H1では伝えきれなかった「読み手が気になる1点」を書く。

- H1: もうLINEでシフト調整しなくていい
- サブ: 希望はスタッフが直接入力 あなたはパズルを組むだけ

サブヘッドは**CLAUDE.mdの「テキストガイドライン」通り、句読点なし・改行でリズム・体言止めOK**。

### CTA（ボタン）

動詞始まり、**結果を書く**。「送信する」「登録する」は避け、**押した後に何が起きるか**を書く。

| ❌ 動詞のみ | ✅ 結果あり |
|---|---|
| 送信する | 無料で相談する |
| 登録する | 無料で使い始める |
| ダウンロード | 資料を受け取る |
| 申し込む | 30秒で申し込む |
| お問い合わせ | 料金を相談する |

**CTAの直下に不安解消**を1行添える（マイクロコピー）：

- 「クレジットカード不要 いつでも解約OK」
- 「30秒で登録 メールだけで始められます」
- 「導入相談は無料です」

## セクションの典型構成（上から順に）

いわゆる「共感→ベネフィット→実績→信頼→クロージング」の流れ。AIDMA・PASの焼き直しだが、**SaaSではこの順が一番効く**。

1. **ヒーロー**（H1 + サブ + CTA + 信頼の一言／ロゴ）
2. **現状の課題への共感**（「こんなことに困っていませんか？」）
3. **主要ベネフィット 3つ**（1ベネフィット＝1セクション、見出しは結果を書く）
4. **社会的証明**（導入店舗数・ロゴ・ユーザーの声・メディア掲載）
5. **機能・使い方**（ベネフィットを裏付ける具体動作を3〜5個）
6. **料金**（プランごとの"できること"で差を書く）
7. **FAQ**（実際に質問される上位5〜8個）
8. **最後のCTA**（ヒーローと同じCTAを再掲＋不安解消の一言）

ロングLPで「何度も同じCTAが出てくる」のは正しい。3〜4回出しても押しつけにならない（出す場所は固定される）。

## 特徴（Feature）→ベネフィット（Benefit）変換

書き手が語りがちな「特徴」を、読み手が欲しい「ベネフィット」に書き直す。

公式：**「〜という機能」ではなく「〜できる・〜になる」で終わらせる**。

| ❌ 特徴（売り手視点） | ✅ ベネフィット（読み手視点） |
|---|---|
| AIによる自動シフト生成 | 希望を入れるだけで 下書きが3分で作れる |
| マルチデバイス対応 | スマホ PC タブレット どこからでも使える |
| リアルタイム同期 | 変更がすぐ全員に届く |
| 充実したレポート機能 | 労働時間の偏りが一目でわかる |
| クラウドベース | バックアップや更新は私たちの仕事 あなたは使うだけ |
| 24/7 サポート | 困ったらいつでも相談できる |

特徴を消す必要はない。ベネフィットの**裏付け**として特徴を添える（「〜できます（◯◯機能）」）。

## 共感セクション（課題の提示）

読み手に「これ、うちのことだ」と思わせる。抽象的な悩みではなく、**具体的なシーン**を書く。

NG：

- 「業務が非効率ですか？」
- 「人材不足に悩んでいませんか？」

OK：

- 「毎週日曜の夜 2時間かけてシフト表を作っていませんか」
- 「LINEのシフト希望が流れて 誰が出せるのか探している」
- 「エクセルを開くたびに フォーマットが崩れていませんか」

**「この文を読んで、自分のことだと5秒で気づけるか？」**が判定基準。

## 社会的証明（ソーシャルプルーフ）

SaaSは無形。信頼を数と事実で代替する。以下のどれかは必ずLPに入れる。

- **導入店舗数／アカウント数**（例: 700店舗で使われています）
- **継続率・満足度**（例: 継続率98％）
- **有名・権威ある導入先のロゴ**
- **ユーザーの声（動画＞写真＞テキスト）**：本人の顔・肩書き・店舗名が出るほど強い
- **第三者の受賞・メディア掲載**

ユーザーの声テンプレ：

```
「[課題]に困っていたが、[具体的な変化・数値]が実現できた」
ーー [店舗名] [役職] [氏名]さん
```

例：「毎週のシフト調整に2時間かかっていたのが、15分で終わるようになった」ーー CAFE HOGE 店長 山田さん

## 料金セクション

- **プラン名は"機能名"ではなく"使う人"で**（例: スタータープラン＝個人店向け）
- プランごとの差は**「できる／できない」ではなく「何人規模まで」「どこまでやるか」**で
- **月額と年額の表記ルールを統一**（例: 月980円（年払い8,800円）※税別）
- 「契約期間の縛りなし／いつでも解約」を目立たせる

## FAQ

LPのFAQは**離脱直前の不安を拾う場**。以下の質問は用意したい：

- 料金・支払い（無料トライアル？いつ課金？）
- 解約方法（縛り？違約金？）
- 導入のしやすさ（どのくらいで使える？）
- データ移行（今のエクセルから移せる？）
- サポート（困ったら誰に聞ける？）
- セキュリティ（データはどこに保管？）

回答は「はい／いいえ」で終わらせず、**1文目で結論、2文目で補足、次の行動があればCTA**。

```
Q. クレジットカードは必要？
A. 登録だけなら不要です。無料期間が終わって継続するときに登録してください。
→ 無料で試してみる
```

## マイクロコピー（LP内の短い添え書き）

マイクロコピーで大きく変わる事例：

- 「Join Us!」→「MAKE MONEY FLIPPING WEBSITES（サイトを売って稼ぐ）」でCVR 33%改善（Empire Flippers）
- 「資料請求」→「無料で課題を解決する」でCVR 1.8倍
- CTA直下に「1分で完了／クレカ不要」を足すだけでフォーム通過率が大幅改善

**使える定型表現（心理的ハードル下げ）**：

- 「無料で」「1分で」「カードいらない」「今すぐ試す」「かんたん30秒」
- 「いつでもキャンセルOK」「データはすぐ消せます」「あとからプラン変更OK」
- 「個人情報の入力なし」「メールだけで」

ただし**嘘はNG**。「かんたん」と書くなら本当に1分で終わる導線にする。

## 小規模店舗・個人事業主に向けて書くとき

想定読者像：

- テック用語に慣れていない（カタカナが並ぶと読み飛ばす）
- 忙しい（読むのは5秒、やるのは夜中）
- 失敗したくない（無料で試せるかが効く）
- 自分の業界の具体シーンで語られると信頼する

効くパターン：

- **業界シーンを固有名詞で**：「ランチの仕込み中にスマホで承認」「閉店後に翌週のシフトを決める」
- **具体数字**：「30人分」「3分」「月980円」
- **比較対象は「今使ってるもの」**：Excel、LINE、紙、手書き
- **テック言葉を業務言葉に**：「データ同期」→「全員に届く」／「ダッシュボード」→「画面」／「インポート」→「取り込み」

避けるパターン：

- 「DX」「効率化」「最適化」「イノベーション」「シームレス」
- 「御社」「貴店」（読み手と距離ができる）
- 「AI」「自動化」は強すぎるので、実態と合わせて使う

## 仕上げ前のLPチェック

1. ヒーローだけで **「誰が何を得られるか」が伝わる**
2. ヒーローの H1・サブ・CTA に**冗長な重複がない**
3. 共感セクションの例が**読み手の具体のシーン**になっている
4. ベネフィットは **数値・時間・対象人数** のいずれかを含む
5. 社会的証明が**最低1種類**入っている（ロゴ／数字／声）
6. CTAは **押した後に何が起きるか** が書かれている
7. FAQ は**実際の離脱原因**に答えている
8. 同じ機能に**違う呼び方**が混ざっていない
