# カフェのシフト作成でよくある悩みと解決方法

カフェのシフト作成は、人数を埋めるだけではうまく回りません。
朝の仕込み、ランチの混雑、夕方の入れ替わり、締め作業など、時間帯ごとに必要な人と役割が変わるからです。

解決の近道は、営業日ごとではなく「時間帯ごとの必要人数」「入れるスタッフ」「変更連絡の扱い」を先に決めて、希望回収に反映することです。
この3つをそろえると、作りながら悩む時間を減らせます。

## カフェのシフトで悩みが増える理由

カフェでは、忙しい時間が1日の中で何度か分かれます。
朝は開店準備、昼は注文と提供、夕方は片付けや翌日の準備が重なります。
同じ2人勤務でも、慣れているスタッフがいるかどうかで回りやすさが変わります。

よくある悩みは、次のようなものです。

- ランチだけ人が足りない
- 学生スタッフの予定が直前に変わる
- 土日だけ希望が偏る
- 新人だけの時間帯ができてしまう
- 変更連絡がLINEや口頭に分かれて埋もれる

どれも「人が足りない」の一言で片づけると、毎月同じところで止まります。
まずは、どの時間帯で、何が足りていないのかを分けて見ます。

## まずピーク帯ごとの必要人数を決める

最初に決めるのは、1日全体の人数ではありません。
時間帯ごとの必要人数です。

たとえば、同じ平日でも朝と昼では必要な動きが違います。
昼だけ混むお店なら、ランチ帯を厚くします。
モーニングが強いお店なら、開店前後に慣れたスタッフを置きます。

| 時間帯 | 起きやすい悩み | 先に決めること |
| --- | --- | --- |
| 開店前後 | 仕込みと接客が重なる | 慣れた人を最低1人入れる |
| ランチ | 注文、会計、提供が集中する | 必要人数と役割を分ける |
| 夕方 | 学生スタッフの入り時間がずれる | 何時から戦力に数えるか決める |
| 閉店前 | 片付けと翌日準備が残る | 締め作業ができる人を入れる |

この表は、きれいに作る必要はありません。
まずはメモで大丈夫です。
「月曜ランチは3人」「土曜は開店から2人」のように、悩む時間帯だけでも書き出します。

## スタッフ属性ごとのズレを見込んでおく

カフェでは、学生、主婦・主夫、掛け持ちスタッフなど、働ける時間が違う人が混ざりやすいです。
そのため、希望をそのまま並べるだけでは、シフト表に入れる段階でズレが出ます。

たとえば、学生スタッフは授業や試験で予定が変わりやすいことがあります。
主婦・主夫スタッフは、日中に入りやすい一方で、夕方以降は難しい場合があります。
掛け持ちスタッフは、別の勤務先の予定が出るまで確定しづらいことがあります。

大切なのは、スタッフを分類して決めつけることではありません。
「この人はいつ入りやすいか」「どの時間帯なら安心して任せられるか」を、シフト作成前に見える形にしておくことです。

| スタッフの状況 | 確認しておきたいこと |
| --- | --- |
| 学生スタッフ | 試験期間、授業後に入れる時間、土日の入り方 |
| 主婦・主夫スタッフ | 平日昼の入りやすさ、長期休み中の変化 |
| 掛け持ちスタッフ | 確定できる日、変更が出やすいタイミング |
| 新人スタッフ | 1人にしない時間帯、教えられるスタッフの有無 |

新人スタッフがいるときは、人数だけで判断しないほうが安心です。
「3人いるけれど、全員が慣れていない」という状態は、実際には足りていないことがあります。

## 希望回収では「入れる時間」と「条件」を分けて聞く

希望を集めるときに、「入れる日だけ」を聞くと、あとから確認が増えます。
カフェでは、日付よりも時間帯が大事になることが多いからです。

聞く内容は、次の3つに分けます。

1. 入れる日
2. 入れる時間帯
3. 条件つきなら、その条件

たとえば「土曜入れます」だけでは、開店から入れるのか、昼からなのか、閉店まで残れるのかが分かりません。
希望回収の時点で「10時から15時」「17時以降」「閉店作業は不可」のように分けて聞くと、シフト表へ写すときに迷いにくくなります。

早番・遅番のような勤務区分で聞きたい場合は、[2交代・3交代・4交代シフトの希望回収](/articles/shift-type-request-guide)も参考になります。
お店の呼び方に合わせて「モーニング」「ランチ」「ディナー」「締め作業」のようにしても問題ありません。

## 変更連絡は受ける場所と締切を決める

カフェのシフトでつまずきやすいのが、あとから来る変更連絡です。
LINE、口頭、メモ、別のスタッフ経由など、連絡の入り口が分かれると、どれが最新か分からなくなります。

変更をゼロにするのは難しいです。
だからこそ、受け方を決めておきます。

- 変更連絡はどこで受けるか
- いつまでなら調整対象にするか
- 反映したらどこに印をつけるか
- 確定後の変更は誰が判断するか

LINEで連絡を受ける場合は、希望回収と変更連絡がトークの中に混ざりやすくなります。
探す時間が増えているなら、[LINEで集めたシフト希望を、あとから探していませんか](/articles/shiftori-line-workflow)のように、提出場所を1つにまとめる考え方から見直すと整理しやすくなります。

## 作成前に確認したいチェックリスト

シフト表を作り始める前に、次の項目だけ確認します。
全部を完璧にするより、毎回同じ順番で見るほうが続けやすいです。

- ピーク帯ごとの必要人数を書いたか
- 慣れたスタッフが必要な時間帯を決めたか
- 新人だけになる時間帯がないか
- 学生スタッフの試験期間や授業後の入り時間を確認したか
- 土日やランチ帯に希望が偏っていないか
- 変更連絡の受ける場所と締切を伝えたか
- 確定シフトを共有する方法を決めたか

ここまで見てからシフト表に入れると、「作りながら足りないところを探す」状態を減らせます。
足りない時間帯が先に分かれば、早めにスタッフへ相談できます。

## シフトリでまとめると楽になること

紙やLINE、Excelでも、カフェのシフトは作れます。
ただ、希望回収、未提出確認、変更反映、確定共有を別々に行うと、確認する場所が増えていきます。

シフトリでは、スタッフから集めた希望を一覧で確認できます。
誰が提出しているか、どの時間帯に入れるかを見ながら、シフト作成へ進められます。
確定後の共有まで同じ流れで扱えるため、送ったつもり、直したつもりを減らしやすくなります。

今のやり方をすべて変える必要はありません。
まずは、次のシフト1回分だけで大丈夫です。
希望回収から確定共有までを同じ場所で進めると、どこが楽になるかを比べやすくなります。

## まとめ

カフェのシフト作成で悩みやすいのは、人数だけでなく、時間帯と役割が毎日変わるからです。
まずピーク帯ごとの必要人数を決めます。
次に、スタッフごとの入りやすい時間と、任せられる役割を見ます。

希望回収では、日付だけでなく時間帯と条件を分けて聞きます。
変更連絡は、受ける場所と締切を決めます。

この順番にするだけで、シフト表を作りながら迷う時間は減らせます。
毎月同じ悩みが出ているなら、次回の希望回収から1つだけ見直してみてください。
